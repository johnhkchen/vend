# T-058-04 — Structure: the blueprint

All changes are confined to `examples/templates/hackathon-seed/`. No `src/` (vend) changes.

## Files created

### `examples/templates/hackathon-seed/src/pages/board.astro` (NEW)

The `/board` route. Build-time inline of `.vend/work-graph.svg` with an honest-empty fallback.

Shape (frontmatter + template):

```
---
import { readFileSync } from "node:fs";
import { join } from "node:path";

// Read the SHIPPED `vend svg` output at build time. `.vend/work-graph.svg` lives at the
// project root (where `vend svg` writes and where the Astro build runs). Missing file =
// no board cast yet → honest-empty panel. Never throws → build stays green.
const svgPath = join(process.cwd(), ".vend", "work-graph.svg");
let boardSvg: string | null = null;
try {
  boardSvg = readFileSync(svgPath, "utf8");
} catch {
  boardSvg = null; // no board yet — render the empty-state panel
}
---

<!doctype html>
<html lang="en">
  <head> … <title>Board — vend</title> </head>
  <body style="margin:0; background:#fafafa;">
    <main style="…centered, padded…">
      <header> <h1>Work-graph board</h1>
        <p>A read-only picture of the work — rendered by <code>vend svg</code>. <a href="/">← app</a></p>
      </header>
      {boardSvg
        ? <section aria-label="vend work-graph board" set:html={boardSvg} />
        : <section> /* empty-state: "No board yet — run `npm run board` (vend svg)…" */ </section>}
    </main>
  </body>
</html>
```

Key properties:
- `set:html` inlines the trusted, locally-generated SVG (no sanitizer dep needed — it is our own
  renderer's deterministic output).
- The `try/catch` guarantees the page never throws at build → AC3 "build stays green" with or without
  a board.
- Read-only: no writes, no renderer call, no client JS → AC2.
- A link back to `/` so the designer toggles between app and board in the same browser.

## Interfaces / boundaries

- **Input contract**: `board.astro` depends only on the file path `.vend/work-graph.svg` relative to
  `process.cwd()` — the seam's documented default (`DEFAULT_SVG_DIR` / `DEFAULT_SVG_FILENAME`). No
  import of vend source; the seed stays a standalone Astro project.
- **No coupling to vend internals**: the page reads a file, not a module. `vend` is invoked only via
  the CLI in the `board` npm script.
- **vend gate untouched**: nothing under `src/`; root tsconfig includes only `src/`; seed has no
  tests. `bun run check` unaffected (verify by running it).
