When a cast trips an andon, the run log records the outcome bucket (gate-failed / budget-exhausted / timed-out) but NOT the specific reason — which gate stopped it, or the spent-vs-ceiling tokens. `castPlay` prints that reason to stdout (the `stopReason` line) but never threads it onto `RunSummary` or the run record. So `vend audit` and the consistency probe cannot tell an honest-empty refusal (the IA-4 gate correctly abstaining) apart from a real budget blow-out — both land as a bare non-success outcome. Surface the structured stop reason (gate name + unit, or spent/ceiling) on the run record so the ledger can separate honest abstentions from censoring. Grounded: T-019-02's sweep hit exactly this blind spot — expand's honest-empty folds invisibly into budget-exhausted because the reason is stdout-only.
