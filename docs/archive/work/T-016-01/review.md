## How it works

`fragment → Signal` is the new top of the pipeline (`fragment → expand → signal → propose → epic →
decompose`). BAML owns **shape** (`Signal` is a closed-enum poka-yoke); the pure core owns
**meaning** via three value-ordered gates that return the engine's play-agnostic `GateVerdict`:

1. **honest-empty (IA-4)** — blank `what`+`why` ⇒ STOP. The model's abstention when a fragment
   grounds no demand; surfaced as a STOP so the (T-016-02) effect stages nothing — "not manufactured
   busywork," P7 no-partial. Checked first.
2. **read-never-invent (PE-1)** — content present but `grounding` blank ⇒ STOP. The pure proxy for
   "read, not invented": a signal that cites no real state is speculation, refused. (PE-2 cite-the-
   source as a poka-yoke, not a semantic oracle.)
3. **value-link (PE-3)** — `advances` empty ⇒ STOP; a `N\d+`/non-goal or dangling `P\d+` ⇒ STOP;
   free-text core-feature prose passes. "Links to a *real* value," reusing propose-core's bounds
   logic over the live charter.

`renderSignalRow()` emits a faithful `demand.md` table row (tier member→alias, every field round-
tripped) for T-016-02's staging effect to write.

## Open concerns / flags for the human

1. **`budget` is a model-proposed string, not the measured E-013 envelope (yet).** By design: the
   core stays ledger-free; T-016-02's assemble/effect is where the measured pre-fill (E-013
   `recalibrate`) should replace the model's rough default. The epic's "pre-filled measured budget"
   AC is **half-met here** (the field exists + renders) and **completed in T-016-02**. Worth a
   conscious check when wiring the effect.
2. **honest-empty conflates abstention with SAP-garbage** — both reach a blank `what`/`why` and STOP
   at `honest-empty`. This is intentional (a blank reply, however it arose, stages nothing), and the
   bridge test still distinguishes garbage (parse-reject) from a blank-but-valid reply at the BAML
   layer. No action needed; noted for transparency.
3. **read-never-invent is a presence check, not a groundedness oracle.** A pure gate cannot verify
   that `grounding` actually corresponds to real project state — it only refuses a signal that names
   *nothing*. The semantic "is this citation real?" check is the human's at pull time (staging-only,
   PE-1: Vend proposes, the human pulls). This is the honest limit of a pure gate, consistent with
   how propose-core's value gate works.
