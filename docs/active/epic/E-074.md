_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

A live field report (2026-07-13) drove vend from inside a sandboxed shell: fresh `vend init`, early charter, then `vend steer`. Three things failed dark. The claude executor needed Keychain access for its macOS config store; the sandbox denied it, and the failure surfaced as a raw error rather than a refusal. The retry was then funded at TIER_BUDGET.standard (12.5k, a clearing-work denomination) against steer's ~400k measured envelope that vend's own shelf prints — a ceiling ~30x below the play's known floor, silently. `claude on PATH` was green, so the preflight lied by omission: reachability-on-PATH is not dispensability-from-here. The play is to make the cold-start path fail SAFE across three named seams: (1) `vend doctor` proves the executor can actually DISPENSE from THIS environment — a near-zero-cost reachability probe (auth/config-store readable, one trivial dispense or the cheapest equivalent) as a named check carrying a fix-it hint (`claude login`; sandbox needs Keychain access); (2) an executor-unreachable failure AT cast time routes to the existing missing-capability amber andon (cast.ts:169–200, today wired only to absent MCPs) with the named cause and hint — never a raw stack, because honest refusal is already the house shape and this failure just isn't routed to it yet; (3) a counter-time UNDERFUNDING warning — when a funded ceiling is far below the play's own known envelope floor (the shelf already computes it), say so before the cast, warn-don't-block per the E-068 ruling so a deliberate thin probe stays possible but never accidental. The persona this protects — an agent inside a sandbox — is one the E-062 rehearsal never covered, and the first gesture on a fresh machine is the highest-stakes moment of the whole channel.

## Value to the design

Realizes a fail-safe cold start: environment- and funding-failures on the first funded cast are caught and NAMED before tokens burn, closing a gap where the highest-stakes gesture on the channel could fail opaquely. Advances P5 (local-first: the fresh-machine first gesture is made trustworthy for the agent-in-a-sandbox operator, not just the human on a login shell), P3 (gates-are-the-contract: an executor that can't dispense becomes a gated honest refusal on the existing andon, not an opaque environment error), and P7 (budget-is-a-hard-contract: the budget gesture is warned at the moment of misallocation, before exhaustion, not after). It strengthens the core clearing guarantee — 'you got what you paid for' — by refusing to let the first cast settle into execution when the environment can't honor it or the funding can't fund it.

## Done looks like

On a machine where the executor is on PATH but cannot dispense (e.g. config-store/Keychain denied in a sandbox), `vend doctor` reports a FAILED named executor-dispensability check with a fix-it hint and spends effectively no tokens; a `vend steer`/cast attempted in that same broken environment halts on the missing-capability amber andon with the named cause and hint (logged countable like other refusals, nothing metered) rather than emitting a raw stack; and a cast funded far below the play's shelf-computed envelope floor prints an underfunding warning before dispensing yet still proceeds if the operator continues (warn, not block). All three observable on the field-report scenario without regressing a clean-environment cast to byte-identical behavior.

## Context & constraints

Grounded in the 2026-07-13 field report and confirmed in-repo: cast.ts:169–200 already routes absent-MCP to the IA-9 amber andon (nothing rendered/dispensed/materialized, logged as `outcome: missing-capability`, usage {}, cost 0) — slice 2 extends this existing refusal path to a new cause, it does not invent one. gather.ts:51–56 defines TIER_BUDGET with `standard: 12.5k`; the shelf already computes and prints a play's measured envelope, which slice 3's warning compares against — no new measurement machinery needed. Prerequisites (name AS prerequisites, not hidden scope): slice 1 needs the executor interface to expose a cheap dispensability/reachability probe distinct from PATH presence — if none exists it must be added on the executor boundary; slice 3 needs the shelf's envelope-floor value reachable at cast/counter time. Non-goals respected: this does NOT change any TIER_BUDGET value (N-none; explicitly out of scope per signal), does NOT auto-escalate or auto-fund budgets (warn-don't-block only, honoring E-068), and does NOT do any sandbox-escape or credential handling beyond DETECTING and NAMING the failure (no N4 executor behavior, no babysitting per N2). Right-sizing (PE-7): this sits at the upper bound of one ~2-hour block — three related slices over two existing seams (doctor check, cast-time andon, counter-time warning). If DecomposeEpic finds slice 1's executor-probe prerequisite is itself substantial, slice 1 may warrant its own epic; the three slices are cohesive enough to clear as one card but should be watched at decompose time for a split.
