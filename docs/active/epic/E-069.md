_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

Add an `--agent <seat>` routing flag to the board-writing gestures — `vend chain "<signal>" --agent codex` and `vend run decompose-epic <card> --agent codex` — so the executor seat is chosen as part of the mint gesture and written into the frontmatter of every ticket the cast materializes. Today (2026-07-11) the codex seat sat underutilized and five E-068 tickets were routed by hand-editing `agent:` frontmatter AFTER mint (commit 03b58b3): the routing itself worked (5/5 codex legs cleared, recorded in `.lisa/provenance.jsonl`), but the gesture was manual, could only apply once the tickets already existed, and raced the live loop's index (the fix commit swept in loop-staged files). This matters NOW because allocating usage across the claude and codex seats is becoming routine operator work, and it should be one gesture at the counter — not a supervised sed pass racing a running loop.

## Value to the design

Realizes allocation-at-mint: the seat becomes part of the two-gesture transaction (pick + budget + seat), directly advancing P2. Because the routing lives in the board rather than in the executor, lisa's claude and codex seats simply read the `agent:` key — the board carries the decision and no gesture assumes a specific executor (P6). And usage allocation stops being supervision-after-the-fact — no racing a live loop with manual edits — which advances P4 (autonomy over babysitting). It strengthens the core clearing function: the seat a unit runs on is decided when the unit is cleared onto the board, not patched in afterward.

## Done looks like

Running a board-writing gesture with `--agent codex` produces tickets whose frontmatter carries `agent: codex` (placed after `priority:`, matching the convention lisa already reads) on every materialized ticket, with no hand editing. Omitting the flag yields output byte-identical to today's mint — no `agent:` key written. Passing an unknown seat (anything outside the known-seat list `claude | codex`) is refused with a named andon BEFORE any file is written. The E-068-style hand-routing pass is no longer necessary to place codex work.

## Context & constraints

Ground against the existing `--after` plumbing, which this flag mirrors exactly: `src/play/project-context.ts` defines `ContextSources` (the CLI-facing input, e.g. `after?: readonly string[]`) passed through to `DecomposeInputs`, and the materialize effect applies it at ticket write time while `render`/`gates` ignore it. Thread `--agent` the same way — a new optional field on `ContextSources` → `DecomposeInputs` pass-through — and have the materialize effect stamp `agent:` into frontmatter at write time. Validate the seat value against the known-seat list (`claude | codex`) using the bare-code guard pattern and refuse an unknown seat with a named andon before any file is written (do not defer to lisa validate). Prerequisites (name them, do not fold in as hidden scope): the known-seat list must be sourced from the existing seat convention, not re-invented here; and the frontmatter placement must match the boilerplate-demo hand-routing convention lisa already consumes (`agent:` after `priority:`). Out of scope: changing how lisa DISPATCHES to a seat (lisa already reads the key — this epic only writes it); adding seats beyond claude|codex; retroactively stamping seats onto already-minted tickets; and any per-ticket (as opposed to per-gesture) seat override — one seat per gesture matches the `--after` shape and keeps this to one ~2-hour feature block (PE-7). Respects N4 (Vend writes the routing onto the board; lisa still executes) and N2 (this removes a supervision gesture rather than adding a dashboard).
