## Intent — the bigger-picture play

E-046 gave the engine real concurrency (castGraph's wave dispatcher runs independent ready nodes via Promise.all), and E-047 proved it live with a survey→[propose×2]→capture-note diamond where two casts overlap. But the macro-wallet is still single-chain: spendDown threads one wallet sequentially, and within a concurrent wave each node authorizes against its own static per-node budget with no shared accounting. So two parallel casts both pass canAfford against the pre-wave balance and both spend — the P7 ceiling leaks across branches exactly where concurrency now lives. This epic makes the macro-wallet span parallel DAG branches: concurrent casts in a wave draw from and respect ONE shared envelope, so the run honors the time/token allocation it was sold no matter how the graph fans out. This matters now because concurrency just shipped and the budget invariant is the contract the whole product rests on — leaving the gap open means every graph playbook can quietly overshoot its budget.

## Value to the design

Realizes a budget hard contract that is correct under concurrency, not just under linear execution — closing the gap between the new graph substrate (E-046/E-047) and P7. It directly advances P7 (the allocated envelope is respected both ways across branches) and P3 (the hard stop becomes a real enforceable gate at wave boundaries rather than a per-cast guess). This is what lets 'you got what you paid for' stay true once playbooks fan out, which is the direction the engine is heading.

## Done looks like

A graph playbook whose fan-out's combined predicted cost exceeds the remaining shared envelope is stopped at the correct hard wall — concurrent branches draw from one wallet, the wave does not authorize casts it cannot collectively afford on wall-clock, and token overshoot from concurrently-settling casts is accounted against the single envelope (not lost or double-counted). A worked example demonstrates that the same fan-out which respects budget under the shared wallet would have overspent under the old per-node budgets, and the existing single-chain budget behavior is unchanged.

## Decomposition (hand-cleared 2026-06-20 — the autonomous mint un-decomposed; a P7 correctness fix, resumed by HAND)

Generalize the SEQUENTIAL spend (`spendDown` over one wallet, `fitNext`/`canAfford`/`debit`) to a
**concurrent WAVE drawing from ONE shared wallet**, mirroring the pure-core ⊥ impure-shell split.

**The load-bearing correctness — the two IA-8 denominations DIVERGE under concurrency** (this is the
whole epic): a wave's branches run *overlapping*, so
- **wall-clock = the wave's ELAPSED = MAX(branch times)** — concurrency *stretches* the wall-clock
  budget (running 2 branches in parallel costs ~the longer one, not their sum — a real benefit of the
  graph); whereas
- **tokens = SUM(branch tokens)** — every branch's burn is real, summed (detect-after, IA-8).

The existing `debit` (per-cast, sums BOTH) is correct *sequentially* but **over-charges wall-clock for a
concurrent wave**. That divergence is the fix's heart.

**S-048-01** (critical path 2): T-048-01 (**the pure wave-budget core** — `authorizeWave(wallet,
ready-set, priceOf)` greedily fits a ready-set into the shared wallet: each node's predicted **time ≤
remaining** (wall-clock is MAX under concurrency, so each-fits ⇒ the wave fits) AND cumulative
**tokens ≤ remaining** (collective sum) — returning which to dispatch concurrently + which are
hard-stopped (the `fitNext` generalization to a SET); `debitWave(wallet, actuals)` folds a settled
wave — **tokens summed, wall-clock MAX** — flooring + surfacing overshoot (IA-8); PURE/TOTAL,
unit-tested incl. single-node-wave == the old sequential `debit` for back-compat) → T-048-02 (**thread
ONE shared wallet through `castGraph`'s wave dispatcher** — each topological ready-set is
`authorizeWave`'d before `Promise.all` (only the affordable subset dispatched; the rest a clean hard
stop at the wave boundary, the `shouldContinue` wallet-exhausted analog), `debitWave`'d after settle;
the single-chain/linear path **unchanged** (a linear graph = single-node waves; `spendDown` untouched);
+ a DETERMINISTIC worked example (stub nodes, no live cast) showing a fan-out the shared wallet **stops
at the correct wall** which the **old per-node budgets would have overspent** — E-048's "Done looks
like" comparison). Epic-scoped ids (R6 — `048` clean). FREE/deterministic (budget arithmetic + stub
example — the live re-cast that would complete E-047's join is a downstream pull). When done: P7 holds
under concurrency — you got what you paid for, no matter how the graph fans out.
