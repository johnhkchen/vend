## Intent — the bigger-picture play

E-058 ran the Set-B non-dev channel proof live and the A3 make-or-break risk **materialized** — recorded
honestly in the gold master: the shipped flow (clone seed → `lisa init` → `vend init --template hackathon`
→ `vend steer`) produces an **honest-empty steer, not a coherent board**. The cause is a wiring gap, not an
engine bug: `vend steer` reads its steering context from `docs/knowledge/charter.md` + a board-ids snapshot
(`src/play/steer.ts:109` `assembleSteerInputs`), and the seed's product intent lives in a **root `SEED.md`
that no code reads** (grep: `SEED.md` is only *written* as a stub by `init-core.ts`, never read). So steer
sees an empty board + a generic charter + **no idea**, and the honest-empty rule (`baml_src/steer.baml:71`)
correctly fires. The seed's own docs *promise* the opposite — `charter.md`: "`vend steer` … read[s] this
file as steering context"; `SEED.md`: "the first move is to cast `vend steer` over this seed."

This epic makes that promise true. It threads the seed's stated intent into steer's input through the
existing go-and-see project snapshot — the string the steer prompt already tells the model to ground demand
against — so a one-line `SEED.md` becomes a readable demand gradient. A coupled second gap: the seed's
*rich* hackathon charter sits at root, but `vend init` overlays only a generic `CHARTER_STUB` into
`docs/knowledge/charter.md` (the path steer reads), so the value function steer grades against is the wrong
one. The template overlay is enriched to write the tuned charter where steer reads it (the intended-but-
unbuilt T-058-03 enrichment — `init-core.ts:174` ships only the SEED stub today). Then the channel proof is
**re-driven live** to confirm steer now stages a real board the designer sees beside their app.

## Value to the design

This is the difference between the non-dev channel *working* and *not* — the riskiest must-be-true on the
PM desk (claim 2b). It advances **P2** (the two-gesture transaction actually produces an outcome: copy →
init → steer now yields a board, not an empty file) and **P5** (the designer gets visual understanding read
off *their own* one-line seed, local-first). It also turns E-058's negative gold master from a dead end into
a closed loop: the corrected drive re-captures `EXPECTED-OUTCOME.md` as a *positive* re-runnable bar and
accrues a genuine forward-E1 record (the Frontier-1 second payoff E-058 set up). The fix is the lightest
honest touch — an optional snapshot field + a tolerant read — so vend-on-itself (no `SEED.md`) is byte-
identical and honest-empty is preserved.

## Done looks like

- `vend steer` on a fresh hackathon seed reads the root `SEED.md` intent and stages a **non-empty, grounded
  board** (≥1 signal traceable to the seed's one line) under `docs/active/pm/staged/steer.md` — no longer an
  honest-empty abstention. On a project with no `SEED.md` (vend itself), the staged steer is **byte-
  identical to today** (honest-empty preserved).
- The hackathon template overlay writes the **tuned charter** to `docs/knowledge/charter.md`, so steer grades
  the seed against the hackathon value function (demonstrable slice over polish), not the generic stub.
- The corrected board renders on the **SVG surface** (E-055/E-056) beside the running app, and the live re-
  drive re-captures `examples/templates/hackathon-seed/EXPECTED-OUTCOME.md` as a *positive* gold master with
  a forward-E1 record.
- `bun run check:*` green; the pure snapshot formatter is test-pinned for both the intent-present and
  intent-absent cases.
