## Intent — the bigger-picture play

The early CLI-ification of Vend's counter. Calling bare `vend` shows a ranked
menu of the **salient, high-leverage actions available right now**, computed from
the project's current state. Selecting by index runs them:

```
vend                 # show the ranked menu of salient actions (the shelf)
vend 1               # run the #1 (highest-leverage) action
vend 1,2,4-6         # run actions 1, 2, 4, 5, 6
vend 1 --budget 2h,50k   # override the default envelope for this pick
```

`vend 1` *is* the vending-machine press — it collapses pick + budget + run into a
single keystroke, the budget defaulting to the action's warranted envelope. This
evolves E-001's static `vend run <play> <target>` into `vend` + an index. The
selection is a tiny mini-language: comma-separated, `a-b` inclusive range,
1-indexed, deduped, whitespace-tolerant (`1,2,4-6 → [1,2,4,5,6]`); an invalid or
out-of-range index is a hard error, never a guess.

## Value to the design

The first real **counter** (P2 — the run is two gestures, here collapsed toward
one) and the core feature made usable: the clearing house surfaced as a shelf you
walk up to. It is **context-aware by construction** — the menu is the value
ranking from `demand.md`, filtered by readiness from `lisa status`, grounded in
git/charter. "Salient" = high-leverage **and** ready; blocked/leaf items are
hidden (behind `--all`). It turns the value-ranking + 2-hour-budget model we
authored into something you can actually *press*.

## Done looks like

- `vend` renders a numbered menu of ready, high-leverage actions with value tier +
  warranted budget + state, computed deterministically from `demand.md` +
  `lisa status` (no LLM) — instant.
- The menu is persisted to `.vend/menu.json` so `vend <sel>` resolves indices
  against **the same list just shown**; a materially-stale menu warns "re-run
  vend" rather than acting on stale indices.
- `vend <sel>` parses the selection mini-language and runs each picked action's
  playbook (today: `DecomposeEpic` on that epic/signal) with the warranted budget
  envelope as default, overridable via `--budget`.
- Multi-select runs **in order**, each its own budgeted run, each appended to the
  countable run log (`T-001-04`).
- `--all` reveals the hidden blocked/leaf rows.
