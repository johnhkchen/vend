## Intent — the bigger-picture play

The synthetic end-to-end trial of the E-079 seam (2026-07-13) proved the machinery works when everything is set — and exposed exactly how it lies when something isn't. Because the hook swallows recorder errors by design (it must never block lisa), a loop without LISA_DURATION_SECS — a state lisa's own hooks-guide calls normal ('when tracked') — silently never fires the cord: the precise failure the seam exists to eliminate. And the first live settle listed all 79 historical epics as 'sweep ready' and claimed 'none since last settle' on a run with no baseline. This epic makes the seam honest at its edges: the recorder records with duration absent rather than refusing; any refusal it does make leaves a trace (one line — timestamp, reason — in a gitignored local seam log) that vend settle surfaces as 'cord: last recording failed — <reason>' when the log is newer than the last claim; settle's verdict shows only open epics awaiting closeout and states a first run's baseline honestly ('first settle — no baseline'); and vend sweep's closeout commit carries .lisa/provenance.jsonl alongside the card flips when it has uncommitted changes, as the manual ritual always did.

## Value to the design

An andon cord that can fail silently is not an andon cord, and a verdict surface that misstates history erodes the exact trust it exists to build. This hardening realizes P3 by making settle's one-screen verdict factually correct (open epics only, honest baseline) — the contract's state made visible must be the true state; P4 because the walk-away loop E-079 closed is only actually closed if its failure modes are visible — a silently-dead cord quietly reinstates the human relay it retired; and P5 because the failure trace is a local gitignored log read by a local free gesture — the seam owns its own failure state on one machine. It also keeps sweep's closeout complete: provenance travels with the flips, so the machine gesture drops nothing the manual ritual carried.

## Done looks like

Re-running the E-079 end-to-end trial against the hardened seam shows the four truths at once: a loop that finishes without LISA_DURATION_SECS still fires the cord and yields a claimable loop-settled marker; a forced recorder refusal is no longer invisible — the next vend settle prints 'cord: last recording failed — <reason>'; on this board of ~79 done epics, settle's verdict lists only the open epics awaiting closeout (not history) and a first-ever settle says 'first settle — no baseline'; and a vend sweep closeout commit lands with .lisa/provenance.jsonl included when it had uncommitted changes.

## Context & constraints

Grounded in the live trial receipts from the 2026-07-13 synthetic end-to-end drive of E-079 (the-loop-closes-itself, now done): recorder refusals 'LISA_PROJECT must be an absolute project root' / 'LISA_TICKETS_DONE must be a non-negative safe integer' / 'LISA_DURATION_SECS must be a non-negative safe integer' were each invisible until run bare, while a successful marker {v:1, kind:lisa-loop-settled, ticketsDone:2, durationSecs:300} was claimed once and consumed correctly. E-079's machinery (recorder hook, settle, sweep) is the shipped substrate this hardens — a settled fact, not a prerequisite. The hook's never-block-lisa containment is right by design and is explicitly NOT changed; the fix is tolerance plus a visible trace, not a louder hook. Non-goals respected: settle remains a pull/event-triggered free verdict, not a babysitting dashboard (N2); sweep's deliberate confirmation keystroke stays. Out of scope: ntfy content changes, and auto-sweep without confirmation. Sizing (PE-7): four slices, each small (tolerate one absent field; append-and-surface a log line; filter done epics + one baseline message; include one file in an existing commit), all on one seam — comfortably one ~2-hour feature block, priced at {1}{W}{U} accordingly.
