## Intent — the bigger-picture play

Make the budget envelope **measured, not guessed**. Today envelopes are hand-picked
numbers (the `--budget` a human types, or a play's default); the run log
(`.vend/runs.jsonl`) already records every cast's actual cost, but nothing reads it
back. This epic closes that loop's **first rung**: a pure core that loads the log,
bounds each play's cost at the value-tier percentile (IA-12), and surfaces it as the
Confirm default with honest confidence (IA-6/IA-8 — "measured · N casts · p90" vs
"estimate"). Without it, IA-6's *accept-the-default* is a guess, and the user pays a
specification cost every cast.

## Value to the design

Makes the budget contract honest (**P7**): the envelope reflects what the play *costs*,
not a hope. Makes the second gesture trustworthy (**P2**): accept-the-default works
because the default is earned. It is the **prerequisite for the "work for 2 hours"
macro mechanic** — a feature-block budget can only be spent *intelligently* across
casts if vend knows each cast's measured price. This is the kaizen feedback the demand
board and `mana-economics.md` keep gesturing at, made real.

## Done looks like

- The run record logs the **allocated envelope** alongside actuals (backward-compatible),
  so cost-vs-budget is recoverable for every cast.
- A **pure** `recalibrate(play, log, tier)` bounds a play's cost (tokens + wall-clock)
  at the value-tier percentile, computed from **successful** runs only (IA-13), and
  returns the envelope **plus confidence** (N runs, censored count).
- A surface (CLI / the Confirm default) shows the **measured** envelope with its
  confidence, falling back to the hand prior when there's no data (cold start).
- `bun run check:*` green; existing casts unaffected (the field is additive).

## Context & constraints

- **Cites:** `information-architecture.md` IA-12 (andon-budget→percentile), IA-13
  (bound-from-successes, informative censoring), IA-6/IA-8 (Confirm default, honest
  confidence); `src/log/run-log.ts` (the record + writer — extend); `.vend/runs.jsonl`
  (the existing countable shape: `play`, `outcome`, `usage`, `costUsd`, `startedAt`/
  `endedAt`); `demand.md` ("calibrated from data — a dozen runs"); `card-model.md`
  (the printed mana cost is the estimate; the Ledger learns the true cost).
- **Estimator:** an **exact percentile over a windowed JSONL read** — *not* t-digest.
  t-digest is a scale optimization; ship the smaller real thing first (the log is small).
- **Censoring (honest):** recalibrate from successes; andon'd runs are right-censored
  lower bounds, not data points to average. The first slice **reads** the andon rate;
  it does not yet auto-tune, probe, or act on it.
- **Non-goals (later rungs, explicitly out):** per-node attribution (the fat tail is
  local — next pull); **asymmetric-hysteresis auto-tuning** (IA-14); uncensored probe
  casts; the Ledger view + **demand-generation** (IA-15); the **macro 2-hour wallet**
  itself (the rung above this). Not an executor (**N4**).
- **First slice:** telemetry (log the envelope) + the pure recalibration core + the
  measured Confirm/CLI default. **Out:** everything in non-goals above.
