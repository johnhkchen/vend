## Intent — the bigger-picture play

E-055 shipped a real `vend svg` artifact, but go-and-see on the live board (12:13p observation) renders it 17128px wide across ~62 epic columns with every dependency edge an identical plain line — exactly the failure the persona research flags (Maya wants ONE glanceable picture; Devraj reads EDGES before boxes). Two grounded, coupled fixes close that gap. (1) GLANCEABILITY: flip only the NON-DEV default — change `DESIGNER_PRESET`'s `groupBy` from `epic` to a coarse axis already in GROUPINGS (`status` ≈5 or `leverage` ≈4 columns), so the default `vend svg` renders a board taken in at a glance. Every grouping stays available; `DEV_PRESET` may keep a finer axis. (2) EDGES-AS-PAYLOAD: enrich `ProjectionLink` (project.ts:59) with status-derived decision weight — computed in `projectGraph` from the frozen graph, mark a link BLOCKED when its upstream `from` isn't done (and optionally on-critical-path); `projectionToSvg` then gives blocked/critical edges visual WEIGHT (thicker + distinct stroke) while satisfied `depends_on` edges stay light, so 'what should I unblock first?' pops off the board. This is the next warranted play NOW because the prescribed honest next step — hand a real .svg to the designer and watch where she gets lost — cannot run against a 62-wide undifferentiated strip.

## Value to the design

Realizes glanceability and dependency-decision-weight on the existing pure projection→SVG surface, directly serving the core feature (simplifying allocation: a board that surfaces what to unblock first IS the allocation/sequencing surface). It advances P5 by keeping the whole change inside the local-first deterministic discipline — pure `projectGraph` + `projectionToSvg`, no clock/random, byte-identical renders, MCP-independent, no live model — so the new edge metadata and coarse default are a calibration of presentation over the same frozen graph, never a new data authority.

## Done looks like

Running the default `vend svg` (non-dev seat) renders a coarse board of a handful of columns, not ~62; a dependency edge whose upstream `from` is not done renders with the heavy/blocked stroke while a satisfied edge renders light; the same input still produces a byte-identical SVG; and the house gates stay green (authority-guard: the frozen graph is read, never written back; MCP-independent; no live model). Observable by opening the produced `.vend/work-graph.svg` and by deterministic tests over the projection.
