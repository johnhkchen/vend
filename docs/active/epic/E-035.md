## Intent — the bigger-picture play

The vision and `stack.md` both promise it: Claude Code is "the **first** executor … behind an executor
interface so open models slot in later." But the interface doesn't exist yet — `castPlay` calls
`dispense` (the `claude -p` subprocess) directly, so the engine is **welded to Claude**. P6 is the
claim that an open model can take Claude's place behind a seam; this epic makes the seam real and
**proves** it isn't secretly Claude-shaped by standing a **second executor** behind it.

Two parts:

- **Extract the `Executor` interface.** Formalize the one boundary `castPlay` actually depends on:
  `dispense({ prompt, model?, maxTurns?, <tool flags>, onMessage, timeoutMs }) → ResultMessage`, streams
  `StreamMessage`s to `onMessage`, throws a timeout error. Refactor today's `dispense`/`buildArgs` into
  a `ClaudeExecutor` behind that interface, add an **executor selector** (default = Claude — byte-
  identical back-compat), and have `castPlay` resolve+call the selected executor. The downstream
  pipeline (SAP-parse → gate → effect → run-log) is untouched — that is the *point*: a non-Claude
  executor must feed the **same** gated, metered, logged machinery.
- **Prove it with a second, non-Claude executor.** An **OpenAI-compatible / local-model adapter** — the
  universal shape open models expose (Ollama, llama.cpp, vLLM all speak `/v1/chat/completions` with
  streaming). It builds the request, consumes the SSE stream, adapts chunks → `StreamMessage`s, and
  synthesizes the terminal `ResultMessage` with token usage (the two-denomination meter, IA-8). Proven
  **free + deterministic against a recorded SSE fixture** — no live model needed in the gate.

**The honest boundary — what this does NOT claim (load-bearing).** A single chat-completion adapter is
**not an agentic runner**: Claude's `dispense` is the *full agent* (reads the repo, runs tools, iterates
turns); a `/v1/chat/completions` call returns one completion. So this proves the **interface + transport
+ metering** are genuinely pluggable, and works for **context-complete** plays (the prompt carries its
context, e.g. decompose from an inlined epic) — but it does **not** prove an open model can do the
**read-the-repo, run-tools** plays (survey/steer/work). An **agentic open-model runner** (a local agent
loop, or an open-source agent framework behind the same interface) is the **named downstream epic**, not
this slice. We extract the seam and prove a second transport; we do not over-claim open-model parity.

## Value to the design

Turns P6 from an aspiration in the vision doc into a **real seam with two implementations** — the
strongest possible evidence that Vend is an orchestrator, not a Claude wrapper. The downstream pipeline
proven executor-agnostic; the metering proven transport-neutral; the door to open models opened (the
OpenAI-compatible adapter is the universal connector) with the remaining agentic gap named, not hidden.

## Done looks like

- **An `Executor` interface** capturing the `dispense`-shaped contract (prompt + budget + `onMessage` →
  metered `ResultMessage`; the timeout-error contract generalized from `ClaudeTimeoutError`).
- **`ClaudeExecutor` behind it** — today's `dispense`/`buildArgs` refactored, no behavior change; the
  agentic options (`maxTurns`, tool flags) are Claude's to honor (a non-agentic executor ignores them,
  documented).
- **An executor selector** — default Claude (back-compat: existing casts byte-identical); a non-Claude
  executor selectable (env, e.g. `VEND_EXECUTOR=openai-compat` + `VEND_OPENAI_BASE_URL`/model). `castPlay`
  routes through it.
- **A second `OpenAICompatExecutor`** — request build + SSE stream consume + `StreamMessage` adapt +
  `ResultMessage` with usage; **unit-tested against a recorded SSE fixture** (free, deterministic).
- **Proven free + deterministic:** a **stub executor** injected into `castPlay` casts a play end-to-end
  through the unchanged parse→gate→effect→log pipeline (the seam is real); the OpenAI-compat executor
  maps a fixture SSE stream → the same `StreamMessage`/`ResultMessage` shapes; the selector defaults to
  Claude (existing tests green). `bun run check:*` green. *(Live open-model smoke against a real
  endpoint is the one deferred human step — the E-017/E-018/E-024 live-cast deferral pattern.)*

## Context & constraints

- **Cites:** `src/engine/cast.ts` (`castPlay` ~188 — the single `dispense` call to route through the
  interface; `resolveMaxTurns`/`toolFlags` already assembled there); `src/executor/claude.ts`
  (`dispense` ~296 / `buildArgs` ~115 / `DispenseOptions` / `ResultMessage` ~34 / `ClaudeTimeoutError`
  / the `makeStreamConsumer`/`StreamMessage` stream types — refactor behind the interface, generalize
  the timeout error); `src/executor/claude.test.ts` (the seam's test precedent); `docs/knowledge/stack.md`
  + `vision.md` (the "first executor behind an interface" promise); IA-8 (the two-denomination meter the
  second executor must feed).
- **Back-compat is the default:** the selector defaults to Claude; the Claude path stays byte-identical
  (its tests green). A second executor is opt-in (env).
- **The interface's options are a superset; executors honor what they can.** `maxTurns`/`mcpConfig`/
  `allowedTools`/`strictMcp` are **agentic hints** — Claude honors them, a completion adapter ignores
  them (documented, not silently dropped). The universal core is: prompt + `timeoutMs` + `onMessage` +
  `model` → a metered `ResultMessage`.
- **Metering must stay honest (IA-8):** wall-clock is transport-universal (the timeout latch); tokens
  come from each executor's own usage (Claude's `usage`, OpenAI's `usage`) mapped into the same `Usage`
  shape so the run-log/recalibration read truth, not a guess.
- **Non-goals:** an **agentic open-model runner** (read-repo/run-tools plays — the named downstream, the
  real open-model parity work); per-play executor selection (a play declaring its executor — natural
  after E-032 but scope creep here; global/env selection first); a live open-model endpoint in the gate
  (the proof is fixture-based; the live smoke is deferred); changing the parse→gate→effect→log pipeline
  (the abstraction's whole point is to leave it untouched); a new metering model (reuse IA-8). Not a
  hosted-API second executor with secrets (local-first; env-config, no committed secret).
- **First slice = the `Executor` interface + `ClaudeExecutor` + the selector (default Claude) + the
  OpenAI-compatible adapter, fixture-proven.** **Out:** the agentic runner, per-play executor, the live
  smoke.
