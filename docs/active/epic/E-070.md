## Intent — the bigger-picture play

E-069 landed the `--agent <seat>` flag with KNOWN_SEATS validation in the materialize effect (`src/play/agent-seat.ts`, `decompose-effect.ts`'s `unknown-seat` outcome). The 2026-07-11 review nit is that this refusal fires AFTER the cast: `vend chain "<signal>" --agent kodex` runs the full generation, clears its gates, and then discards the entire board because one keystroke of the seat string was wrong — converting a typo into a re-run plus overseer babysit. This epic flips that disposition. On an unknown seat, materialize the full board on lisa's DEFAULT seat and proceed with an honest warning, instead of refusing the whole generation. The house already rules this way twice: reduced-grounding (T-060-01-01) proceeds degraded and records; the over-envelope run (S-068-02) materializes with a marker rather than discard. A mis-typed routing preference belongs in that same "degrade, don't discard" family — refusal is for structural invalidity, not for a preference the board can safely default.

## Value to the design

Realizes degrade-don't-discard for the routing seat, advancing three invariants at once. P4 (autonomy over supervision): a keystroke no longer forces a re-run and an overseer babysit — the board proceeds and records the degradation rather than escalating to a human. P6 (executor-agnostic): the board always lands runnable, because routing is a preference the materialize path can safely default, never a condition on whether the generation is valid. P2 (budget is honored): the spend the budget gesture authorized reaches the board — work that cleared its gates is not thrown away over a routing typo. It strengthens the core clearing function: a cleared unit settles onto the board, and requested-vs-actual honesty is preserved in provenance rather than paid for with a discarded cast.

## Done looks like

Running a board-writing gesture with an unknown seat (e.g. `--agent kodex`) materializes the FULL board — the unrouted tickets are byte-identical to today's default mint (no `agent:` key, so lisa runs its default seat) — emits an honest warning naming the requested-vs-default seats, and writes a named seat-defaulted degradation marker to the run record (the requested raw seat string, the default applied, and why), instead of discarding the generation. A valid `--agent codex` still stamps `agent: codex` as E-069 landed it. The `unknown-seat` hard refusal is gone from the effect path; refusal now fires only for structural invalidity (graph, story contract, bare codes), and KNOWN_SEATS remains the single source of the valid-seat list — now consulted to decide default-vs-route, not to reject.

## Context & constraints

Ground against E-069's shipped code: `src/play/agent-seat.ts` (KNOWN_SEATS, the seat guard) and the materialize effect's `unknown-seat` outcome in `decompose-effect.ts`, plus the two precedents this mirrors — the reduced-grounding degraded-and-record path (T-060-01-01) and the over-envelope warning marker on the run record (S-068-02 / run-log.ts's one-way marker pattern). The three moves this epic covers: (1) on an unknown seat, OMIT the `agent:` key entirely and materialize the full board; (2) carry a named seat-defaulted degradation marker on the run record — the seat-defaulted analog of the over-envelope marker — so provenance keeps the requested-vs-actual honesty its schema already models; (3) retire the `unknown-seat` hard refusal from the effect path. Prerequisites (named, not folded in as hidden scope): E-069's `--agent` flag + KNOWN_SEATS validation must already be landed (it is, 2026-07-11); and KNOWN_SEATS stays the single source of the valid-seat list — reused to choose default-vs-route, not re-invented. Out of scope: changing how lisa DISPATCHES to a seat; adding seats beyond claude|codex; per-ticket seat overrides; and altering refusal for genuine structural invalidity (graph/story-contract/bare-code refusals stay). Respects N4 (Vend writes/defaults the routing; lisa still executes) and N2 (this removes a re-run-and-babysit loop rather than adding a dashboard).
