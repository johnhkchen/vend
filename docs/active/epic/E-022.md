## Intent — the bigger-picture play

E-019 found, and E-020's proof sharpened, that the articulation plays (expand/survey/steer)
**censor invalid output but don't converge valid output** — run the same play twice on the
same input and the result differs ~0.50–0.72 (line-set Jaccard). Before *fixing* that, decide
**what consistency Vend even promises.** Two axes: **(1) validity consistency** — every cast
yields a valid, gated, grounded output (the gates' job, the real promise; E-014 ≈ 21%); **(2)
lexical/content consistency** — repeated casts yield the *same wording* (where the 0.50–0.72
lives). The fork: is axis (2) part of the promise, or is divergence-among-valid-outputs
*by-design* (diversity)? **You cannot decide it without one missing measurement:** is the
dispersion **equivalent-diversity** (the N valid outputs mean the same thing, worded
differently — fine) or **genuine-disagreement** (the play proposes a *different* thing each
run — troubling, trust-eroding)? This epic measures that, then captures the contract.

## Value to the design

Defines the **consistency contract** (P3 — gates are the contract; this says *what the
contract is about*) for the core value prop the vision names ("repeatability over a natively
probabilistic process"). Measure-then-decide (the E-014/E-019 discipline): don't build a
convergence lever before knowing whether divergence is even a defect. The output is a
*decision + a captured contract*, not a speculative mechanism.

## Done looks like

- **The diagnosis:** the consistency probe is extended with a **semantic-equivalence check**
  (a judge cast over a play's N divergent outputs on a fixed input) classifying the dispersion
  per play as **equivalent-diversity** or **genuine-disagreement** — the data the fork needs.
- **The decision + contract:** a captured **consistency contract** (an IA note) stating what
  Vend promises — *validity/grounding consistency via the gates, not lexical identity* — and,
  per play, **converge** (where the diagnosis finds genuine disagreement) or **by-design**
  (where it's equivalent diversity), with the human fork surfaced for assent.
- A clear **next pull** if convergence is warranted anywhere (the lever is a *downstream*
  epic, not this one).

## Context & constraints

- **Cites:** `src/probe/run-consistency-probe.ts` + `consistency.ts` (E-019 — extend with the
  equivalence judge); `src/probe/variance.ts` (the dispersion metric); E-014 (validity-axis

  `vision.md` ("consistency over a probabilistic process"); `information-architecture.md`
  (where the contract note lands); the recalibrated play budgets (expand 250k / survey 300k /
  steer 400k) for the probe casts.
- **Cost discipline:** the judge adds casts — keep N small (≈3) and the sweep bounded; log
  what was sampled (no silent caps); a directional read, not a proof (the E-014 honest-sample
  discipline).
- **Non-goals:** *building* a convergence mechanism (lower temperature, normalization, a
  canonical-form gate, dedup, caching) — that's a **downstream epic**, pulled only if the
  diagnosis + the human fork choose "converge"; re-litigating the validity axis (the gates
  already own it). Not an executor (**N4**).
- **First slice = the decision** (~1 block): the equivalence diagnosis + the captured contract
  + the surfaced fork. **Out:** any convergence lever.
