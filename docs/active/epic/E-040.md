## Intent — the bigger-picture play

Today vend can only be driven in the repo that birthed it: a fresh project's clearing-house structures must be built by hand. Author `vend init` — a single idempotent command that layers vend's structures onto a bare lisa project (the way `lisa init` sets up the lisa side): the demand board, the `pm/` desk, `epic/ stories/ tickets/ work/`, knowledge stubs, the cleared-demand archive, and `.vend/` state. It detects an existing lisa project (`CLAUDE.md`/`.lisa.toml`) and seeds structure + knowledge, NEVER demand — a fresh board starts honestly empty (IA-3/IA-4); the first real move is a Survey/Steer cast off a seed. This is the PRD-recommended first pull of Frontier 7 and the foundation every later distribution piece (doctor, templates/examples, Homebrew delivery) builds on. It matters now because the spine is done; the binding constraint is adoption, and a clearing house nobody can stand up clears nobody's work — and every new driveable project feeds the keystone with the cleared forward-E1 runs that earn trust.

## Value to the design

Realizes scaffold-a-fresh-project capability: vend becomes usable beyond its author, advancing P5 (local-first — the command owns and writes the project's own `.vend/` state on one machine, no service) and P2 (the two-gesture transaction survives onboarding — setup is one command, config lives at authoring, not at the counter). It directly extends the core feature — clearing work for ANY project rather than only this repo — turning vend from an author-only tool into a product a second person can drive.

## Done looks like

Running `vend init` in a bare lisa project produces a complete, valid vend+lisa structure (demand board, `pm/` desk, `epic/ stories/ tickets/ work/`, knowledge stubs, cleared-demand archive, `.vend/` state) that `lisa validate` accepts, with the board honestly empty (no fabricated demand) and ready for a first Survey/Steer cast. Re-running the command is safe and idempotent — it converges, adds no duplicates, and clobbers no existing content — across the states a `lisa init` project can be left in. Observably: a second person can go from a bare lisa project to a driveable vend project in one command.
