## Intent — the bigger-picture play

Stand up Vend's single metered lever: one hardcoded play — `DecomposeEpic` — that
takes an epic + the charter + current project context and **dispenses** real work
through `claude -p`, gated and budgeted, streamed live and logged in a countable
shape. This is the keystone. Until something real is dispensed, the whole charter
is only documents. Everything later — the multi-node DAG, open-model executors,
conversational authoring, the read-side consistency layer — generalizes out of
what this one hardcoded play forces us to make real. So build only this.

## Value to the design

This is the lever the entire charter hangs on, at its smallest honest size: intent
in, allocatable work out, against this project, gated so nothing unworthy settles.
It proves the three invariants that make everything after it trustworthy — the
play is authored once and run by gesture (**P1**); its gates stop the line rather
than self-certify (**P3**); the run respects the budget it was given, both ways
(**P7**). It also lays the Bun scaffold and the `bun run check:*` command surface
that the CI backstop (the next pull, E-002) and every later epic build on. Pulling
the CI signal first surfaced its dependency on exactly this work — so this is the
correctly-sequenced first purchase.

## Done looks like

- Running `DecomposeEpic` on a real epic dispenses a gated, budgeted `WorkPlan`
  via `claude -p` — and **refuses** (stops the line, says why) when a clearing
  gate fails, rather than emitting half-cleared work.
- Every run leaves a **countable log** (the terminal result's usage / cost /
  outcome), kept by default, so the consistency layer is later just reading data
  already there — not a new system.
- Budget exhaustion is a clean hard stop with a clear andon, not a silent overrun.
- The scaffold exists as a Bun project with at least `bun run check:test` passing
  — the command surface the play invokes as an andon gate and CI invokes
  independently.

## Context & constraints

- The codebase at the moment is **docs only** — this epic creates the scaffold.
- Stack is settled (`stack.md`): TypeScript on Bun; **BAML authoring-only** (render
  via `b.request.*`, SAP-parse via `b.parse.*` — it is not the transport);
  `claude -p --output-format stream-json` as the single dispense seam, per the
  proven pattern in `mc-design-eval/src/sdk-binding.mjs` (spawn → stream each
  message to a hook → read the terminal `result` for `usage`/`total_cost_usd`;
  wall-clock guard for a non-returning child).
- **BAML-on-Bun: VERIFIED (2026-06-18).** Smoke-tested on Bun 1.3.9 / BAML
  0.222.0 — `baml-cli generate`, `b.request` (render) and `b.parse` (SAP) all run
  clean under Bun, native bindings load. The one thing mc-design-eval (Node + tsx)
  didn't prove is now proven; the stack holds. (Re-check only if the BAML major
  version moves.)
- The play's shape and its four value-ordered clearing gates are in
  `playbook-decompose-epic.md`; honor them.
- **Out of scope** (generalizes later, do not build): the multi-node DAG, the
  open-model executor, conversational authoring, the read-side consistency layer.
  The single lever must dispense something real first.
