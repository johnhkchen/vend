## Intent — the bigger-picture play

The structural-defect backstop. CI is the **inspection** layer — the *weakest check
type* by our own lens (`ci-strategy.md`) — so it is deliberately narrow: be the
trustworthy, *independent* detector for **compile / type / test / lint** only, so the
play never has to self-certify those. Built on **Dagger** so the *same* check runs in
two places without drift — the play's andon gate mid-run, and CI's independent
inspection after a commit.

## Value to the design

A gate that **de-risks every parallel-fleet build** (**P3**): it catches the structural
spindle the play couldn't, *before* a defect travels into the next agent's working
context. An enabler, not the core feature — which is exactly why it sits on the
opposite side of the next-pull fork from E-003.

## Done looks like (the first slice only)

- **One gate, end to end:** `bun run check:test` runs the suite standalone;
  `/ci/src/test.ts` (a `Test` sub-class) spins a Bun container and invokes it;
  `/ci/src/index.ts` routes `test()` and **nothing else yet**; engine pinned `v0.21.4`,
  module on the **Node** runtime.
- The identical `bun run check:test` agrees three ways — standalone, as the play's andon
  gate, and as CI's independent run.

## Context & constraints

- **Full steering:** `ci-strategy.md` (the Central Rule — *Dagger invokes, never
  defines*; the `/ci` structure; the three-class defect model; the measured ~18.4s
  cold-start → keep-warm mandatory).
- **The repeatable how-to:** `playbooks/ci-structural-gate.md` (the 6-step add-a-gate
  process; this epic's first slice is its `X = test` instance).
- **Tooling verified:** dagger `v0.21.4` operational end-to-end (Docker daemon must be
  up); pin it in `/ci/dagger.json`.
- **Out of scope** (generalizes out of the one honest gate, do not bundle): `lint`,
  `typecheck`, `consistency`, parallel DAG composition, keep-warm tuning. Get one gate
  honest first.
