_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

Add a gate that verifies the committed HEAD itself builds — by checking out HEAD in isolation from the working tree and running the existing `check` (baml:gen → typecheck → test) against that clean tree, not the dirty workspace. E-008's `check:committed` proves source is committed; it cannot prove the committed source is *sound*. E-007 is the live proof: `cast.ts` was committed without `play.ts`, yet local `check` stayed green because the working tree still held the uncommitted `play.ts` — HEAD was broken and nothing noticed. We want partial/inconsistent-but-committed commits caught at session-stop, the way a fresh CI clone would catch them, so 'I committed' can be trusted to mean 'what I committed works'.

## Value to the design

Closes the last gap between local-green and HEAD-green, which is the gap that lets autonomous sessions end on a broken-but-committed tree (P3 — the gate makes 'done' enforceable at the commit level; P4 — it catches the defect without a human reviewing the diff, so autonomy stays safe). It hardens the same clearing-house contract E-008 began: the committed artifact is the deliverable, and now the committed artifact is checked as the executor would see it.

## Done looks like

Ending a session whose HEAD doesn't build from a clean checkout (e.g. a source file committed without its dependency, à la E-007) fails the gate and blocks/flags the stop; a fully-committed, building HEAD passes. The build runs against the committed tree in isolation, so a stray uncommitted file in the working directory can no longer mask a broken commit. Re-running E-007's exact partial-commit scenario through the gate reproduces a failure.

## Context & constraints

Grounds on the live repo: `package.json` already defines `check` = `baml:gen && check:typecheck && check:test`, and `check:committed` (src/ci/check-committed.ts, E-008/T-008-01) is wired into the lisa on-stop hook (T-008-02). This epic reuses both — it is the build-soundness companion to the commit-completeness gate, and it hangs off the same on-stop wiring point. PREREQUISITE (name, don't absorb): E-008's `check:committed` gate and its on-stop hook must exist — they do. DESIGN BOUNDARY to settle at decompose, not pre-decided here: the signal says 'container,' but P5 (local-first) argues against a hard Docker dependency for a single-machine offline tool; a clean `git worktree`/temp-clone of HEAD running the existing `check` likely satisfies the requirement without new infra, with a literal container left optional. Respects N4 (this is a gate, not an executor — it builds to verify, it does not run playbooks) and N2 (a stop-time pass/fail check, not a babysitting dashboard). OUT OF SCOPE: it detects, it does not auto-repair the broken HEAD; it does not replace `check:committed` (different defect class); no decomposition into stories/tickets here — that is DecomposeEpic's job. Right-sized as one ~2-hour block (isolate-checkout + run-check + wire-to-hook + tests).
