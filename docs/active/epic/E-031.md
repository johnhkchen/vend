## Intent — the bigger-picture play

IA-1 says Home is **recommendation-first**: the top of the screen is *the one thing worth doing now,
and why*. IA-2 splits that Home into **Board** (demand — ranked signals; bare `vend`) and **Shelf**
(supply — the authored playbooks; `vend shelf`, E-030). Both halves now exist as **separate
surfaces**, but the **single fused screen DL-6 specifies does not** — and DL-6 itself names this as
its one honest gap: *"there is no single `renderHome` composite emitter today — Home is the CLI
composing `renderMenu` + the shelf + the audit summary line; a TUI closes the gap against this mock."*

This epic builds that composite. Per DL-6: the **board leads** at column 0 (ranked by leverage, the
recommendation), the **shelf recedes beneath** as the inventory that serves it, and a **one-line
ledger summary** (E-028's provenance-split — forward vs attested) sits at the foot giving trust at a
glance. Whitespace divides the three regions — no boxes, no rules (DL-1/DL-9).

It is the same move E-030 made for the shelf, applied to the whole screen: compose the emitters we
already have rather than re-derive. The deliberate, load-bearing constraint: **the press contract is
untouched.** Bare `vend` already persists `.vend/menu.json` and `vend <sel>` resolves the selection
against that cache *by index, independent of printed text* — so Home can enrich what `vend` **prints**
(board + shelf + ledger) while the board stays the selection namespace and every existing `vend <n>`
press behaves exactly as today.

## Value to the design

Closes the **last named gap in the design language** (DL-6's `renderHome`) and makes IA-1's
recommendation-first Home a real composite emitter, not three disjoint verbs. Advances **P1**
(author-once — the supply shelf is shown in its proper place, serving the recommendation) and **P2**
(the two-gesture browse→press transaction — the browse surface becomes the full Home, the press is
preserved bit-for-bit). The fusion is where the IA spine (E-029) and the shelf (E-030) become one
screen a TUI can implement directly.

## Done looks like

- **A pure `renderHome` composite** that lays out the three DL-6 regions into one screen: the board
  menu leading at column 0, the supply shelf receding beneath (as inventory, **not** competing for the
  press selection namespace), a one-line ledger summary at the foot. Clean-typographic, whitespace-
  divided, no boxes/rules (DL-1/DL-9). Unit-tested on fixtures (incl. empty board, empty ledger).
- **A one-line ledger summary** (`homeLedgerLine`) derived from the walk-away report (E-028) — the
  compact `E1 walk-away N% (k/n) └ forward … · attested …` foot; the full readout stays `vend audit`
  (DL-8). Honest-empty when there are no runs yet (no fabricated trust number — the standing discipline).
- **Bare `vend` becomes the Home** — gather the board (rank + persist the **same** `.vend/menu.json`),
  the shelf rows (registry plays + run log, E-030), and the walk-away report (E-028), compose via
  `renderHome`, print. `--all` still reveals hidden board rows.
- **The press contract is preserved exactly** — `.vend/menu.json` shape unchanged, `vend <sel>` still
  resolves against the persisted board cache by index; no selection regression.
- **Proven free + deterministic:** `vend` prints board + shelf + ledger foot in one screen; `vend <n>`
  still presses the board pick; `.vend/menu.json` unchanged — no LLM cast. `bun run check:*` green.

- **Cites:** `src/shelf/menu.ts` (`renderMenu` ~151 — the board region to lead with; `formatBudget`);

  `src/ledger/walk-away.ts` (`formatWalkAwayFindings` ~238 / `auditWalkAway` / the forward·attested
  split — the foot line, E-028); `src/log/run-log.ts` (`readRuns` — the run log for shelf + ledger);
  `src/cli.ts` (the `browse` arm ~533 + `import.meta.main` — where Home composes); `src/shelf/press.ts`
  (`pressShelf` ~44 — the contract that must NOT regress); `design-language.md` DL-6 (the Home mock +
  the named `renderHome` gap), DL-1/DL-3/DL-9 (clean-typographic, hierarchy, no cards); IA-1
  (recommendation-first) / IA-2 (board=demand, shelf=supply).
- **Press contract is load-bearing:** the selection namespace stays the **board** (`.vend/menu.json`,
  index-resolved by `pressShelf`); the shelf-beneath is **receding inventory, not press-selectable**
  in this slice. `vend <sel>` must behave bit-for-bit as today; the cache shape is frozen.
- **Honest-empty ledger (standing discipline):** with no run log the foot line says so plainly — never
  a fabricated trust number; reuse E-028's split, don't invent the E1 bit.
- **Non-goals:** unifying the selection namespace so shelf playbooks are directly castable from Home
  (the DL-6 mock's ▶/· board + numbered shelf destination — that migrates the press from board→shelf
  and is its **own** epic); a TUI/`blessed` render (this is the CLI composite the TUI implements
  against — **N4**, not an executor); changing the board's ranking or the shelf's recalibration
  (E-003 / E-013 own those); the detached/notify mechanism or fleet/DAG andon board (separate open
  design threads).
- **First slice = the `renderHome` composite + `homeLedgerLine` + bare `vend` fused, press preserved.**
  **Out:** shelf-as-selection-namespace, the TUI.
