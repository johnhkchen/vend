## Intent — the bigger-picture play

When we wired `codebase-memory-mcp` into the E-031 tickets, we did it **by hand** — the play had no way
to say "I use the codebase graph." Today a cast (`claude -p`) inherits whatever MCPs the **machine's
global config** (`~/.claude.json`) happens to define: the same firehose for every play, silently absent
on a machine that didn't configure it, and never declared by the play that actually depends on it.

This epic makes a play **carry its own tool requirements**, the same way E-015 put the warranted
turn-cap on the play (`maxTurns?`). Three parts, cleanly split (the answer to the signal's "project
based?"):

- **Requirement → on the `Play` contract (author-once, P1).** A play declares *which* capabilities it
  needs — `tools?: { mcp?: string[]; allow?: string[]; skills?: string[] }`. Encoded judgment that
  travels with the play (e.g. decompose declares `codebase-memory-mcp` + the read tools it cites graph
  coordinates with).
- **Binding → project-based, NOT global.** The *concrete* server spec (launch command, args) lives in a
  project `.mcp.json` (Claude Code's native convention — `claude -p --mcp-config` reads it), committed
  with the repo. The play references a server *by id*; the project says *how* to launch it. This is the
  piece that does not exist yet (it's global in `~/.claude.json` today).
- **Resolution → at cast, least-privilege.** Vend resolves the play's declared ids against the project
  registry and threads **only those** into the seam (`--mcp-config` + `--allowedTools` +
  `--strict-mcp-config`), closing the global firehose for declared plays. A **required MCP the project
  doesn't provide → amber andon** (IA-9 — a run missing its declared capability is not the same run,
  IA-17), never a silent blind run.

Back-compat is the default (the signal's sub-question #1): a play that declares **no** `tools` is
**unchanged** — no strict flag, inherits the global set exactly as today. Only a play that opts in gets
scoped. So decompose-epic tightens; nothing else moves.

## Value to the design

Makes a play a **complete, portable spec** — its tool needs are part of the author-once judgment (P1),
not an out-of-band global config that silently varies per machine. And it cleans the **executor
interface** (P6): the play→executor contract now states the capabilities the executor must satisfy, so
an open-model executor slots in against the same declared needs. Bonus: **least-privilege** — a play's
tool surface becomes its blast radius (capture-note needn't get Bash; decompose gets the codebase graph
read tools, not write-everywhere).

## Done looks like

- **The `Play` contract carries `tools?`** — `{ mcp?: string[]; allow?: string[]; skills?: string[] }`,
  optional (back-compat: undeclared ⇒ unchanged). Mirrors the `maxTurns?` precedent (E-015).
- **A project MCP registry** — a committed `.mcp.json` (Claude Code convention) defining the project's
  servers incl. `codebase-memory-mcp`, *portable* (no machine-specific secret paths in the committed
  file — env-var indirection if needed). The binding is project-based, not the global `~/.claude.json`.
- **Cast-time resolution + least-privilege wiring** — the seam resolves a play's declared `tools`
  against the registry and emits `--mcp-config`/`--allowedTools`/`--strict-mcp-config` for **only** the
  declared servers; an undeclared play is unchanged.
- **A missing required MCP andons** — a play requiring a server the project doesn't provide halts as an
  amber missing-capability refusal (IA-9), nothing cast.
- **decompose-epic declares + uses `codebase-memory-mcp`** — the proof play references the MCP itself.
- **Proven free + deterministic:** the built `claude -p` argv for decompose-epic carries the
  scoping flags (mcp-config + allowedTools + strict); an undeclared play's argv is unchanged; an absent
  required MCP andons — all via `buildArgs`/`resolveTools` inspection, **no live cast**. `bun run
  check:*` green.

## Context & constraints

- **Cites:** `src/engine/play.ts` (the `Play` contract ~127 + the `maxTurns?` field ~157 — the exact
  optional-per-play-field precedent to mirror; `registry`); `src/executor/claude.ts` (`buildArgs` ~115
  — the PURE argv builder where `--max-turns` is appended, add the MCP/tool flags here; `dispense` ~259
  — the impure seam); `src/engine/cast.ts` (`castPlay` ~111 + `CastOptions`/`resolveMaxTurns` — where
  per-cast resolution + the andon path live); `src/play/decompose-epic.ts` (the proof play to give
  `tools`); the Claude Code CLI (`claude -p --mcp-config <file> --allowedTools <list>
  --strict-mcp-config` — **verify exact flag names against `claude -p --help`** before wiring; this is
  the one external contract to confirm, not assume).
- **No project `.mcp.json` today** — `codebase-memory-mcp` is global in `~/.claude.json`. Establishing
  the project registry IS part of this epic; keep the committed file portable (no secrets / absolute
  machine paths — use env-var/`${VAR}` indirection if the server spec needs them).
- **`--strict-mcp-config` is scoped to Vend's casts only** — it constrains the `claude -p` subprocess
  Vend spawns, never the human's interactive Claude Code session; and only for plays that **declare**
  tools. The user's own MCP access is untouched.
- **Honest scope cut — MCPs + tool-allowlist first; skills declared-but-deferred.** The `skills` field
  may exist on the type (forward-compatible) but full skill-injection wiring is downstream — Claude
  Code skills are a different mechanism (md+frontmatter), not an MCP server. The signal named MCPs
  first ("decompose references the MCP"); deliver that, name skills as the next slice.
- **Non-goals:** a Vend-specific MCP config format (reuse `.mcp.json` — don't reinvent); per-play
  *secrets* management (env indirection only); strict-by-default for undeclared plays (back-compat now,
  strict opt-in — the signal's sub-question #1); skill-injection wiring (deferred above); changing
  what's cast or the gates (this scopes the executor's tool surface, not the play's logic). Not an
  executor swap (**N4** — still Claude Code).
- **First slice = `tools?` on the contract + the project `.mcp.json` + cast-time resolution &
  least-privilege wiring + the missing-MCP andon + decompose as the proof play.** **Out:** skills
  injection, strict-by-default, secrets.
