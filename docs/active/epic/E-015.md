## Intent — the bigger-picture play

The token fat-tail is **agentic wandering, not input size** — E-014's E2 probe quantified
it: `decompose-epic` blew its 50k token envelope **~80% of the time across 20 fresh casts**
(tail ~85–95k), which *censored* the consistency measurement until the budget was raised to
180k. `claude -p` is the full agent; nothing today caps how many tool-use turns it takes.
This epic adds **`--max-turns`** at the dispense seam (`buildArgs` → `claude -p`), threaded
from the cast, so wandering is bounded **mid-flight** — turning the token denomination from a
detect-after andon (IA-8) into a near-real wall, and shrinking the fat tail at its source.

## Value to the design

Makes the **token half of the budget contract enforceable** (**P7**) — today only wall-clock
halts mid-flight; `--max-turns` is the closest the seam offers to a mid-flight token bound
(IA-8: "what would make ◇ a real wall"). Directly **de-risks the macro-wallet**: a depleting
macro budget is leaky while one runaway cast can overspend the slice the loop subtracted (the
exact dependency `demand.md` now records). Cheap, compounding cost predictability on *every*
cast.

## Done looks like

- `buildArgs` emits `--max-turns <n>` when a turn cap is set; **pure**, unit-tested
  (present / absent / value), threaded through `dispense` → `castPlay`'s `CastOptions`.
- A **warranted default** turn cap is wired (generous enough not to cut off legitimate runs,
  tight enough to rein in egregious wandering), with a **per-cast override**.
- The turns each cast used are **observable** (logged/surfaced) so the cap can be calibrated
  from data later (the E-016-style measurement loop), not frozen as a guess.
- **Live check:** a `decompose-epic` cast on a meaty epic now caps turns and its token spend
  lands **below the prior ~85–95k tail** — the fat tail visibly shrinks.

## Context & constraints

- **Cites:** `src/executor/claude.ts` (`buildArgs` — already threads `--model`/`--effort`/
  `--system-prompt`; add `--max-turns`; `dispense` spawns it); `src/engine/cast.ts`
  (`CastOptions` — thread `maxTurns`); `information-architecture.md` IA-8 (tokens are
  detect-after — this makes them mid-flight-bounded); `demand.md` (the sharpened signal +
  the macro-wallet dependency); E-014 findings (the quantified tail).
- **Default value is a judgment, not a guess to freeze:** pick a defensible cap *and* log
  turns-used so it's tunable from real data (decompose's clean runs vs its wandering ones).
  Erring generous avoids false andons on legitimate runs.
- **Non-goals:** the **system-prompt economy constraint** is a *separate lever* (the signal's
  angle (a)/(b)) — try `--max-turns` first; add a prompt bound only if turns alone don't tame
  the tail. The **envelope recalibration** (decompose's 50k is a cold-start guess too thin) is
  E-013's domain, not this epic. Not building our own executor (**N4** — this *configures* the
  `claude -p` we already use).
- **First slice = the whole thing** (~1–2h): the `--max-turns` mechanism + a warranted
  default + observability. **Out:** the system-prompt lever; per-play turn-budget tuning.
