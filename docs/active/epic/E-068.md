## Intent — the bigger-picture play

Today the budget wall enforces the wrong number. countTokens sums all four token buckets at parity, but cache reads — which dominate the count on any grown board (boilerplate-demo's failed E-008 decompose: cache_read=443,711 vs input=14+output=23,965, ~4.5% real work) — cost roughly a tenth of a fresh input token. So the counted 'spend' scales with turns × board-context-size while true dollar cost barely moves: a bigger board inflates the meter past ANY fixed ceiling, which is exactly the worker-reported 'raising the ceiling just spends proportionally more.' The meter was never converging on cost — it was measuring cached context. The play is to make the number the hard contract enforces measure actual cost, and to make the wall's two failure modes honest: a run that DID the job and only overshot must not be thrown away, and a decompose leg that dies terminally must not leave a green-doctor orphan epic behind. This matters now because it is the single units error standing between P7's hard contract and P4 autonomy — E-008 failed 3/3 walk-away attempts at 25k/80k/400k ceilings on a meter that couldn't be satisfied.

## Value to the design

Realizes a price-true budget meter: one shared cost-weighted accounting (input 1.0, cache_read ~0.1, cache_creation 1.25, output ~5) replacing the parity sum, so every ceiling, andon, and recalibrated default is denominated in cost rather than turns × cached-context. This directly repairs P7 (the hard contract now enforces a number that measures cost), unblocks P4 (false budget-exhausted andons that convert walk-away into babysitting stop firing), and stabilizes P2 (a budget gesture can be right because its units no longer drift with board size). The gate-honesty slices make the contract's refusals trustworthy: refusal is reserved for true runaways and gate failures, not for work that landed.

## Done looks like

Under one shared cost-weighted successor to countTokens, boilerplate-demo's recorded E-008 decompose (spent=525,180 old units) recomputes from the four buckets already in runs.jsonl to a figure that reflects ~real cost and fits within a sane ceiling, so an equivalent gates-passing decompose materializes instead of refusing; recalibrate re-derives its p90 budget defaults in the new units without re-running any history; a run whose gates PASS but whose spend overshoots its envelope materializes WITH an over-envelope warning on its record rather than being discarded; and a terminally-failed decompose leg leaves no orphan behind — doctor flags (or the chain rolls back) an epic with zero stories/tickets instead of staying green on the half-minted state.
