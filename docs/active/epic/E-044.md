## Intent — the bigger-picture play

Twice now — E-037 and E-039 — `vend steer` ranked a **self-referential** signal #1: *"Run and settle
the E-037 live macro sweep…"* Both times the operator had to **re-point the sweep at concrete demand
by hand** (E-039's sweep-log records the re-point). That manual step is exactly the friction P4
("autonomy by default — walk away") removes: an unattended wallet sweep, pointed at a self-referential
#1, either 0-clears (propose can't mint a clean epic for "go run the tool") or mints a degenerate
meta-epic.

**Why the existing gates don't catch it (grounding the fix's home):** steer ranks via the **BAML
model** (`steer.baml`: "RANK BY LEVERAGE … the board's ORDER is the ranking"); the pure gates
(`steer-core.ts`) are **structural poka-yokes** — `readNeverInventGate` (grounding present),
`forkGenuinenessGate` (fork shape), `leverageRankGate` (order monotonic) — and they explicitly
*"cannot judge semantic consequence."* The self-referential signal **passes all three**: it's grounded
(cites the run-log/board), correctly ranked keystone (proving the wallet genuinely IS high-leverage),
and well-ordered. The defect is purely **semantic** — it's an *operational meta-task*, not concrete
product demand — so the fix belongs where the semantic judgment is made: **the ranker prompt**, the
E-020 (honest-empty) precedent.

**The fix — prompt recalibration, NOT a keyword gate.** Recalibrate `steer.baml` (and `survey.baml`,
which ranks the same board shape) so a demand signal must be **concrete product demand** — a buildable
feature/change that decomposes into an epic — and **self-referential / operational meta-tasks** (run
Vend on itself, "run the sweep", settle a prior run, dogfood the loop) are **demoted beneath concrete
demand or excluded** as process notes, not demand. A pure keyword gate is deliberately **NOT** the
mechanism: distinguishing *"run `vend work`"* (meta) from *"improve `vend work`'s budget handling"*
(real product demand) is a semantic judgment a brittle pattern would get wrong (false-positives on
legitimate signals about Vend's own features). The model judges; the structural gates stay structural.

## The honest boundary — what this is and is NOT

- **Prompt-only semantic recalibration** (the E-020 model). The deterministic proof shows the contract
  changed (`baml:gen` green; the prompt carries the concrete-demand rule); it does **not** prove the
  model now ranks correctly — that's a **live confirmation**, deferred to the **next steer cast** (the
  next Frontier 1 sweep or Frontier 7 pull stages a steer board — observe the #1 there). The exact
  E-038→E-039 shape: ship the deterministic fix, confirm it live on the next cast that needs it.
- **No brittle keyword gate** (the named non-goal) — semantic, model-side, on purpose.
- **The structural gates are unchanged** — read-never-invent / fork-genuineness / leverage-rank stay
  as-is; this only adds the concrete-demand steering to the ranker prompt.
- **A self-referential signal is still legitimate for a HUMAN** — "prove the wallet" IS the keystone;
  the fix demotes it as an *autonomous-clear target* (it doesn't decompose), it does not deny it
  matters. (If the model judges it real human demand, it may still surface it — just not as the #1 the
  unattended wallet mints.)

## Value to the design

Removes the one manual step both live sweeps required — the difference between "walk away and it clears
concrete work" and "walk away and it 0-clears on a meta-task." Directly advances P4 (true unattended
autonomy) and the demand-extraction quality the articulation trilogy (E-016/17/18) is built on.
