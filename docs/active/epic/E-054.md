## Intent — the bigger-picture play

Close the last unbuilt graph primitive on the E-046 DAG substrate: exception semantics for a node's cast. Cycle detection is DONE and explicitly out of scope here — validateDag refuses cyclic specs, topoSort returns {cycle} without hanging, and both runners already refuse a cycle cleanly (cast nothing, every node skipped, gate-failed + halted). The genuinely-remaining gap, deferred at E-046 (graph-core.ts:15, 'Elaborate cross-branch error semantics are OUT'), is a node whose cast THROWS. Today the halt/skip machinery handles non-success OUTCOMES (a clean RunSummary with outcome gate-failed/budget-exhausted) but NOT an exception: a thrown cast thunk inside runGraphConcurrent's Promise.all REJECTS the entire wave, crashing the graph cast and discarding the partial GraphResult — including the completed work of independent siblings that had nothing to do with the failing node. The play is to make a throw behave like every other non-success: wrap each node's cast so an exception becomes a DETERMINISTIC node outcome (an 'errored'/'crashed' RunOutcome, or routed through the existing non-success path) that halts its dependent subgraph via the EXISTING halt-dependent-subgraph machinery. This matters NOW because it is the final property the typed-DAG substrate is missing, and it is exactly the hardening the open-model runner (Frontier 2, P6) depends on — an open-model cast that throws (network drop, malformed tool call, executor bug) must degrade to a marked node, not take down the whole graph.

## Value to the design

Realizes crash-safety of the graph executor: the dispatcher becomes total over node behavior — success, non-success outcome, AND exception all resolve to a clean GraphResult rather than an uncaught rejection. This advances the core feature (trustworthy autonomous clearing of graph-structured work) by making the IA-9 'a successful refusal is not a crash' stance hold for thrown casts too (P3 — gates are the contract: a failure is a marked, gated outcome, not a process death). It protects autonomy-by-default (P4): an unattended cast that throws self-contains into an errored node and lets independent branches finish, instead of requiring a human to restart a wave that died mid-flight. And it hardens the executor-agnostic substrate (P6): the open-model runner can throw in ways Claude Code's SDK does not, and the runner must absorb that the same deterministic way.

## Done looks like

A node whose cast throws is observably marked as an errored/failed node in the returned GraphResult (never surfaced as an uncaught exception), its transitively-dependent subgraph is skipped with a recorded reason, and any independent sibling node still completes — and this holds IDENTICALLY in both runGraph (the sequential runner) and runGraphConcurrent/castGraph (the concurrent wave dispatcher), proven deterministically with a stub throwing thunk and no live model. The graph cast always returns a clean GraphResult; the suite stays green.

## Context & constraints

Substrate: src/engine/graph-core.ts hosts BOTH executors over one pure core — runGraph (sequential topo-order for-loop, T-046-02) and runGraphConcurrent (concurrent Promise.all wave dispatcher, T-046-03) — plus the shared decideThread halt-dependent-subgraph gate that runChain also uses. Decomposes ~like E-049: a pure error-outcome/halt core (a throw maps to a deterministic errored RunOutcome routed through the existing non-success/skip path) threaded into BOTH runners (sequential for-loop → per-node try/catch; concurrent Promise.all → per-thunk try/catch, or Promise.allSettled). Keep the pure-core / impure-shell split intact; reuse the existing halt-dependent-subgraph machinery rather than reimplementing it (mirror E-049's discipline). PREREQUISITES (already satisfied, named as prerequisites not scope): the E-046 typed-DAG substrate, the halt/skip + decideThread gate, and the cycle-refusal path — all live. NON-GOALS / OUT OF SCOPE: do NOT re-propose or touch cycle detection (DONE — graph-core.ts:109-124); no retry/backoff or error-recovery policy (a throw becomes a marked terminal node, it is not re-cast); no live-model proof (deterministic stub thunk only — N4: Vend clears, it does not execute); no change to the success or budget-exhausted paths beyond routing throws alongside them. Right-sized as one ~2-hour feature block (PE-7): one pure error-outcome unit + threading it into the two runners + deterministic sequential-and-concurrent tests.
