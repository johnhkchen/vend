# E-077 — decompose-fails-soft

## Intent
A lisa-project field report (2026-07-13, vend 0.4.0-rc.5, their E-045) shows decompose failing hard and explaining itself dishonestly. Two default-path casts ran deep into the agentic fat tail (displayed 392k/200k tokens, 23/15 turns), stalled, were manually interrupted, and left an orphaned epic and a red doctor. One retry refused the *entire* spent generation over two bare charter codes (N4, N2). A `--no-gates` retry cleared at 22k/2 turns — leading the reporters to conclude gates were regenerating work. Their mechanics read is wrong (castPlay is ONE dispense; `--no-gates` changes nothing about generation), but every pain is real, and all four reduce to the same fault: the decompose surface does not fail soft and does not tell the truth about what it is doing. The play is to close that gap on the live decompose path across four slices: (1) a cited charter code the charter can't resolve **degrades, not discards** — strip or annotate the cite with an honest marker on the run record and materialize the board; refusal is reserved for structural invalidity (graph, IDs, missing fields, story-contract absence); (2) the live progress line finishes T-072-04's job — stop counting stream events against an agent-turn denominator (the 23/15 unlike-units bug the summary line already fixed), count the capped unit or label both, and mark the token fraction detect-after in-line; (3) an interrupted or failed decompose leaves a **resumable** state — persist epic, latest parsed draft, gate findings, and next repair action; doctor names it `resumable-decompose` with the resume command, and `vend run decompose-epic` resumes from the draft instead of re-dispensing cold; (4) characterize in a test whether `--max-turns` actually binds at the executor seam. This matters NOW because a field operator concluded the contract itself was hurting them — that is trust debt, and it is compounding on the surfaces we ship.

## Value
Realizes fail-soft, self-explaining decompose on the live path. Advances **P3** (refusal proportional to the finding — a two-line editorial defect can't cost a whole cast; the gate stays a contract, not a cliff), **P7** (spend legible at the moment it happens — detect-after made visible, not implied — and unlike-units removed so every displayed number is a real boundary), **P4** (walk-away is only safe if interruption is safe; a resumable draft is what makes autonomy trustworthy on the long tail), and **P5** (the operator concluded gates hurt because the local surfaces couldn't explain themselves — on a local-first tool, label debt IS trust debt). Directly serves the core feature: the clearing house's guarantee is worthless if operators route around it because the surface lied.

## doneLooksLike
- A decompose that cites a charter code the charter cannot resolve in story/ticket prose **materializes the board** with the cite stripped or annotated by an honest marker on the run record — it no longer refuses the whole cast. Refusal still fires for structural invalidity (bad graph, duplicate/missing IDs, missing required fields, absent story contract).
- The live progress line no longer shows unlike-units (no `23/15` of stream-events-over-agent-turns); it counts the capped unit or labels both, and the token fraction is marked detect-after in-line where the operator is watching.
- An interrupted or failed decompose is **resumable**: state (epic + latest parsed draft + gate findings + next repair action) is persisted; `vend doctor` reports `resumable-decompose` with the resume command rather than a bare orphan red; `vend run decompose-epic` on that epic resumes from the draft instead of re-dispensing cold.
- A test pins what the `--max-turns` CLI cap bounds, what our accumulator counts, and what happens at cap-hit at the executor seam (T-072-04-01's diagnosis extended to the live path).
