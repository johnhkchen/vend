## Intent — the bigger-picture play

A **single-use play** (a *sorcery*, per `card-model.md`) for a specific moment: now.
Cross-reference the knowledge base (`docs/knowledge/**`) with the project roadmap
(`demand.md`, the epics `E-001…E-005`, `.vend/decisions.jsonl`) and **plan the rest of
the stories** — the next wave of work, grounded in the charter and sequenced by the
value + budget model.

This is the genesis pattern itself: *bootstrap a project by reading what's known and
charting what's next.* It recurs across every lisa-loop project — which is exactly why
it's worth casting once here, watching what comes out, and later considering reprinting
it as a permanent (a reusable "bootstrap a project" playbook).

## Value to the design

It is `project-steering.md` (the meta-play) realized as one concrete cast, run
autonomously rather than hand-driven (**P4**). The output — a sequenced plan of the
remaining stories — is the forward motion a steering session is supposed to produce.
And casting it teaches us what the BAML single-use-play (sorcery) engine must support.

## Done looks like

- A **roadmap plan** artifact that: (a) cross-references the KB — cites which knowledge
  docs drive which choices; (b) lays out the next wave of **stories** for the staged
  epics (`E-002` CI, `E-003` shelf, `E-004` id-guard, `E-005` model-id, + the
  single-use-play / casting-engine capability); (c) sequences them by the value +
  budget model (`demand.md`); (d) **flags any contradiction** between knowledge docs.
- It plans *stories*, it does not materialize every ticket (that's each epic's own
  later cast).

## Context & constraints

- **Realized as a lisa epic → S-006 → T-006-01** (one story, one ticket). lisa reads
  only stories/tickets; this epic is the Vend-side card.
- Ids hand-namespaced to E-006 (`S-006`, `T-006-01`) so they can't collide with the
  live board — the by-hand version of exactly what `E-004` will automate.
- **Watch the friction:** forcing a one-shot planning *sorcery* through lisa's RDSPI
  (built to implement code *permanents*) is a deliberate mismatch — what "Implement"
  even means for a survey is the signal that motivates the BAML sorcery engine.
- Out of scope: building the casting engine; materializing the planned stories' tickets.
