## Intent — the bigger-picture play

The macro-wallet is Vend's **headline gesture**: fund a feature-block budget at the counter, walk
away, and let Vend spend it down autonomously — pulling the highest-leverage affordable signal,
casting the propose→decompose chain, debiting the wallet by the actuals, until a clean stop (P4 +
P7). It is **wired and green** (E-024 `castWork`/`spendDown`/`allocate`; E-025 fixed auth==exec so a
chain runs under exactly the price it was authorized at; E-027 refuses a stale board). E-025's
re-sweep even cleared **one** chain live.

But `castWork`'s own header says it plainly: *"IMPURE; NOT unit-tested … Proven LIVE."* The thing a
unit test cannot give is the thing the charter promises — **fund a block, walk away, and it spends
down across casts to a clean stop**. That full demonstration (multi-cast spend-down, not a single
re-sweep) has **never been watched**. And the trust gate the wallet rests on is **provisional**:
`vend audit` reads **forward (live) walk-away 50% (1/2)** — two genuine forward records against a
≥10 bar. The instrument is built (E-026); the *evidence* is thin.

This epic **casts the wallet for real**, once, bounded: stage a fresh board, fund a bounded macro
budget, run `vend work --no-intervened`, and watch it spend down. The run is doubly load-bearing —
it **demonstrates the headline feature** and **accrues the forward-E1** that ungates further
autonomy. Proving the loop *is* collecting the evidence that confirms it.

## The honest boundary — what this IS and is NOT (load-bearing)

- **This is a LIVE, METERED spend.** Real `claude -p` casts, real tokens, real wall-clock, real
  epics+tickets minted onto the board. It is **NOT** a free deterministic proof like the gate epics
  this session — the deliverable *is* the watched run. The **budget bounds it** (P7); a bounded
  block (recommend ~1h / ~1M tokens ≈ 2 chains) is a real-but-contained demonstration, not the 2M
  default.
- **One session moves the forward-E1 count; it does NOT alone meet the ≥10-genuine bar.** Each
  `--no-intervened` chain that clears is **one** forward walk-away record. A sweep clearing 2–3
  accrues 2–3. The bar is met over repeated sessions — so this epic **moves the count and sets the
  cadence**, it does not close the bar in one shot. **Do not over-claim "forward-confirmed"** off a
  small sample — that is exactly the E-026/T-026-04 trap this project already corrected once.
- **Trust = walk-away QUALITY, not just that it cleared.** The settle assesses whether what it
  autonomously minted is *sound* (a real, grounded epic for a real frontier), not merely that the
  wallet emptied. A walk-away that clears junk is not trust.

## Value to the design

Turns the project's central claim from *"Vend **can** work for two hours"* into *"we **watched** Vend
work for two hours."* It is the single highest-leverage move on the board: it exercises P4
(autonomy) and P7 (the budget contract) end-to-end on the real executor, and it feeds the forward-E1
gate that the rest of the autonomy roadmap waits on. Whatever it clears is genuine forward progress
(the board is down to the six frontiers — the sweep pulls a real one and decomposes it).

## Done looks like

- **Pre-flight (free):** deterministically confirm the metered run *will* clear — the wallet prices
  the chain from the live ledger, a bounded budget affords ≥1 chain (`canAfford`/`fitNext`),
  `--no-intervened` threads `intervened:false` into the cast so cleared records carry forward-E1, and
  the freshness-gate math passes for a board staged at run time. A go/no-go that auth==exec holds
  (E-025), so no live spend is burned for nothing (the E-024 "cleared 0" lesson).
- **The live sweep (metered — the proof):** a fresh board staged (`vend steer`/`survey`, a live
  cast), a bounded macro budget funded at the counter, `vend work --no-intervened` run against it —
  spending down across casts to a **clean P7 stop** (wallet-exhausted or board-cleared; nothing
  partial; truthful receipt). **≥1 real pull cleared** end-to-end (a real epic+tickets minted, `lisa
  validate` green on them), with auth==exec (E-025). The forward-E1 record(s) appended to the ledger
  carrying `intervened:false`.
- **The settle (free):** `vend audit` forward (live) count moved off 1/2 — the new forward rate
  recorded; an honest assessment of WHAT cleared (sound or not), whether P7 held, and whether the go
  stays **provisional** with a named cadence to ≥10, or graduates. **No over-claim.**
