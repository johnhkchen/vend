## Intent ‚Äî the bigger-picture play

The Ledger recalibrates a play's envelope from the p90 of its SUCCESSFUL runs (E-013), and right-censors andon'd runs (`budget-exhausted` / `timed-out`) out of the percentile sample ‚Äî correct, because a censored run's true cost is only a lower bound (IA-13). But the funding envelope a run is then RUN under is that same thin percentile (cold-start: the hand prior). So when the prior under-bounds ‚Äî as `decompose-epic`'s 120k token prior does ‚Äî the run hits the wall, is censored, contributes nothing to the success sample, and the **next** run is funded at the same thin guess and censors again. The envelope can never learn the tail it keeps cutting off. This is the censoring ratchet, now TWICE confirmed: E-045 (the heavy DAG epic depleted decompose) and E-049 (`vend chain`'s decompose exhausted at ~265k against 120k, clearing only when hand-bumped to 350k).

`recalibrate.ts` itself names the missing rung ‚Äî it READS the censored rate into `confidence` but "does not yet ACTUATE on it (IA-14 ‚Äî auto-widen/slow-tighten ‚Äî is a later rung)." This epic is that rung, shaped by a hard fact: **a censored run already RECORDS its actual usage** (the budget-exhausted decompose logged 264,866 real tokens before the wall). That recorded usage is a strong lower bound on the true cost ‚Äî currently thrown away. The fix: in the under-calibrated regime (cold-start, or a high recent censored rate), the funding envelope a cast RUNS under is widened to clear the observed censored lower bounds (√ó a measurement headroom), so the run FINISHES and records its real cost ‚Äî which then enters the success sample and lets `recalibrate` tighten honestly from data. The quoted PRICE (shelf / affordability / the confidence label) stays the honest p90/estimate ‚Äî exactly as E-038 split the wall-clock runaway-guard (`timeoutMsFor` √ó headroom) from the price. We prefer early runs to record what they use over terminating them off a guess.

## Value to the design

Turns the budget contract from a self-censoring ratchet into a self-calibrating loop, without weakening it. It advances P7 (the allocated envelope becomes honest and learnable ‚Äî a measured bound earned from data, not a guess that perpetuates itself) and P3 (the andon/censored-rate signal `recalibrate` already surfaces finally ACTUATES ‚Äî auto-widen ‚Äî instead of being read and ignored). It is the token-dimension twin of E-038's wall-clock headroom, and the actuation half of E-013's measurement loop: the difference between a calibration that can only ratchet tighter-and-wronger and one that finds the true cost from what early runs actually spend.

## Done looks like

An under-calibrated play (cold-start, or a high recent censored rate) is FUNDED with a measurement headroom over the observed censored lower bounds, so a run that would have hit the thin wall instead finishes and records its actual usage; that recorded cost then enters the success sample and `recalibrate` proposes a higher, honest p90. Concretely: an E-049-shaped `decompose-epic` (120k cold-start prior, ~265k true cost already logged once as a censored lower bound) is now funded above ~265k and COMPLETES, no manual `--budget` bump. The quoted price / shelf / `formatEnvelopeLabel` is UNCHANGED (still the honest p90/estimate ‚Äî IA-8: the meter doesn't lie). P7 still holds ‚Äî the funding headroom is a FINITE runaway-guard (a bounded multiple, like `timeoutMsFor`), never unbounded, so a run still cannot overspend its guard. Proven deterministically with stub run-records (no live model); the existing check gate green.
