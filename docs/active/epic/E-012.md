## Intent — the bigger-picture play

Close the check:committed gate's self-exempt blind spot. Today SOURCE_PREFIXES (committed-core.ts) is ["src/", "baml_src/", "ci/"] — it polices the application source but NOT .lisa/hooks/, the very shell scripts (on-stop/on-clear/on-heartbeat/on-idle) that trigger the gates. An edit to a hook can sit uncommitted or untracked and the gate stays green, meaning the enforcement layer is the one layer not under enforcement. The play: widen the shared R12 contract so authored lisa hooks are first-class policed source, and prove it with the classifier's existing pure-test discipline. This matters NOW because E-008/E-010 just made hooks the live trigger surface — the enforcers are actively being edited, so the gap is load-bearing, not theoretical.

## Value to the design

Realizes a self-consistent commit gate: the contract that defines 'uncommitted source' (P3 — gates are the contract) extends to the scripts that invoke that contract, removing a class of silent drift (D-005-style) in exactly the files most able to corrupt the gate. Strengthens the clearing guarantee — nothing that enforces settlement can itself be left unsettled.

## Done looks like

An uncommitted or untracked file under .lisa/hooks/ causes `bun run check:committed` to ANDON (exit 1) and lists that file as an offender, exactly as a dirty src/ file does today; a clean tree with all hooks committed still exits 0; the widened scope lives as a one-line change to the single SOURCE_PREFIXES contract (every consumer derives from it, none re-list it), backed by committed-core's pure tests covering a dirty-hook case.

## Context & constraints

Ground: src/ci/committed-core.ts:28 defines SOURCE_PREFIXES = ["src/", "baml_src/", "ci/"] as the R12 SHARED CONTRACT; classifyPorcelain matches paths against it; check-committed.ts is the thin impure shell; tests are pure (committed-core.test.ts). The fix is a one-line widening plus test coverage — comfortably under one 2-hour block (PE-7), small/common. SCOPE PRECISELY: police .lisa/hooks/ (the authored shell enforcers), NOT all of .lisa/ — that tree may hold lisa-managed runtime/state that is legitimately uncommitted or gitignored; widening to the whole dir risks false ANDONs and is out of bounds. Prerequisite: none — committed-core/check-committed already exist (E-008) and are live (E-010 wired on-clear). Non-goals respected: still a check:* script, no executor behavior added (N4); does not touch the hook trigger mechanism, check:head, or the Dagger /ci path. Out of scope: any change to how hooks are invoked, or to gitignore policy for .lisa/.
