## Intent — the bigger-picture play

Today a vend cast that hits a missing or misconfigured dependency fails the way an unguarded program fails: somewhere mid-run, with a cryptic stack trace, after the user has already committed a budget. Author `vend doctor` — an `envinfo`-backed preflight that probes the real runtime deps and reports honestly: lisa and claude on PATH, the BAML client bundled and its native addon loadable, and the executor-specific config (default Claude needs no API key; the open-model path needs its endpoint vars). A failed check is a clean gated refusal — a specific, actionable fix-it hint and a non-zero exit — never a crash. Mirror lisa's own `check_required_deps`-before-`run_loop` pattern so the same check also stands as a precondition before a cast. This is the smallest, most-ready piece of Frontier 7 (PRD §7.2 feature 2; §8 v1) and is immediately useful now — it gives the in-flight live sweeps a one-command way to confirm the vend+lisa combo is wired before spending a budget, instead of discovering a broken dep at p90 kill-switch time.

## Value to the design

Realizes a self-diagnosing correctness gate. It advances P3 (gates are the contract — doctor IS an enforceable precondition gate; a dep that can't be verified means the cast doesn't settle), P4 (autonomy by default — a deliberate, gated refusal at the door is exactly the escalation event the charter wants, replacing an unsupervised mid-run failure), and P6 (executor-agnostic — the check is written around the executor seam, verifying default-Claude vs open-model config rather than assuming one executor). It directly serves the core feature: clearing work you can trust at volume requires that 'the environment can actually run this' be checkable, not assumed.

## Done looks like

Running `vend doctor` in a correctly wired project reports each dep green and exits 0. Running it in a project with any single dep broken — lisa or claude off PATH, the BAML addon unloadable, or the active executor's required config absent — produces a clean refusal: the specific failing check named, an actionable fix-it hint, and a non-zero exit, with no stack trace. The same check, wired as a precondition before a cast, makes a cast against a broken environment refuse cleanly at the door rather than crash partway through a metered run. Observably: a missing dependency becomes a one-line 'here's what to fix,' not a cryptic mid-run failure.
