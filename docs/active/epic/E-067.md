_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

When materialize cuts stories and tickets, it should resolve every cited charter shorthand code (P/N/PE) into the one-line text it carried at cut time, keeping the code alongside for traceability. Today a cut artifact says only 'advances: P3' — a bare pointer that dereferences against whatever the charter says *later*, so meaning drifts after cut, and the opaque code leaks verbatim into downstream agent context where it means nothing. This bit for real: honey-kitchen dogfood's #1 prioritized fix is auto-strip N-codes, and cross-vendor agent:codex workers arrive with no charter at all to dereference against. The play is snapshot-at-cut: pay the resolution once, at the moment of materialization, so the artifact is self-contained forever after.

## Value to the design

Makes cut artifacts trustworthy inputs for autonomous execution (P4): an agent works against the ticket's own carried meaning, not live human clarification of what 'N2' meant. Makes them executor-agnostic (P6): a worker with zero access to Vend's charter — the agent:codex case observed in dogfood — still receives full grounding. And it converts charter drift from silent semantic rot into a non-event: the artifact preserves what the code meant when the work was warranted.

## Done looks like

A freshly materialized story or ticket that cites any charter code shows the code plus its carried one-line text inline (e.g. 'P3 — Gates are the contract: quality lives inside the work'), and reads fully standalone with the charter file deleted from view. A downstream agent transcript over such an artifact contains no bare, unexplained P/N/PE codes — the honey-kitchen 'strip N-codes' complaint has nothing left to strip.

## Context & constraints

Grounds directly on the E-066 story-layer arc just landed: materialize already writes the story contract body (T-066-01-03) and the story-completeness gate STOPs shell artifacts (T-066-01-02) — this epic extends the same cut point, so E-066's materialize path is a prerequisite (already in place, not hidden scope). Source of demand: honey-kitchen dogfood fix #1 (auto-strip N-codes) and the agent:codex no-charter observation. In scope: resolution at cut time in materialize, for stories and tickets, code kept for traceability. Out of scope: rewriting the charter, retroactively patching already-cut artifacts, or read-time stripping in downstream tooling — fixing at the write side makes those unnecessary. Respects N1–N4; the charter itself stays the single mutable anchor (its amendment rule is untouched — snapshots capture, they don't fork it). Right-sized at one ~2-hour block: one cut-time resolution step plus a gate check that cut artifacts carry no bare codes.
