## Intent — the bigger-picture play

E-025's live proof exposed a correctness gap the moment `vend work` started working: it spends down
**whatever staged board it finds**, with no regard for the board's age. The proof spent a `steer.md`
from 11:54 against a project that had moved through the entire measurement sprint + E-024/E-025/E-026
by 22:30 — and cleared its #1 signal ("run the E1 sprint") into a real epic, on a premise
(*E-014 is HOLD*) that was already `go`. The spend was mechanically correct; the **board was stale**,
so the work it cleared was superseded. A wallet spent on a stale board is **overproduction** (`tps.md`)
— the most expensive kind, because it looks like progress.

The fix is a **freshness gate** in the andon family (IA-9): before funding the wallet, compare the
board's staging time against the project's last change (the newest of `docs/active/{epic,stories,
tickets}`). If the board predates the live state, **refuse** — amber, "the board is stale, re-survey
before spending," with the next move handed over (the andon summons the fix, TPS) — rather than spend
the wallet on yesterday's demand. An explicit `--stale-ok` override keeps the human in charge (IA-5).

## Value to the design

Makes the macro-wallet's promise honest: "fund it, walk away, it clears the **right** work" — not
"whatever was on the board whenever it was last staged." It is P3 (a gate is the contract — here, the
contract that an autonomous spend reflects current demand) wearing the IA-9 andon stance (a stale
board is a *successful refusal*, not an error), guarding P4 (the autonomous loop). Without it, the
headline feature can clear stale work convincingly — the worst failure mode for a consistency product.

## Done looks like

- **A freshness gate:** `vend work` reads the board's staging time + the project's last-change time;
  if the board is older, it returns a **`stale-board`** outcome instead of funding the wallet.
- **An andon refusal (IA-9):** the CLI renders it amber — "board staged <when>; project changed since
  <when> — re-survey before spending" — and hands the next move (`vend survey`/`vend steer`, then
  `vend work`). A *successful refusal*, exiting like the other broken-precondition outcomes
  (no-board / empty-board), never a crash.
- **An override (IA-5):** `--stale-ok` spends anyway — the human stays in charge; the default is safe.
- **Proven free + deterministic:** `vend work` against the live (stale) `steer.md` now refuses
  *without casting* — no LLM spend needed to verify (unlike E-024/E-025).

## Context & constraints

- **Cites:** `src/play/work.ts` (`readBoard` + `castWork` + the `WorkResult` union — add the
  `stale-board` variant and the freshness check before `allocate`); `src/cli.ts` (`parseWorkArgs` +
  the `work` dispatch arm — render the andon, add `--stale-ok`); `work/T-025-01/sweep-logs/findings.md`
  (the concrete failure); `information-architecture.md` IA-9 (andon = successful refusal, amber), IA-5
  (recommend-never-auto — the override, not silent auto-survey), IA-1 (the board is the demand the
  spend honors); `tps.md` (stale-board spend = overproduction).
- **The staleness signal:** the board is stale iff its mtime < the newest mtime across
  `docs/active/{epic,stories,tickets}`. Local, no new plumbing. **Honest caveat:** mtime is a
  heuristic (git operations can reset it) — hence `--stale-ok` as the escape hatch and an honest note,
  not a hard lock. A stronger signal (a board-embedded state fingerprint) is a downstream refinement.
- **Non-goals:** **auto-re-survey** (silently casting survey/steer to refresh — that spends the
  wallet on work the human didn't fund; it violates IA-5/the funding gesture; a `--refresh` opt-in is
  a *downstream* pull, deliberately not this slice); per-candidate "is this specific signal still
  live" filtering (a different, harder problem — the gate is board-level, not signal-level);
  re-deriving the board (survey/steer own that). Not an executor (**N4**).
- **First slice = the board-level freshness gate + the andon + `--stale-ok`** (one ticket). **Out:**
  auto-refresh, per-signal liveness, state-fingerprinting.
