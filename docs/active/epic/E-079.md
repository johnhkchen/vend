_Proposed by Vend's `propose-epic` play._

## Intent — the bigger-picture play

A full working day at the vend desk currently reduces to the operator retyping machine state at machines: 'lisa loop done' relays an event lisa already emitted, 'status update' commissions an inspection of board state vend can already read, and the sweep hand-assembles a closeout that is pure computation over done tickets. Only the pull is genuinely human. This epic closes the loop from the machine side: `vend settle` makes the after-loop verdict a FREE deterministic gesture — board delta since the last settle, gate and presweep result, named open review concerns, exceptions in red with the exact next action; `vend sweep` makes closeout a gesture — compute the card flips for epics whose tickets are all done, stage them with provenance as one pathspec-limited commit, present it for one-keystroke confirmation; and the andon cord carries lisa's loop-complete event across the seam as a loop-settled marker vend can read, so settle can be event-triggered (or run standing) and the relay prompt disappears. What remains of the operator's day is exactly what P4 always promised: pull deliberately, handle exceptions.

## Value to the design

This is the honest root of forward-E1 sitting at 0/10: the walk-away promise is false while the walk-back — relay, inspect, hand-assemble — is load-bearing. Retiring the message-bus and inspector roles advances P4 directly; the transaction stays two gestures because the OTHER four standing gestures disappear rather than being better tolerated (P2); and settle turns the gate contract into a surface — the one-screen verdict is the contract's state made visible, not a new judgment (P3). It also resolves the staged detached/notify design pull by grounding it in the real loop instead of a hypothetical UI.

## Done looks like

After a lisa loop completes, the loop-settled marker reaches the operator without polling or relaying; `vend settle` (typed, or triggered by the event) prints for free the one-screen verdict the operator today commissions by hand — 'E-078: 7/7 cleared on codex, gate green 1831 tests, 1 review concern named, sweep ready' — with exceptions in red and the exact next action; `vend sweep` shows the assembled closeout commit and lands it on one keystroke. A full steer→chain→build→sweep cycle, observed end-to-end, contains exactly two human gestures: the pull and any exception.

## Context & constraints

Grounded in the 2026-07-13 operator reflection (a day at the desk = typing 'lisa loop done; status update; sweep; pull and chain' on repeat — a factory worker monitoring machine signals). Prerequisite, named as such: the andon cord CONSUMES lisa's existing on-stop/ntfy plumbing across the vend/lisa seam — the loop-settled marker is a seam contract to be agreed with lisa, not invented unilaterally on vend's side. Settle composes existing machinery (the `bun run check` gate, presweep, board frontmatter phases, work dirs, review artifacts): it reads and reports; it adds no new judgment. Deliberate confirmation stays in sweep — the assembly is machine work, the keystroke is not. Non-goals respected: settle is a pull- or event-triggered verdict gesture, not a babysitting dashboard (N2), and no step-approval enters the loop. Out of scope: auto-pulling signals (promotion stays a deliberate human gesture — N-invariant), any TUI (the interactive surface re-enters later as `vend interactive`; CLI+SVG remains v1), and rehoming the --intervened self-report (fork 1 stays open, a separate ruling). Sizing (PE-7): three slices puts this at the top of one ~2-hour envelope; the adopt-path mint-validation jank (propose's adopt-before-mint accepting agent-written cards vend's own graph loader then refuses, hand-repaired 2026-07-13) rides along as slice 4 ONLY if decompose prices it in — otherwise it remains the next staged candidate, not hidden scope here.
