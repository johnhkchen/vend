## Intent — the bigger-picture play

E-024's first autonomous spend exposed a half-wired premise. The wallet recalibrates each pull's
price — **propose 227k + decompose 227k** (E-013) — `canAfford`-authorizes on it, and reserves it.
But `castWork` then calls `castProposeDecomposeChain` **with no budget**, so each step runs under
its *static play default* (propose-epic's 150k). The #1 pull's propose needed **175k** — under what
the wallet reserved (227k), over the default it actually cast with (150k) — so it `budget-exhausted`
andon'd and **cleared 0**. The wallet knew the price, authorized on it, then didn't spend at it.

This epic threads the reserved price into the cast: the chain accepts **per-step budgets** (propose
and decompose recalibrated separately — they can differ), and `castWork` passes the same two
envelopes it already computes for `canAfford`. Then **authorization == execution**: a pull the
wallet says it can afford runs under exactly that budget, and clears.

## Value to the design

Makes P7 honest in both directions: not just "the wallet refuses what it can't afford" (already
true — proven live) but "the wallet *spends at the price it fitted*." Without this, the macro-wallet
authorizes on measured prices but executes on cold-start guesses — the two numbers that E-013 worked
to separate, re-conflated at the last inch. It is the wire that turns `vend work` from built-but-inert
into clearing-real-work (P4), and unblocks the live re-sweep that becomes the forward E1.

## Done looks like

- **The chain casts at the wallet's price:** `castProposeDecomposeChain` accepts **per-step**
  budgets (propose / decompose); `castWork` passes the recalibrated envelopes it already computes
  (the same numbers `canAfford` authorized on). The uniform `budget` stays for back-compat.
- **Authorization == execution:** a pull the wallet affords casts under the reserved per-step
  budget, not the static play default. The E-024 sweep's #1 pull (propose ~175k) now clears under
  its 227k reservation instead of andon'ing at 150k.
- **Proven by re-running the live sweep:** `vend work` against the staged board clears **≥1** pull
  (the AC E-024's sweep deferred), Settles truthfully, hard-stops still hold.

## Context & constraints

- **Cites:** `src/play/work.ts` (the call site, line ~130 — omits the budget; it already computes
  the two recalibrated envelopes, today only summed for `canAfford` then discarded); `src/play/chain-
  propose-decompose.ts` (`ChainProposeDecomposeOptions` — already has a *uniform* `budget`; extend to
  per-step); `src/ledger/recalibrate.ts` (E-013 — the per-play price source); `work/T-024-03/sweep-
  logs/findings.md` (the diagnosis + the exact failure); IA-8 (the two denominations the per-step
  budget carries).
- **Not the fix:** bumping propose-epic's static 150k default — `vend work` already has the better
  measured number (227k) and must *use* it; the static default stays the cold-start fallback for a
  bare `vend run`. Don't re-litigate per-play envelopes here (E-013 owns them).
- **Hard-stops unchanged (P7):** the reserved price is what the cast may spend; the wall-clock still
  halts mid-flight, the token meter still depletes detect-after, an andon still settles the session.
  This wire only changes *which number* the cast runs under, never that it can overshoot.
- **Non-goals:** per-candidate pricing by board tier (the chain casts the same two plays — flat price
  is correct today, the tier-aware refinement is E-024's noted downstream); a budget-split policy
  beyond per-step (propose-vs-decompose is the only split). Not an executor (**N4**).
- **First slice = the wire + the live re-sweep proof** (one ticket).
