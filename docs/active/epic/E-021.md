## Intent — the bigger-picture play

A designer/teammate needs to see and collaborate on the work Vend authors, but the canonical epic→story→ticket DAG (markdown + YAML frontmatter) is dev-layer jargon — illegible to them and unsafe to edit directly. The bigger-picture play: stand up the presentation layer as a clean data/presentation split. Vend's graph stays the fixed source of truth; the human reads a *projection* of it governed by a *presentation spec* they can tune (vocabulary, density, field-visibility, grouping, metaphor, labels, color-language). Same graph, many renders; calibration edits the spec, never the data. Start MCP-independent on purpose — prove the projection + its calibration against Vend's own live board (paper/markdown render) before any Linear renderer is wired in. One-way authority is the hard boundary: the projection reads the graph and never writes back.

## Value to the design

Realizes a reusable read-only projection capability and a calibratable presentation-spec — the data/presentation seam that turns the authoritative work-graph into human-legible presentation without exposing or risking the canonical source. Advances P5 (local-first: one-way authority keeps canonical state owned and offline on this machine; the surface renders, never writes) and P1 (the presentation/calibration spec is authored once and reused by every render — cost lives at authoring, not per-view). It is the render contract every downstream surface epic builds on.

## Done looks like

Against Vend's current epic/story/ticket DAG, the presentation layer emits a human-digestible projection (plain title · why-it-matters · state · breakdown, with dev detail behind disclosure) governed by a calibratable presentation spec. Re-tuning a knob in the spec demonstrably changes the rendered projection without editing any dev-layer markdown; a saved 'designer' preset reproduces the same view on demand. The projection is provably one-way: no path writes back to the canonical graph. Verifiable at the epic level by rendering the live board in the designer preset and re-calibrating it against the five-dimension 'good enough' rubric (comprehension, structure, density, language, navigability).
