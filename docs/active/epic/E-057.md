## Intent — the bigger-picture play

Give the SVG surface (E-055/E-056) its non-dev verb. The board is now glanceable and shows what's blocked; this is what a designer or founder DOES with it — Maya's originating 'telephone-killer' request. A teammate annotates a node on the rendered work-graph; that feedback comes back to Vend as a PROPOSED, priced demand signal a steward pulls to clear. It is authoring (capture judgment once, into a priced signal), not babysitting (N2/P4) — two-way data, one-way authority. Job-story 7 (docs/active/pm/job-stories-visual-surface.md) is the contract. The play REUSES expand-fragment (E-016): an annotation is just a fragment plus provenance, so the clearing path, gates, and staging are inherited, not rebuilt. This is intent only; the slice breakdown (provenance/back-link core, then the `vend annotate` CLI seam) is DecomposeEpic's job.

## Value to the design

Realizes the inbound half of the visual surface — feedback from the people who can't (or shouldn't) open the code becomes cleared demand the clearing house can allocate. It advances the core feature directly: a new, lower-friction mouth for intent feeding the same clearing function. P1 (the spec is captured once, at annotation, as a priced signal — never re-derived at run time). P4 (the round-trip is a deliberate, gated human pull off the staged queue, not a live approval loop — escalation as an event, not supervision). P5 (the gesture is MCP-independent: the annotation arrives as text input on one machine, with the live comment FETCH deferred behind a seam, exactly as E-055 deferred the live Linear render).

## Done looks like

Running `vend annotate <node-id> "<feedback>" [--seat designer]` produces a staged Signal under docs/active/pm/staged/ that (a) is priced by the same expand clearing as `vend expand`, (b) carries a PROVENANCE trailer naming the annotation's source (seat X, on node Y), and (c) carries a BACK-LINK to the annotated work item so the proposed→pulled round-trip is traceable. Observably: the board is NEVER touched — no write to demand.md, epic/, stories/, or tickets/ (one-way authority holds); and the new wiring is deterministically testable by stubbing expand-fragment's cast, exactly as expand-fragment's own tests do. `vend annotate` exists as a real, reusable gesture alongside `vend expand`.
