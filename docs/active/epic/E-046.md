## Intent — the bigger-picture play

The v1 vision sells 'typed, graph-structured agent orchestration,' but today the engine only casts single plays and walks one hardcoded linear propose→decompose chain — the graph it promises does not exist. This epic builds the foundational slice of that graph: a typed node/edge model a playbook can declare, and an executor that traverses it in topological order with the first genuinely non-linear shape — fan-out (one node feeding several) and join (several nodes feeding one). It matters NOW because this is the largest piece of vision-distance still open and the substrate everything composable (richer playbooks, the open-model runner) has to sit on top of; nothing downstream gets more expressive until the graph underneath does.

## Value to the design

Realizes the core 'typed, graph-structured orchestration' capability at its minimum honest form: plays compose into a declared DAG instead of a fixed chain. Advances P1 (the playbook declares its graph once at authoring; runs traverse it) and P6 (a typed graph the open-model executor can be slotted under without re-deriving control flow). It is the structural unlock for every later composition feature.

## Done looks like

A playbook can declare a multi-node typed graph containing at least one fan-out and one join; the executor runs that graph to completion in correct topological order — parallel branches both execute and their results converge at the join node — and an executable example demonstrates this end-to-end, covered by a test that fails against the old linear-chain-only engine.

## Context & constraints

Ground against the current engine: it casts single plays plus one linear propose→decompose chain (castWork composition layer / work-core.ts), behind the existing executor interface and play-casting primitive — those are PREREQUISITES this epic builds on, not part of its scope. PE-7 flag: the full signal (fan-out + join + CONDITIONAL edges + arbitrary composition) is larger than one ~2-hour block and should be treated as SEVERAL epics; this card is deliberately right-sized to the foundational substrate — the typed graph data model and a topological executor proving fan-out + join. Out of scope / follow-on epics: conditional edges, cycle/error semantics, budget accounting across parallel branches, and the open-model runner itself (the P6 consumer, downstream). Non-goals respected: N4 — the DAG orchestrates Vend's own clearing plays, it does not turn Vend into an executor of arbitrary user code; lisa still executes. No invariant is regressed to add the graph.

## Decomposition (hand-cleared 2026-06-20 — the autonomous mint's decompose hit budget-exhausted; the DAG is the architectural keystone, so resumed by HAND)

Generalize the existing **linear chain** (`chain-core.ts` `runChain` / `chain.ts` `castChain`) to a
typed graph, mirroring its **pure-core ⊥ impure-shell** split. The chain is a *path* graph (one
`upstream: string`, sequential); the DAG generalizes three things — a node/edge model (a node may have
many in-edges = **join**, many out-edges = **fan-out**), `upstream: string` → a *collection* of upstream
`produced` refs (a join receives several), and the sequential loop → **topological order with parallel
independent branches**. The `produced`-string thread (never a typed play) stays the edge payload, so
the pure core stays addon-free + fake-testable (the `chain-core.test.ts` discipline).

**S-046-01** (critical path 3): T-046-01 (**typed graph model + topological sort** — a `DagNode {id,
cast}` (the `ChainStep` generalized with a node id) + `DagEdge {from, to}` + `DagSpec`; pure validation
(edges reference real nodes; **acyclic** — cycles DETECTED+refused, not run, per the out-of-scope note);
a pure/total deterministic `topoSort`; fully unit-tested: linear/fan-out/join/diamond/cycle-rejected)
→ T-046-02 (**`runGraph` pure core** — generalize `runChain`: walk in topological order, thread each
node's UPSTREAMS' `produced` into its cast (a JOIN receives several; a FAN-OUT's output feeds all its
out-edges), per-edge `decideThread` halt that skips only the **dependent subgraph** (siblings run on);
PURE given injected casts; fan-out/join/halt proven with fakes) → T-046-03 (**`castGraph` shell +
parallel branches + the worked example** — the `chain.ts` analog injecting `adapt → castPlay`, running
ready independent nodes CONCURRENTLY (the "run plays in parallel" the card names); a deterministic
executable example — a declared graph with ≥1 fan-out + ≥1 join over STUB nodes (canned summaries, no
live cast — a live real-play graph is a downstream metered proof) — and the AC test that **fails
against the old linear-chain engine** (its join receives two upstreams `runChain`'s single thread can't
deliver)). Epic-scoped ids (R6 — `046` clean). When done: a playbook can declare a typed multi-node
graph with fan-out + join, and the executor runs it to completion in topological order, parallel
branches converging at the join — the v1 vision's substrate at its minimum honest form.
