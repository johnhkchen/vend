## Intent — the bigger-picture play

The cold-start move (IA-3/IA-4): a fresh or rough install has **no demand board** and the
human is out of articulation energy. `Survey` reads the project — repo + docs + run-log +
board state — and proposes a **ranked shortlist of the highest-leverage candidate moves it
sees**, staged for a human to review and pull. It is `expand-fragment` (E-016) at *project
scale*: where expand turns *one* fragment into a signal, Survey reads the *whole rough
project* and proposes the board — *"articulating is exhausting" → "review a list."* It is the
run-0 bootstrap the entire onboarding arc (IA-3→IA-1, State 0→1) names but has never had.

## Value to the design

Completes the cold-start experience (**P2** — the run-0 two-gesture: cast Survey → a board,
then pull). It is the second brick of automating the **STEER** stage (after `expand-fragment`):
the survey-the-rough-project move the PM agent does by hand, made a registered play. Not
trust-gated — ships while the macro-wallet is parked, and like expand it accrues E1 data by
getting the builder running Vend.

## Done looks like

- A `vend survey` gesture casts the play: project state → a **ranked set of candidate
  signals** (the `demand.md` shape, reusing E-016's `Signal`), **staged** (not auto-promoted)
  for a human pull.
- **Read, never invent (PE-1) + honest-empty (IA-4):** every candidate traces to real project
  state; a project with **no real demand gradient** yields an **honest empty board**, not a
  manufactured list. (The signature discipline — a board-stocker that fabricates is worse than
  none.)
- Ranked by **leverage** (highest first); cast through the one metered seam — gated, budgeted,
  run-logged.
