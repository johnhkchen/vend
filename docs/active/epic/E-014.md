## Intent — the bigger-picture play

Before Vend builds the **macro-wallet** (spend a 2h budget down autonomously), prove the
two load-bearing assumptions the whole autonomy story rests on — and which the PM
discovery found **unvalidated**: (A2) the gates earn enough trust that the author *walks
away*, and (A5) the gates actually make probabilistic output **consistent** (the core
promise — never measured). Both are testable now, cheaply, against `.vend/runs.jsonl`
data Vend already keeps. This epic builds the **minimum instruments** to read them and
returns a **go/reroute** decision. It is measurement, not a feature — the discipline is
to build nothing beyond what the two numbers require.

## Value to the design

Stops Vend building the wrong thing next (P4 — autonomy is the bet; don't scale it
unproven) and finally **quantifies the core promise** — how much the gates reduce
variance (P3 — gates are the contract; here we measure the contract holds). The single
most important number for the product's case, produced for a fraction of a build.

## Done looks like

- **E1 (trust):** a *minimal* mid-run **intervention capture** (an `intervened` flag /
  one-line self-report per run), plus a walk-away audit over `runs.jsonl` — andon-rate vs
  the IA-12 budget, outcome mix, cost-vs-envelope, and the intervention trend.
- **E2 (consistency):** one playbook run **5× with gates and 5× without** on a fixed
  input; the **gate-driven variance reduction** reported as a single number (needs a
  minimal `--no-gates` run mode or a raw-dispense comparison).
- **The decision:** a one-page findings note returning **go (build the wallet) / reroute
  (andon-UX if E1 weak; fix consistency if E2 weak)** — produced with **≤5 cheap casts**.

## Context & constraints

- **Cites:** `docs/active/pm/PRD-trust-consistency-evidence-gate.md` (the spec — KR1–KR4,
  §7 features); `brief.md`; `discovery-foundation.md` (E1/E2, the decision framework);
  `src/log/run-log.ts` + `src/ledger/recalibrate.ts` (the run-log substrate to read);
  `src/gate/gates.ts` (the gates E2 toggles off); `information-architecture.md` IA-12
  (the andon budget E1 reads against), IA-13.
- **Anti-scope-creep (the PRD's whole point):** measurement, **not a build**. No new
  service, no framework, no schema overhaul — one optional log field + one run flag. If it
  starts to grow, that is the creep this epic exists to prevent.
- **Honest limits:** one self-reporting user (the author); a tiny sample; the walk-away
  judgment is self-reported. A directional signal from one honest user beats a confident
  guess from none — accept it.
- **Non-goals:** the macro-wallet build itself (this *gates* it, downstream); turning
  intervention capture into a permanent auto-signal (IA-15, later); the P6 second-executor
  and pricing/WTP probes (their own briefs). Not an executor change (**N4**).
- **First slice = the whole thing** (it's ~0.5–1 block): the instruments + the findings note.
