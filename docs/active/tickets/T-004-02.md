## Context

Wire the detector (T-004-01) into the write so a collision **stops the line**
before any file is written. `materialize` (`src/play/materialize.ts`) currently
writes `WorkPlan` stories/tickets into the target dirs unconditionally; make it
first gather the ids already on the board and refuse on collision. Reuse the
existing-id listing already in `src/play/project-context.ts` (`listIds`). The
runner (`src/play/decompose-epic.ts`) surfaces and logs the outcome.

## Acceptance Criteria

- [ ] Before any write, `materialize` collects the existing story + ticket ids
      under its target dirs and runs `detectCollisions` against the plan's ids.
- [ ] On collision it **refuses — andon, no write** (a typed error / STOP naming
      the colliding ids); **zero files written** (verified on disk, not just a
      flag) — matching P7's "no partial materialization".
- [ ] The runner maps the refusal to a named run-log outcome (e.g. `id-collision`
      in `src/log/run-log.ts`'s `RunOutcome`) — one countable record, no partial
      materialization, a clear stdout andon.
- [ ] A fresh/disjoint board still materializes normally and passes `lisa validate`.
- [ ] A fixture test covers both paths: populated board → refuse (named ids, no
      write); fresh board → write. `bun run check:test`/`check:typecheck` green.
