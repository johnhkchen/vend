## Context

Make **bare `vend`** the fused DL-6 Home — and preserve the press contract exactly. Today the `browse`
arm (`src/cli.ts` ~533) calls `browseShelf` (gather → rank → `renderMenu` → **persist `.vend/menu.json`**
→ print the board). This ticket keeps all of that and **enriches what prints**: board (leads) + shelf
(receding) + ledger (foot), via T-031-01's `renderHome`.

The work:

1. **Gather the three regions** on the bare-`vend` path (instant, deterministic, no LLM):
   - **Board** — keep `browseShelf`'s rank + **persist the same `MenuCache` to `.vend/menu.json`**
     (the press contract). Reuse its `renderMenu` output as `boardMenu`.
   - **Shelf** — `shelfRows(registry plays, readRuns(.vend/runs.jsonl))` (E-030 / T-030-01).
   - **Ledger** — `auditWalkAway(records)` → `homeLedgerLine(report)` (T-031-01 / E-028), reusing the
     same `readRuns` records (read once).
   Then `renderHome({ boardMenu, shelfRows, ledger })` → print. `--all` still reveals hidden board rows
   (pass it through to the board region exactly as today).
2. **Preserve the press contract bit-for-bit.** `.vend/menu.json` shape/content is **unchanged** (still
   `visibleActions(ranked, all)`, same `stateHash`); `vend <sel>` (`pressShelf`, `src/shelf/press.ts`
   ~44) resolves against it by index, **independent of the new printed text**. The board stays the
   selection namespace; the shelf-beneath is informational only. No press regression.

Degrade gracefully: no run log yet → shelf rows are all cold-start `default` (E-030) and the ledger
foot is the honest "no runs yet" line (T-031-01) — `vend` still renders. Keep `browseShelf`'s
persistence as the single writer of the cache (don't double-write).

**Codebase-memory (use it — project coding-task suggestion):** before touching the `browse` arm,
confirm the persistence + press wiring with the **codebase-memory-mcp** (project
`Volumes-ext1-swe-repos-vend`, indexed/ready) rather than re-deriving by raw grep:

- `query_graph("MATCH (f)-[:CALLS]->(g) WHERE g.name IN ['browseShelf','writeMenuCache'] RETURN f.qualified_name, g.name")` — every caller of the browse/persist seam, so the bare-`vend` change is the only edit and nothing else regresses.
