## Context

Compose T-029-01's visual atoms into the **surfaces** — the part a TUI epic reads to know what to
build — and settle the **card-as-lens** decision. Continue `docs/knowledge/design-language.md` with a
DL principle per surface, each in the clean-typographic key, each with a **small ASCII reference mock**
(the kind the human picked — type-led, minimal chrome, amber only on the andon):

- **Home** — the recommendation-first board ("NOW + why" leading, IA-1) above the receding shelf;
  the ledger summary line (now provenance-split, E-028 — forward vs attested at a glance).
- **The Counter** — Confirm → Run → Settle (IA-6): the funding gesture, the production line during
  Run (IA-7, node-level not raw stream, against the two-denomination meter IA-8), the Settle receipt
  (what cleared / cost / remaining / stop reason — amber on an andon stop, IA-9).
- **The Ledger** — run history + the walk-away/andon readout (andon rate framed as "gates working,"
  IA-10, never a red defect count).
- **The card-as-lens decision (explicit DL principle):** the MTG card/mana model is a **doc/spec
  lens** (`card-model.md`, the epic cards), **not TUI chrome** — the surfaces above render *no* boxed
  cards; cost may appear as a compact inline glyph (e.g. `{U}`) at most. State it so no TUI epic
  re-introduces card chrome by default.

Keep it capped + durable; the mocks are *reference*, not pixel specs. Where a future TUI and this
charter disagree, fix one — no drift (the IA doc's rule).

**Cites:** T-029-01 (the atoms — palette/type/meter/andon — these surfaces compose); `information-
architecture.md` (IA-1 Home, IA-6 Counter, IA-7 production line, IA-9/10 andon/ledger); the live
`vend work` receipt + production line + `vend audit` split (E-028) as the reference the mocks
formalize; `card-model.md` (the lens being scoped out of chrome).
