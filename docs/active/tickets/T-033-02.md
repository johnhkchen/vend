## Context

Wire T-033-01's policy into a real git `pre-commit` hook, make it installable + un-silenceable, and
activate it. Four parts:

1. **The committed hook (`.githooks/pre-commit`).** A thin POSIX-sh invoker (mirror `.lisa/hooks/on-stop.sh`
   — *only invokes*, fail-open on its own errors): run the test gate (`bun run check:test`), capture
   exit + output, feed `{ ran, exitCode, stderr }` to `classifyPrecommit` (T-033-01), and **exit
   non-zero to BLOCK when `block` is true**, printing the named-failure message; exit 0 otherwise.
   Inability to run bun ⇒ allow (fail-open). Run **`check:test`** (headline; add `check:typecheck` if it
   stays sub-second) — **not** `baml:gen` (slow; `check:head` owns the full build at clear). The thin
   shell may call a tiny `src/ci/check-precommit.ts` runner that does the spawn + delegates to the pure
   core, keeping the shell minimal (the `check:committed` shape).
2. **The installer (`hooks:install`).** A `package.json` script that sets `git config core.hooksPath
   .githooks` (idempotent) so the committed hook is active — `core.hooksPath` is local config, not
   committed, so a fresh clone needs this once. Keep it portable (no absolute paths).
3. **The guard (`check:hooks`).** A `package.json` script + tiny runner reading `git config --get
   core.hooksPath` and passing it to `hookInstallState` (T-033-01) — prints active / "run `bun run
   hooks:install`" and exits non-zero when **not** active, so the gate can't be silently absent (the
   E-012 "police the gate" move). *(Wiring it into a lisa hook is optional/non-goal — the script alone
   satisfies this slice.)*
4. **Activate in this repo** — run the installer so `core.hooksPath` = `.githooks` here, and confirm.

Don't change `check:committed`/`check:head` (this is the third layer beside them); don't run the full
`check` per commit; don't add a lisa `on-commit` hook (git pre-commit is the seam).

**Codebase-memory (use it — project coding-task suggestion):** before wiring, confirm the invoker shape
+ script conventions with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`,
indexed/ready):
- `search_code(pattern: "check-committed|committed-core|process.exit|Bun.spawn", regex: true, mode: "full", path_filter: "^src/ci/")` — the thin-invoker + spawn + exit-code pattern to mirror.
Confirmed coordinates at authoring: `src/ci/check-committed.ts` (the invoker); `committed-core.ts`
(consumed pure core); `.lisa/hooks/on-stop.sh` (the fail-open hook); `package.json` `check:*` scripts.

**Cites:** T-033-01 (`classifyPrecommit` + `hookInstallState`); `src/ci/check-committed.ts` (the thin
invoker/spawn precedent); `.lisa/hooks/on-stop.sh` (fail-open POSIX-sh invoker to mirror);
`package.json` (the `check:*` script block — add `hooks:install` + `check:hooks`); `docs/active/epic/E-012.md`
(police-the-gate precedent); `docs/knowledge/ci-strategy.md` (hook-only-invokes).

## Acceptance Criteria

- [ ] A committed `.githooks/pre-commit` runs `check:test`, **blocks a red commit** (non-zero, naming
      the failure via `classifyPrecommit`), allows a green one, and **fails open** when it can't run.
- [ ] `bun run hooks:install` sets `core.hooksPath` to the committed dir (idempotent, portable); the
      repo is activated (verified: `git config --get core.hooksPath` → `.githooks`).
- [ ] `bun run check:hooks` reports active when installed and **exits non-zero / names the fix** when
      not (the gate can't be silently absent).
- [ ] **Live proof (free, no cast):** with a deliberately failing test in the tree, `git commit` is
      **blocked** by the hook naming the failure; with a green tree it proceeds; `check:hooks` reports
      active. (Demonstrate against a throwaway/temporary red, leaving the tree green.) `bun run check:*`
      green.
