## Context

The **E1 / trust** arm (PRD KR1–KR2). Two parts, both minimal:

1. **Intervention capture** — record one bit per run: did the author step in mid-run, or
   let it clear? An `intervened` field on the run record (and/or a one-line self-report at
   stop). Backward-compatible (absent ⇒ unknown). This is the only *forward-looking*
   instrument; the walk-away trend needs it.
2. **Walk-away audit** — a **pure** analysis over `.vend/runs.jsonl`: andon-rate vs the
   IA-12 budget, outcome mix (success / censored / budget-exhausted), cost-vs-envelope, and
   — once data accrues — the **intervention trend** (target: → 0 while andon stays in budget).

**Anti-scope-creep (PRD §7):** one optional field + one analysis. No schema overhaul, no
new service.

**Cites:** `src/log/run-log.ts` (the record — add `intervened`); the pure reader from
E-013 (`recalibrate.ts` / run-log reader); `information-architecture.md` IA-12 (the budget
the andon-rate is read against); the PRD KR1/KR2.

## Acceptance Criteria

- [ ] The run record carries an optional `intervened` bit (backward-compatible; absent
      records still parse), writable at run-end (a stop-time self-report path and/or flag).
- [ ] A **pure** walk-away-audit function over a run-log slice returns: andon-rate vs
      budget, outcome mix, cost-vs-envelope, and the intervention rate/trend; unit-tested
      against a fixture log (incl. records with and without `intervened`).
- [ ] A surface prints the E1 findings fragment (the trust numbers) for the findings note.
- [ ] `bun run check:*` green; existing run records unaffected.
