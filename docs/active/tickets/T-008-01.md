## Context

The gate's logic, in one place (the Central Rule — `ci-strategy.md`): a
`bun run check:committed` that **fails when there is uncommitted or untracked
*source*** (`src/`, `baml_src/`, `ci/`) in the working tree — i.e., the code isn't
committed. Pure classifier (parse `git status --porcelain` → the offending source
paths, ignoring gitignored runtime) plus a thin git invocation (R1). This is exactly
what would have caught the E-001 / E-006 / E-007 residuals (`D-005` / `D-010`).

**Cites:** `D-005` / `D-010` (the recurring andon + the regression); `ci-strategy.md`
(the Central Rule — logic in `check:*` scripts); `.gitignore` (the allowed-runtime
exclusions — `.vend/*` except `decisions.jsonl`, `node_modules`, `baml_client`).

## Acceptance Criteria

- [ ] `package.json` has a `check:committed` script that exits **non-zero** iff
      `git status --porcelain` shows uncommitted/untracked paths under `src/` /
      `baml_src/` / `ci/`; exits **0** otherwise. Gitignored runtime is never a failure
      (it won't appear in porcelain anyway — verify).
- [ ] A **pure** helper classifies porcelain output → the offending source paths (the
      `git` call is the thin impure verb); unit-tested with porcelain fixtures: dirty
      source → fail-list; clean → empty; an untracked `src/*.ts` → flagged.
- [ ] `bun run check:test` / `check:typecheck` green; the script runs standalone.
