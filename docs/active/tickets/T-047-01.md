## Context

**Build the first real-play graph and cast it LIVE through `castGraph`.** E-046 proved the DAG substrate
with stub nodes; this proves it carries real plays + real concurrency. The graph:

```
survey (project → board path) ──┬─→ propose-epic (board → signal #1 → epic path) ──┐
                                └─→ propose-epic (board → signal #2 → epic path) ──┴─→ capture-note (both epics → consolidating note)
              FAN-OUT (run CONCURRENTLY by castGraph)                                JOIN (multi-upstream)
```

> **This ticket performs a LIVE, METERED spend** — ~4 real `claude -p` casts (survey + 2 proposes +
> note); the 2 proposes run concurrently. The human running `lisa loop` authorizes it; the budget
> bounds it (P7). NOT a free proof.

Two halves:

1. **Build the real-play `DagSpec` (deterministic, unit-tested with stubs).** Mirror
   `graph-example.ts`'s pattern but with REAL play nodes — each node a `DagNode {id, cast}` where `cast`
   is the `adapt → castPlay` wiring (the `chain.ts` `PlayStep`/`adapt` shape, generalized to
   `NodeUpstreams`):
   - **survey node** — `adapt` (no upstream) → survey inputs (the project); produces the staged board path.
   - **two propose nodes** — each `adapt(upstreams)` reads the survey node's produced board path, applies
     **`parseBoardSignals`** (`work-core.ts`) to take signal #1 / #2, and builds `ProposeEpicInputs`
     (`assembleProposeEpicInputs` or the existing helper); produces an epic path. (Guard: if the board
     has < 2 signals, the graph degrades honestly — record it.)
   - **capture-note node (join)** — `adapt(upstreams)` consumes **both** propose nodes' produced epic
     paths (the multi-entry `NodeUpstreams` map) and builds the note text consolidating the two epics;
     produces a note path.
   - **Unit-test the wiring deterministically** (stub `castPlay` / canned summaries, the
     `graph-example.test.ts` discipline): the fan-out adapters extract signal #1/#2 from a fixture
     board; the join adapter receives BOTH epic paths and builds a note referencing both. No live model
     in the unit test.
2. **Cast it LIVE through `castGraph`** (`bun run …` a small entry, or `vend`-verb if wired). Capture to
   `docs/active/work/T-047-01/graph-cast-log.md` (verbatim): the run, **evidence the two proposes ran
   CONCURRENTLY** (overlapping start/end timestamps in the run-log, or the wave dispatcher's order), the
   join note's content (it references BOTH minted epics), and the minted epic ids + note path. Confirm
   `lisa validate` after (E-043 held — no orphan).

**Honest on outcome:** the headline is the **concurrency** (two real casts overlapping) + the **join
receiving both** — not the artifacts. A degraded run (e.g. a propose andon) is recorded with the cause.
P7 holds (bounded, clean).
