## Context

**Recalibrate the steer/survey ranker so a demand signal must be concrete product demand — demote
self-referential meta-tasks beneath it.** Both live sweeps (E-037, E-039) had `vend steer` rank a
self-referential signal #1 (*"Run and settle the E-037 live macro sweep…"*); the operator re-pointed
at concrete demand by hand (E-039's sweep-log). An unattended sweep (P4) can't re-point — it 0-clears
or mints a degenerate meta-epic.

**Why a prompt fix, not a gate (grounded):** steer ranks via the **BAML model** (`baml_src/steer.baml`
— "RANK BY LEVERAGE … the board's ORDER is the ranking"); the pure gates (`src/play/steer-core.ts` —
`readNeverInventGate`/`forkGenuinenessGate`/`leverageRankGate`) are **structural poka-yokes** that
*"cannot judge semantic consequence."* The self-referential signal passes all three (grounded, ranked
keystone, well-ordered). The defect is semantic, so the fix lives in the ranker prompt — the E-020
(honest-empty) precedent.

**The change:**

1. **Recalibrate `baml_src/steer.baml`** (the board rules, ~78-90): add a rule that a demand signal
   must be **concrete product demand** — a buildable feature/change that **decomposes into an epic** —
   and that **self-referential / operational meta-tasks** (run Vend on itself, "run the sweep", settle
   a prior run, dogfood the loop) are **NOT** demand signals: demote them beneath concrete demand or
   exclude them (they are process notes, not demand). Phrase as steering (like the existing RANK/GROUND
   rules + the honest-empty steering), not a new output field.
2. **Recalibrate `baml_src/survey.baml`** the same way (it ranks the same board shape — keep the two
   rankers consistent).
3. `baml:gen` must stay green; `ClaudeStub` + every other function unchanged.

**Do NOT** add a keyword/pattern gate for "self-referential" — distinguishing *"run `vend work`"*
(meta) from *"improve `vend work`'s budget handling"* (real demand) is a semantic judgment a brittle
pattern gets wrong (false-positives on legit vend-feature signals). The structural gates stay
structural and unchanged.

**Deterministic proof (no live model):** `baml:gen` green with the recalibrated prompt; a contract
assertion (a test, the `render-client`/prompt-contract style) that the steer + survey prompts now
carry the concrete-demand / anti-self-referential steering. **Live confirmation is deferred** to the
next `vend steer`/sweep (it stages a board whose #1 is concrete, not "run the sweep") — name it, don't
claim it here (the E-038→E-039 shape).

**Codebase-memory (use it — project coding-task suggestion):** confirm the ranker prompt + the
structural-gate boundary with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`,
indexed/ready) via `search_code`/`query_graph`, not raw grep:
- `search_code(pattern: "RANK BY LEVERAGE|honest-empty|abstain|signals|leverageRankGate|readNeverInvent", regex: true, mode: "full", path_filter: "^(baml_src|src/play)/")` — the ranker prompt rules to extend + the structural gates to leave unchanged.
Confirmed coordinates at authoring: `steer.baml` board rules ~78-90; `survey.baml` (same shape);
`steer-core.ts` gates `readNeverInventGate`:93 / `forkGenuinenessGate`:120 / `leverageRankGate`:161.

**Cites:** `baml_src/steer.baml` (the ranker prompt — recalibrate); `baml_src/survey.baml` (the same,
recalibrate in step); `src/play/steer-core.ts` (structural gates — unchanged); `docs/active/epic/
E-020.md` (the prompt-only honest-empty recalibration — the precedent + style); `work/T-039-01/
sweep-log.md` + `work/T-037-02/sweep-log.md` (the self-referential-#1 incidents).
