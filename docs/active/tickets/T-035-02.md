## Context

The second, non-Claude executor behind T-035-01's interface — an **OpenAI-compatible / local-model
adapter** (the universal shape open models expose: Ollama, llama.cpp, vLLM all speak
`/v1/chat/completions` with streaming). Four parts:

1. **The executor.** `OpenAICompatExecutor` implementing `Executor` (`src/executor/openai-compat.ts`).
   Config from env — **local-first, no secrets**: `VEND_OPENAI_BASE_URL` (default a local default, e.g.
   `http://localhost:11434/v1` for Ollama), the model id (`VEND_EXECUTOR_MODEL`), optional
   `VEND_OPENAI_API_KEY` (only if set — for endpoints that require it). Build a streaming
   `/v1/chat/completions` request from `opts.prompt` (one user message; `opts.system` → a system
   message if present). The **agentic options are ignored** (documented — a completion is not an agent),
   honoring T-035-01's honor-when-able contract.
2. **Stream adaptation.** Consume the SSE response (`data: {…}\n\n` frames, terminated by `data:
   [DONE]`), and for each delta adapt it to a `StreamMessage` handed to `opts.onMessage` **in order**
   (mirror Claude's streaming so the live stdout + transcript surfaces are identical downstream). Be
   total on a blank/`[DONE]`/non-JSON frame (skip), the `parseStreamJsonLine` discipline.
3. **The metered result (IA-8).** Synthesize the terminal `ResultMessage` the pipeline consumes: map the
   API's `usage` (`prompt_tokens`/`completion_tokens`/`total_tokens`) into the same `Usage` shape the
   run-log/recalibration read (so tokens are **truth from the API**, not a guess); stamp the real model
   id; set a subtype. Wall-clock is the transport-universal denomination (the timeout latch from the
   interface). A genuinely absent result (stream produced nothing) throws, like Claude's.
4. **Selection + register.** Wire it into `executorFor` (T-035-01) under `VEND_EXECUTOR=openai-compat`;
   Claude stays the default.

**Honest, named, deferred (do NOT over-claim):** this is a **single-completion** adapter — it proves the
interface + transport + token metering are pluggable and works for **context-complete** plays (prompt
carries its context). It is **not an agentic runner**; **read-the-repo/run-tools** plays
(survey/steer/work) need the downstream agentic open-model runner. The **live smoke** against a real
endpoint is the one deferred human step (env + a running Ollama) — the proof here is **fixture-based**.

**Codebase-memory (use it — project coding-task suggestion):** confirm the stream + result shapes with
the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`, indexed/ready):
- `search_code(pattern: "StreamMessage|ResultMessage|makeStreamConsumer|parseStreamJsonLine|Usage|normalizeUsage", regex: true, mode: "full", path_filter: "^src/(executor|log)/")` — the exact `StreamMessage`/`ResultMessage`/`Usage` shapes to produce + the usage normalizer to reuse.
Confirmed coordinates at authoring: `ResultMessage` `src/executor/claude.ts:34`; `parseStreamJsonLine`
in `claude.ts`; `Usage`/`normalizeUsage` in `src/log/run-log.ts`; `executorFor` (T-035-01).

**Cites:** T-035-01 (`Executor`/`executorFor`/`ExecutorTimeoutError`); `src/executor/claude.ts`
(`ResultMessage`/`StreamMessage`/`parseStreamJsonLine` — the shapes to mirror); `src/log/run-log.ts`
(`Usage`/`normalizeUsage` — map the API usage into the metered shape); IA-8 (the two-denomination meter);
`docs/knowledge/stack.md` (the open-model intent).

## Acceptance Criteria

- [ ] `OpenAICompatExecutor` implements `Executor`: builds a streaming `/v1/chat/completions` request
      (env base-url/model, local-first, no committed secret), consumes the SSE stream → `StreamMessage`s
      to `onMessage` in order, and synthesizes a `ResultMessage` with **API-mapped token usage** (IA-8)
      + the real model id. Agentic options ignored (documented).
- [ ] Selectable via `VEND_EXECUTOR=openai-compat` through `executorFor`; **Claude stays the default**.
- [ ] **Unit-tested against a recorded SSE fixture** (free, deterministic — no live model): the fixture
      stream yields the expected ordered `StreamMessage`s and a `ResultMessage` whose `Usage` matches the
      fixture's `usage`; a `[DONE]`/blank/garbage frame is tolerated; an empty stream throws.
- [ ] The **non-agentic gap + the deferred live smoke** are recorded in the work artifact (not
      over-claimed as open-model parity). `bun run check:*` green.
