## Context

**`runGraph` ‚Äî the pure executor core, `runChain` generalized to a graph.** `runChain`
(`chain-core.ts:90`) walks a `ChainStep[]` SEQUENTIALLY, threading one `upstream: string | undefined`
into each step and halting on the first non-success (`decideThread`). This ticket generalizes that to
a DAG using T-046-01's model + `topoSort` ‚Äî keeping it **PURE given injected cast thunks** (it spawns
nothing; fakes prove it, the `chain-core.test.ts` discipline).

**The generalization (the three things the graph changes):**

1. **Multi-upstream threading (the JOIN).** `runChain`'s node receives a single `upstream` string; a
   graph node receives the `produced` of **all its upstream nodes** (its in-edges). Generalize the cast
   signature to receive a collection ‚Äî e.g. `(upstreams: ReadonlyMap<NodeId, string>)` keyed by the
   from-node (a source node gets an empty map; a linear node a 1-entry map; a **join** a multi-entry
   map). This is the load-bearing change ‚Äî the linear engine structurally cannot deliver two upstreams
   to one node.
2. **Fan-out.** A node's `produced` feeds **every** out-edge ‚Äî its downstream nodes each receive it
   (keyed by this node's id) in their upstream map. Topological order (T-046-01) guarantees a node runs
   only after all its upstreams.
3. **Halt the DEPENDENT subgraph.** Reuse `decideThread`'s per-edge discipline (success **and** a
   `produced` to thread ‚Üí proceed). When a node fails/produces-nothing, skip exactly the nodes that
   **transitively depend on it** (its downstream closure) ‚Äî **siblings independent of it still run**
   (the honest minimal generalization; elaborate cross-branch error semantics are OUT, per E-046). A
   skipped node is recorded as skipped, not cast.

**`GraphResult`** (the `ChainResult` analog): per-node `RunSummary`s (cast nodes only; skipped nodes
noted), a terminal outcome, `halted` + which subgraph was skipped + reason, and the **sink** node(s)'
`produced` (the leaves ‚Äî a graph may have several). Keep purity: type-only imports; the cast thunks are
injected (T-046-03 injects `adapt ‚Üí castPlay`; the test injects fakes returning canned summaries).

**Note on parallelism:** the pure core defines the topological READY-SET (which nodes' upstreams are
all done) ‚Äî actual concurrent EXECUTION of independent ready nodes is the impure shell's job
(T-046-03), exactly as `runChain` is pure and `castChain` spawns. Keep `runGraph` deterministic; if it
awaits per node, that's fine ‚Äî the shell owns real parallelism.
