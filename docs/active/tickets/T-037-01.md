## Context

**Prove the metered run WILL clear — deterministically, before any live spend.** The E-024 sweep
burned a real spend and **cleared 0** because authorization (recalibrated price) ≠ execution (static
default); E-025 fixed it. This ticket confirms that fix still holds and the bounded run is viable, so
T-037-02's live cast isn't wasted. **No live model — pure/deterministic inspection + the existing
tests.** Four checks:

1. **The wallet prices the chain from the LIVE ledger.** Confirm `castWork`'s price path
   (`work.ts:177-185`): `recalibrate(proposeEpicPlay.name, records, "standard", prior).envelope` +
   `recalibrate(decomposeEpicPlay.name, …).envelope`, summed (`sumBudgets`). Record the **predicted
   price** (the two per-step envelopes + their sum) from the current `.vend/runs.jsonl` — the same
   numbers `vend shelf`/`vend envelope` surface (decompose ≈227k, propose ≈227k at last read).
2. **A bounded budget affords ≥1 chain.** Using `canAfford`/`fitNext` (`spend-core.ts:93-102`),
   confirm the **recommended bounded budget (~1h / ~1M tokens)** affords ≥1 chain (price ≤ funded) and
   ~2 before exhaustion — so the run demonstrates spend-down, not a single cast. Note how many chains
   the bound affords at the predicted price.
3. **`--no-intervened` threads the forward-E1 bit.** Confirm `intervened:false` flows
   `WorkOptions.intervened` → `castProposeDecomposeChain({…, intervened})` (`work.ts:198-200`) → the
   chain's records, so each cleared record carries the bit `auditWalkAway` reads as **forward (live)**
   (not attested back-fill). Verify via the pure path / existing tests, not a live cast.
4. **The freshness math passes for a run-time board.** Confirm `isBoardStale(boardMtime, liveMtime)`
   (`work-core.ts`) returns false when the board is staged *after* the newest `docs/active/**` change
   — i.e. a board staged at run time (T-037-02) clears the E-027 gate. (DEFAULT_BOARDS were removed,
   so the run stages fresh.)
