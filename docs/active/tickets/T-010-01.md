## Context

The gate logic (the Central Rule): a `bun run check:head` that builds the **committed
HEAD in isolation from the working tree** and reports pass/fail — catching an
inconsistent-but-committed commit (E-007's `cast.ts` without its `play.ts`) that the
in-place `check` masks, because the dirty working tree still holds the uncommitted dep.

**Design decision (settled per E-010's flagged boundary):** use a **git worktree /
temp checkout of HEAD, NOT Docker** — a single-machine offline tool shouldn't require
Docker for this (charter **P5 local-first**; the signal's literal "container" is left
optional). Mechanism: `git worktree add --detach <tmp> HEAD` → in `<tmp>`: `bun install`
+ `bun run check` (`baml:gen → check:typecheck → check:test`) → capture exit →
`git worktree remove <tmp>` **in all paths** (success / fail / error). A pure core
classifies the result; the worktree+build is the thin impure verb.

**Cites:** `epic/E-010.md` (the design boundary it flagged); `src/ci/check-committed.ts`
(E-008 — the sibling gate pattern); `package.json` (`check`); `ci-strategy.md` (Central
Rule — logic in the script).

## Acceptance Criteria

- [ ] `package.json` `check:head` builds HEAD in an isolated git worktree/temp checkout,
      runs the app `check`, exits **non-zero iff** that clean-HEAD build fails, and
      **removes the worktree in every path** (no leak on failure).
- [ ] **No Docker dependency** — runs offline (P5).
- [ ] The pure result-classifier is unit-tested; an integration test with a **synthetic
      broken HEAD** (a committed source file importing a missing module) shows `check:head`
      **fails**, and a clean HEAD **passes** — reproducing E-007's defect class.
- [ ] In-place `bun run check:*` stay green; `check:committed` (E-008) is untouched.
