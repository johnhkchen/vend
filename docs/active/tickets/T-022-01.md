## Context

Extend E-019's consistency probe with a **semantic-equivalence judge** (R12 — over the same
harness): line-set Jaccard tells us outputs *differ*, not whether the difference *matters*.
For a play's N divergent outputs on a fixed input, judge whether they are **equivalent-
diversity** (the same intent / proposed work, worded differently — acceptable) or **genuine-
disagreement** (they propose *different* things — a real inconsistency). The judge is a cast
(reuse the dispense seam) over the N collected outputs; classify per play (expand/survey/steer).

Split per the house pattern: a **pure aggregation core** (given per-pair / per-set equivalence
verdicts → an `equivalent-diversity | genuine-disagreement | mixed` classification + a score),
**unit-tested**; the judge cast + sweep is the impure harness (not unit-tested, like
`run-consistency-probe.ts`).

**Cites:** `src/probe/run-consistency-probe.ts` + `src/probe/consistency.ts` (the harness to
extend); `src/probe/variance.ts` (the dispersion the judge sits beside); `src/engine/cast.ts`
(the dispense seam for the judge); E-019 findings (the dispersion data); a disposable temp
ledger (no real-log pollution — the probe discipline).

## Acceptance Criteria

- [ ] A **pure** core that, given equivalence verdicts over a play's N outputs, classifies the
      dispersion as **equivalent-diversity / genuine-disagreement / mixed** + a summary score;
      **unit-tested** on fixtures (all-equivalent → diversity; all-different → disagreement; a
      mix → mixed).
- [ ] A judge harness casts the equivalence judgment over the N collected outputs (reusing the
      consistency probe's collection + temp-ledger seeding), printing the classification beside
      the existing dispersion number.
- [ ] `bun run check:*` green; the existing consistency-probe path is unaffected (extend, don't
      break).
