## Context

The pure core of `SteerProject-lite` (R1 pure-core-first, R3 foundation), mirroring
`survey-core.ts` and adding the **forks**. Define the **`Fork`** type (question · 2–4
options · why-it-matters · recommendation), the **`Steer`** output (a ranked `Signal[]`
board — **reuse E-016's `Signal`** — **plus** a `Fork[]`), and the `SteerProject` BAML
function (`b.request.SteerProject(project, charter)` renders; `b.parse.SteerProject(text)`
SAP-parses to the `Steer`; authoring-only via a `steer-bridge`). The **pure gates**:

- **Read-never-invent (PE-1):** every board signal traces to real project state (reuse the
  survey-core pattern).
- **Fork-genuineness (the new signature gate):** a `Fork` must be a *real* trade-off the human
  must make; a fake/inconsequential choice is refused, and an empty `Fork[]` is valid (a clear
  path surfaces no forks — never manufacture a decision to seem useful). The fork-side sibling
  of honest-empty.
- **Leverage-rank:** the board is ordered highest-leverage first.

**Cites:** `src/play/survey-core.ts` (the board + read-never-invent/honest-empty pattern to
mirror); `src/play/expand-core.ts` (the `Signal` shape to reuse); `baml_src/` +
`src/baml/survey-bridge.ts` (the BAML authoring + bridge to mirror); `playbooks/project-
steering.md` (the steer moves); `clearing-dynamics.md` (author + assent); the AskUserQuestion
forks raised by hand this session (real-fork exemplars).
