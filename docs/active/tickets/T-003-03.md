## Context

The pure parser for the press. A tiny selection mini-language: `1,2,4-6` → the
picked indices. Independent of the menu (takes the menu length as a parameter), so
it **runs in parallel** with the menu work (rule R5). Fully testable, no I/O.

**Cites:** `epic/E-003.md` (the mini-language spec: comma-separated, `a-b`
inclusive range, 1-indexed, deduped, whitespace-tolerant; invalid or out-of-range
index is a hard error, never a guess).

## Acceptance Criteria

- [ ] `src/shelf/select.ts` exports
      `parseSelection(s: string, menuLength: number) -> number[]` — comma-separated,
      `a-b` inclusive, 1-indexed, deduped, sorted, whitespace-tolerant.
- [ ] `0`, out-of-range, reversed range, and non-integer fields throw a typed
      `SelectionError` (the boundary surfaces it) — no silent coercion.
- [ ] Fully unit-tested (`1,2,4-6` → `[1,2,4,5,6]`; dedup; the error cases).
- [ ] No dependency on the menu/CLI modules. `bun run check:test` /
      `check:typecheck` green. Advances **P2**.
