## Context

The macro-wallet's foundation (P7 — the budget is a hard contract that *depletes*). A pure `Wallet`
over the **two denominations the meter must never conflate** (IA-8): **wall-clock ms** (a hard wall —
halts mid-flight) and **tokens** (a burn that depletes, detect-after). Build it as a pure value with
pure operations — the house pattern (mirrors `budget.ts` / `recalibrate.ts`):

- `allocate(macro: Budget) → Wallet` — fund the wallet once with the feature-block budget.
- `canAfford(wallet, predictedEnvelope: Budget) → boolean` — does the next cast's **measured price**
  (E-013's predicted envelope, passed in) fit in what remains? Honest per denomination: a cast that
  fits on tokens but not wall-clock does **not** fit.
- `debit(wallet, actual: Usage | Budget) → Wallet` — subtract a cleared cast's **actual** cost,
  returning a new wallet (immutable; a spent wallet is spent — never goes negative-silently, floors
  at zero with the overshoot surfaced for the detect-after token case, IA-8).
- `remaining(wallet) → Budget` + a `formatWallet` readout (both denominations, never one bar).

No I/O, no casting — just the depleting-budget algebra. The loop (T-024-02) and the gesture
(T-024-03) compose it.

**Cites:** `src/budget/budget.ts` (`Budget`/`Usage`/`countTokens`/`check` — reuse the per-cast
envelope types and the token count; do **not** duplicate them); `information-architecture.md` **IA-8**
(the two denominations — wall-clock wall vs token detect-after — the wallet must honor); E-013
`recalibrate.ts` (the source of the *predicted* envelope `canAfford` is checked against — consumed,
not built here); the pure-core/unit-tested house split (`budget.test.ts` as the shape to mirror).

## Acceptance Criteria

- [ ] A **pure** `Wallet` + `allocate` / `canAfford` / `debit` / `remaining` / `formatWallet`, over
      both denominations, immutable (debit returns a new wallet), flooring at zero with the token
      overshoot honestly surfaced (IA-8 detect-after) and the wall-clock treated as a hard wall.
- [ ] **Unit-tested** on fixtures: fresh wallet affords a fitting cast and refuses an over-cost one
      (per denomination — including the fits-on-tokens-not-time case); a sequence of debits depletes
      monotonically to zero; an actual that overshoots the remaining tokens floors and reports the
      overshoot; `remaining` + `formatWallet` are truthful at each step.
- [ ] `bun run check:*` green. No I/O in the core (purity preserved).
