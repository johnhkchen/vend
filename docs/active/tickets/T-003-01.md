## Context

The pure, addon-free core of the shelf: rank salient actions by **leverage** (not
effort) and render them as a numbered menu. "Salient" = ready *and* high-leverage;
deterministic — **no LLM** (E-003's precompute fork ships deterministic first).
Define the `MenuCache` shape the menu persists to (`.vend/menu.json`), since
`vend <sel>` must later resolve indices against *the same list shown*.

**Cites:** `epic/E-003.md` (the grammar; the deterministic-vs-LLM fork; the
`.vend/menu.json` seam); `demand.md` (the value-ranking the menu renders —
Keystone > High > Standard > Leaf).

## Acceptance Criteria

- [ ] `src/shelf/menu.ts` exports, all **pure** (no fs / network / LLM):
      `rankActions(actions: Action[]) -> Action[]` (leverage tier, then readiness;
      stable order); `renderMenu(actions, opts?) -> string` (numbered, each row =
      value tier + budget + state; `opts.all` reveals blocked/leaf, else hidden);
      and the `MenuCache` / `Action` types (the persisted shape).
- [ ] Fully unit-tested with fixtures (ranking order, hidden-row behavior, render).
- [ ] No dependency on gather/CLI — composed by `T-003-02`.
