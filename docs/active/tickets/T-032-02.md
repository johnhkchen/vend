## Context

Bind the declarations to a **project** registry and wire them through the cast — with a true andon when
a required capability is absent, and `decompose-epic` as the proof play. Four parts:

1. **The project MCP registry (`.mcp.json`).** Establish a committed project `.mcp.json` at the repo
   root (Claude Code's native convention — the same file `claude -p --mcp-config` reads) defining the
   project's servers, **including `codebase-memory-mcp`**. Today this server is global in
   `~/.claude.json`; mirror its server spec into the project file so the binding is **project-based,
   not global**. Keep the committed file **portable** — no secrets / absolute machine paths; use
   env-var/`${VAR}` indirection if the server command needs a local path. A small reader returns the
   set of server ids the project provides (the `available` set T-032-01's `resolveTools` consumes).
2. **Cast-time resolution.** In the cast path (`src/engine/cast.ts` `castPlay` ~111, beside
   `resolveMaxTurns`), read the project registry, call `resolveTools(play.tools, available)` (T-032-01),
   and thread the result into `dispense`/`buildArgs`:
   - **resolved** → pass `mcpConfig` (the `.mcp.json` path) + `allowedTools` + `strict` so **only** the
     declared servers are wired in (global firehose closed for this cast).
   - **passthrough** (undeclared play) → **no flags, unchanged** (back-compat — inherits global exactly
     as today). This must be byte-identical to current behavior; existing cast tests stay green.
3. **The missing-capability andon (IA-9).** `resolveTools` → `{ ok:false, missing }` → halt the cast as
   an **amber missing-capability refusal** *before* dispensing (nothing cast, nothing materialized) —
   reuse the existing andon/gate-failure path (the `--no-gates`/budget-andon precedent in `castPlay`),
   surfaced like the other honest refusals (IA-9 amber, IA-17 — a run missing its declared capability
   is not the same run). A required MCP absent from the project registry is a STOP, not a silent
   blind run.
4. **decompose-epic as the proof play.** Give `decompose-epic` (`src/play/decompose-epic.ts`)
   `tools: { mcp: ["codebase-memory-mcp"], allow: [<the read tools it needs>] }` — the play now declares
   the MCP we wired by hand into the E-031 tickets.

Single writer / no surprises: don't touch `.vend/menu.json` or the press; don't flip strict on for
undeclared plays; don't inject skills (E-032 scope cut).

## Acceptance Criteria

- [ ] A committed, **portable** project `.mcp.json` defines the project's servers incl.
      `codebase-memory-mcp` (no secrets/absolute machine paths in the committed file); a reader yields
      the `available` id set. The binding is project-based, not the global `~/.claude.json`.
- [ ] The cast path resolves each play's `tools` and threads the flags into `dispense`: a **declared**
      play casts with `--mcp-config`/`--allowedTools`/`--strict-mcp-config` for **only** its servers;
      an **undeclared** play's invocation is **byte-identical to today** (existing cast tests green).
- [ ] A play requiring an MCP **absent** from the project registry **andons** (amber missing-capability
      refusal, IA-9) **before** dispensing — nothing cast, nothing materialized; surfaced like the
      other honest refusals.
- [ ] `decompose-epic` declares `tools: { mcp: ["codebase-memory-mcp"], allow: [...] }`.
- [ ] **Live proof (free, no cast):** inspecting the built `claude -p` argv for `decompose-epic` shows
      the scoping flags (mcp-config + allowedTools + strict for `codebase-memory-mcp`); an undeclared
      play's argv is unchanged; a play declaring an absent MCP produces the andon — all via
      `buildArgs`/`resolveTools` inspection, **no live LLM cast**. `bun run check:*` green.
