## Context

Wire the pure core (T-017-01) into a registered, castable play with a gesture — the
headline: `vend survey` reads the rough project and **stages a ranked board** in one
gesture. Mirrors `expand-fragment.ts` / `propose-epic.ts`: assemble project context (via
`project-context.ts`), `castPlay(surveyPlay, …)` through the metered seam, and a **staging
effect that writes the ranked board** (the `Signal[]` rendered in the `demand.md` shape) to
the `docs/active/pm/` staging space — never auto-promoted to the board.

**Budget — heed E-016's finding:** a project-scale read is heavier than expand (which
under-shot at 211k). Pre-fill `surveyPlay.budget` **generously** from the start (a measured
floor, not a cold-start guess); leave a `// recalibrate from the log (E-013)` note.

**Cites:** `src/play/expand-fragment.ts` + `expand-effect.ts` (the register/effect/gesture
pattern to mirror); `src/play/project-context.ts` (`assembleInputs` — reading project state);
`src/engine/play.ts` (registry) + `cast.ts` (`castPlay`); `src/cli.ts` (the gesture);
`docs/active/pm/` (the staging contract).

## Acceptance Criteria

- [ ] `surveyPlay` registered on the engine with a **generously pre-filled budget**;
      `castSurvey({ project, budget, … })` casts it (render → dispense → gates → effect →
      log), like the other plays.
- [ ] A gesture — `vend survey` — casts the play: on success it **stages the ranked board**
      (the `Signal[]` in the `demand.md` shape) for review; a honest-empty / read-never-invent
      refusal halts with a clear andon (no fabricated board materialized).
- [ ] A fixture/canned-reply test proves project → staged board end to end; the effect writes
      to staging, not the live `demand.md`.
- [ ] `bun run check:*` green. (The live cast — `vend survey` on this repo → a staged board —
      is the human verification at sweep.)
