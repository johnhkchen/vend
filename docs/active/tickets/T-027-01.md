## Context

E-025's live re-sweep proved `vend work` spends down **whatever staged board it finds**, regardless of
age: it cleared the #1 signal of a `steer.md` from 11:54 against a project that had moved to 22:30,
minting E-026 on an already-resolved premise (`work/T-025-01/sweep-logs/findings.md`). A wallet spent
on a stale board is overproduction. Add a **board-level freshness gate** before the wallet is funded.

Build it the house way (pure decision + impure gather + a new outcome + CLI render):

1. **Pure** ‚Äî `isBoardStale(boardMtimeMs: number, liveMtimeMs: number): boolean` (board older than the
   live state ‚áí stale; equal/newer ‚áí fresh). Trivial but the unit-tested contract (boundary cases).
2. **Impure** ‚Äî in `castWork`, after `readBoard` and before `allocate`: stat the board's mtime + the
   **newest** mtime across `docs/active/{epic,stories,tickets}` (the decomposed state the board
   ranks). If `isBoardStale` and not `opts.staleOk`, return a new **`stale-board`** `WorkResult`
   variant (`{ kind: "stale-board"; boardPath; boardMtimeMs; liveMtimeMs }`) ‚Äî do **not** fund/cast.
3. **CLI** ‚Äî render `stale-board` as an **andon** (amber, IA-9): "board staged <when>; project changed
   since <when> ‚Äî re-survey before spending," and hand the next move (`vend survey`/`vend steer`, then
   `vend work`). Exit like the other broken-precondition outcomes (no-board/empty-board), not a crash.
   Add a `--stale-ok` flag (parsed in `parseWorkArgs`, threaded to `WorkOptions.staleOk`) to spend
   anyway ‚Äî the human-in-charge override (IA-5).

The staleness signal is mtime-based and local. **Honest caveat in code + the andon copy:** mtime is a
heuristic (git checkout can reset it); `--stale-ok` is the escape hatch. Do **not** auto-re-survey
(out of scope ‚Äî it would spend the wallet on unfunded work).

**Cites:** `src/play/work.ts` (`readBoard` ~80, `castWork` ~101 before `allocate`, the `WorkResult`
union ~66 ‚Äî add `stale-board`); `src/cli.ts` (`parseWorkArgs` + the `work` dispatch arm ‚Äî render +
`--stale-ok`); `src/cli.test.ts` (the `work` parser tests to extend); IA-9 (andon = successful
refusal), IA-5 (the override, not silent auto-survey); `work/T-025-01/sweep-logs/findings.md`.

## Acceptance Criteria

- [ ] **Pure** `isBoardStale(boardMtimeMs, liveMtimeMs)` ‚Äî **unit-tested**: board older ‚áí true, board
      newer ‚áí false, equal ‚áí false (fresh-on-tie). No I/O.
- [ ] `castWork` gathers the board mtime + the newest `docs/active/{epic,stories,tickets}` mtime and,
      when stale and not `--stale-ok`, returns the `stale-board` outcome **before funding the wallet**
      (no cast happens); `--stale-ok` spends anyway.
- [ ] The CLI renders `stale-board` as an **amber andon** with both timestamps + the re-survey next
      move, exiting like no-board/empty-board; `parseWorkArgs` accepts `--stale-ok` (**unit-tested**,
      like the other `work` flags).
- [ ] **Live proof (free, deterministic ‚Äî no LLM cast):** `bun run src/cli.ts work` against the live
      stale `steer.md` **refuses** with the andon and does **not** cast; `--stale-ok` proceeds. `bun
      run check:*` green.
