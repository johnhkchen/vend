## Context

**The typed node/edge graph model + a pure topological sort — the data substrate the DAG executor
walks.** Today the engine's only composition primitive is the LINEAR chain: `runChain` takes a
`ChainStep[]` (an implicit path — each step one upstream, one downstream) and the pure core sees only
the `produced` STRING thread (`src/engine/chain-core.ts`). This ticket generalizes the *shape* from a
list to an explicit typed graph.

**The model (pure types + validation):**

1. **`DagNode`** — the `ChainStep` generalized with a stable identity: `{ id: NodeId; cast: (...) =>
   Promise<RunSummary> }`. (The cast's exact input signature — how it receives upstream `produced`
   refs — is T-046-02's concern; here a node is `{id, cast}`.)
2. **`DagEdge`** — `{ from: NodeId; to: NodeId }`: a directed dependency that carries the upstream
   node's `produced` thread to the downstream node. A node's **in-edges** define its upstreams (≥2 = a
   **join**); a node's **out-edges** its downstreams (≥2 = a **fan-out**).
3. **`DagSpec`** — `{ nodes: DagNode[]; edges: DagEdge[] }`, the graph a playbook declares.
4. **Pure validation** (`validateDag` or similar, TOTAL — returns offenses, never throws): every edge
   references DECLARED node ids; **no duplicate node ids**; the graph is **ACYCLIC**. Per the epic,
   cycles are OUT of scope to *run* — but they must be **DETECTED and refused** (a loud andon), never
   hang.
5. **`topoSort(spec) → { order: NodeId[] } | { cycle: NodeId[] }`** — pure, total, DETERMINISTIC
   (stable tie-break — e.g. declaration order — so a fan-out's ready siblings have a reproducible
   order). Kahn's or DFS; a cycle returns the offending nodes, not a partial order.

**Purity (the `chain-core.ts` discipline):** type-only imports (`RunSummary` from `cast.ts`, erased);
no fs/clock/network/addon. The model owns SHAPE + ORDERING only — it spawns nothing, casts nothing
(T-046-02 owns the run). Fully unit-tested: a linear path, a fan-out (1→2), a join (2→1), a diamond
(fan-out then join), a disconnected graph, a cycle (rejected), a dangling edge (rejected), the empty
graph (vacuous).

**Codebase-memory (use it — project coding-task suggestion):** confirm the chain primitive to
generalize with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`, indexed/ready) via
`search_code`/`query_graph`, not raw grep:
- `search_code(pattern: "ChainStep|runChain|decideThread|RunSummary|produced", regex: true, mode: "full", path_filter: "^src/engine/")` — the linear chain (`ChainStep`/`runChain`) this graph model generalizes + the `RunSummary.produced` thread the edges carry.
Confirmed coordinates at authoring: `ChainStep` `src/engine/chain-core.ts:29`; `runChain` `:90`;
`RunSummary` (`outcome`/`produced`) `src/engine/cast.ts:101`.

**Cites:** `src/engine/chain-core.ts` (`ChainStep`/`runChain`/`decideThread` — the linear primitive +
