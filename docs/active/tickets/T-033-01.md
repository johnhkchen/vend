## Context

The pure policy of the per-commit gate — the decision, no I/O, no spawning (T-033-02 does the invoking).
Mirror the E-008 split: the hook will *invoke*, this core *decides* (the `ci-strategy.md` central rule;
`src/ci/committed-core.ts` is the precedent). Create `src/ci/precommit-core.ts` with two pure functions:

1. **`classifyPrecommit(runResult)` — the block/allow decision.** Given the outcome of running the test
   gate — model it as `{ ran: boolean; exitCode: number | null; stderr?: string }` (or an equivalent
   tagged shape) — return `{ block: boolean; reason: "green" | "tests-failed" | "could-not-run";
   message: string }`. The **load-bearing nuance**:
   - **ran && exitCode === 0** → `{ block:false, reason:"green" }`.
   - **ran && exitCode !== 0** → `{ block:true, reason:"tests-failed", message }` — **fail-closed**, the
     gate's whole job; the message names the failure (the E-008 "name the file" style — surface the
     failing-test summary so the model sees the andon and fixes before committing).
   - **!ran** (no bun, not a repo, spawn/env error) → `{ block:false, reason:"could-not-run", message }`
     — **fail-open** (mirrors on-stop: a broken checker must never wedge the loop), with a dim note so
     the skip is visible, not silent.
2. **`hookInstallState(hooksPath)` — the guard for T-033-02's `check:hooks`.** Given the configured
   `core.hooksPath` value (string | null/undefined), return `{ active: boolean; message: string }` —
   `active` iff it points at the committed hooks dir (e.g. `.githooks`); otherwise a message telling the
   reader to run `bun run hooks:install`. So the gate can't be silently absent (E-012 spirit).

Pure/total: no `Bun.spawn`, no `readFile`, no git calls — the impure runner in T-033-02 supplies the
inputs (the test-run result, the configured hooks path). Keep both functions exhaustively switched (no
`default` branch on `reason`) so `tsc` proves every case is handled.

**Codebase-memory (use it — project coding-task suggestion):** confirm the invoker/pure-core precedent
with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`, indexed/ready) rather than
re-deriving by raw grep:
- `search_code(pattern: "committed-core|classifyCommitted|isUncommitted|head-build-core", regex: true, mode: "full", path_filter: "^src/ci/")` — the pure-core + `*-core.test.ts` shape to mirror.
Confirmed coordinates at authoring: `src/ci/committed-core.ts` (the pure core), `committed-core.test.ts`
(the test precedent), `src/ci/check-committed.ts` (the thin invoker), `.lisa/hooks/on-stop.sh` (the
fail-open invoker comment).

**Cites:** `src/ci/committed-core.ts` + `committed-core.test.ts` (the pure-core/test precedent);
`src/ci/check-committed.ts` (the invoker that consumes the core); `.lisa/hooks/on-stop.sh` (the
fail-open discipline to mirror); `docs/knowledge/ci-strategy.md` (hook-only-invokes rule);
`docs/active/epic/E-008.md` (the commit-gate lineage).
