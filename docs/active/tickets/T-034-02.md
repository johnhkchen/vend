## Context

The impure sweep that turns T-034-01's policy into a runnable audit. Create `src/ci/check-history.ts`
(the thin invoker, the `check-head.ts` precedent) + a `check:history` script. Three parts:

1. **Resolve the range (bounded).** Parse an optional range arg (`check:history [<range>]`, default a
   sensible bound — e.g. `main..HEAD` or last-N when none given), resolve it via `git rev-list
   <range>` to an ordered sha list, and pass it through `boundRange` (T-034-01) so the swept set is
   bounded and whatever is dropped is stated loudly.
2. **Build + test each commit in isolation.** **Generalize `buildCommittedHead`** (`src/ci/head-build-core.ts`
   ~53 — today it hardcodes `HEAD` in the `worktree add` and a `vend-head-` temp prefix) to accept an
   arbitrary **commit-ish**, so the sweep can build any sha in a fresh, isolated worktree (cleaned up in
   ALL paths via the existing `finally` — no working-tree disturbance, no leak). Run a command that
   **includes codegen** so a fresh checkout is valid — the full `check` (its default) or
   `baml:gen && check:test`; map each `BuildOutcome` (via `classifyBuildOutcome`) to a `history-core`
   result `{ sha, subject, green, summary }` (subject from `git log -1 --format=%s <sha>`).
3. **Report + exit.** Feed the results to `classifyHistory` (T-034-01), print the report, and **exit
   non-zero iff `anyRed`** (so it fails CI on a red commit in range). Honest-empty range → clean exit
   with the "no commits in range" line.

Keep the invoker thin (policy is in T-034-01); reuse the worktree machinery (don't reimplement it);
sequential is fine (parallelism is a non-goal this slice). Cost note: each commit ≈ worktree + install +
build — this is a periodic/CI audit, not the hot path.

**Codebase-memory (use it — project coding-task suggestion):** before generalizing the worktree builder,
confirm its call sites + shape with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`,
indexed/ready):
- `search_code(pattern: "buildCommittedHead|classifyBuildOutcome|HeadBuildOptions|worktree add", regex: true, mode: "full", path_filter: "^src/ci/")` — the function to generalize + its options type + the `HEAD`-hardcode to parameterize.
- `query_graph("MATCH (f)-[:CALLS]->(g) WHERE g.name = 'buildCommittedHead' RETURN f.qualified_name")` — every caller, so widening the signature (commit-ish param, default `HEAD`) doesn't break `check:head`.
Confirmed coordinates at authoring: `buildCommittedHead` `src/ci/check-head.ts:53` (re-exported from
`head-build-core`); `classifyBuildOutcome`/`BuildOutcome` in `head-build-core.ts`; `check-committed.ts`
(the thin-invoker precedent).

**Cites:** T-034-01 (`classifyHistory` + `boundRange`); `src/ci/head-build-core.ts` /
