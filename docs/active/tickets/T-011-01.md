## Context

The engine's first composition primitive (R12 shared contract): a `castChain` that runs
a sequence of plays, **threading each play's produced output into the next's input**.
The minimal enabler: a play's effect must *surface what it produced* — ProposeEpic
writes `E-0XX.md`, but its `RunSummary` today carries only `{runId, outcome,
materialized}`; extend the effect result + summary to expose a **`produced`** reference
(e.g. the minted epic path) so the next play can consume it. On any gate STOP, the chain
**halts** (no downstream cast).

**Cites:** `src/engine/cast.ts` (`castPlay` + `RunSummary` — extend to surface
`produced`); `src/engine/play.ts` (the `Play.effect` return); `src/play/note.ts` +
`src/play/propose-epic.ts` (effects that write artifacts).

## Acceptance Criteria

- [ ] `src/engine/chain.ts`: `castChain(steps)` runs plays in sequence via `castPlay`,
      threading each step's `produced` output → the next via an adapter; **halts on any
      STOP** (no downstream cast); appends one run-log record per step.
- [ ] A play's `effect` result + `RunSummary` surface a **`produced`** reference (the
      artifact it wrote — e.g. the epic path). Minimal, backward-compatible extension.
- [ ] The threading + halt logic is **pure and unit-tested**: a two-step fixture (step-1
      `produced` → step-2 input); a step-1 STOP → step-2 never runs.
- [ ] `bun run check:*` green; existing single-play casts unaffected.
