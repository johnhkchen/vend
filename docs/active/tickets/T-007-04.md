## Context

Prove the engine is *genuinely* play-agnostic by registering a **second,
non-DecomposeEpic play** and casting it through the same `castPlay` — a different
BAML function, different gates, a different effect. Keep it **minimal but real**: a
small **sorcery** (one-shot) whose effect writes a real artifact, not a toy echo.
This is E-007's keystone done-signal: **≥2 plays cast through one engine.**

*(Full real plays — `ProposeEpic`, and the survey-as-cast that fully resolves
E-006's F4 — are follow-up registrations onto the now-proven engine, not this slice.)*

**Cites:** `card-model.md` (Sorcery vs Permanent); `playbooks/propose-epic.md` /
`playbook-decompose-epic.md` (a play's shape); `src/engine/play.ts` (the registry);
`baml_src/` (where the second play's BAML function lives).

## Acceptance Criteria

- [ ] A second play is authored and registered — its own BAML function (render +
      SAP parse), its own gate(s), and a **real effect** (e.g. writes a markdown
      artifact). Minimal, but not a stub.
- [ ] **Both** `DecomposeEpic` and the sorcery cast through the **same** `castPlay`,
      each appending its own countable run-log record — the loop has zero
      per-play branches.
- [ ] A test/cast demonstrates the second play producing its artifact under its
      budget and passing its gate (and stopping the line when its gate fails).
- [ ] `bun run check:test` / `check:typecheck` green.
