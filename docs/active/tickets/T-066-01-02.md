## Context

`castPlay` runs render → dispense → parse → gates → effect, so the gate is where a shell story must
die — before any file is written. Add a story-completeness gate to `decompose-epic`'s gate list:
every parsed story must carry non-empty scope, acceptance, honest boundary, wave rationale, and
out-of-slice. Refusal is a STOP with a named andon (`story-incomplete`, naming the story id and the
missing sections), logged like every other honest refusal — never a warning, never a pad-to-pass.
Pure-core tested with fabricated parses: today's ten-line shell is the failing fixture, an
S-066-01-shaped story the passing one.

_Advances: P3 (gates are the contract — an ungated output degrades to the minimum accepted; this
gate is the floor)._

## Acceptance Criteria

- [ ] Gate unit tests: a shell-shaped parse STOPs with an andon naming the story id + missing
      sections; a contract-shaped parse passes; `gateResults` records the verdict either way.
- [ ] A cast-level test through a stub executor proves a dispensed shell never reaches the effect —
      no story file exists after the refused cast.
