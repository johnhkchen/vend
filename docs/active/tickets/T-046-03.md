## Context

**The impure `castGraph` shell + parallel branches + the executable example that proves the
substrate.** `runGraph` (T-046-02) is the pure executor; this is its `chain.ts` analog — the shell
that wires real casts and runs independent branches concurrently — plus the end-to-end demonstration
E-046's "Done looks like" requires.

Three parts:

1. **`castGraph` (impure shell, the `chain.ts` `castChain` analog).** For each `DagNode`, inject the
   play-specific adapter `adapt → castPlay` (upstream `produced` map → the node's typed inputs →
   `castPlay`), exactly as `castChain` injects `adapt → castPlay` for the linear case. Run **independent
   ready nodes CONCURRENTLY** — the "run plays in parallel" the card names: each topological ready-set
   (nodes whose upstreams are all done, from T-046-02's core) is dispatched together (e.g.
   `Promise.all`), the next ready-set after they settle. The pure core owns ordering/threading/halt;
   the shell owns spawning + concurrency (the `cast` ⊥ `cast-core` purity seam). Value-imports
   `castPlay` (the executor/addon), so — like `chain.ts` — **no `bun test` value-imports this module**.
2. **The worked example — DETERMINISTIC, stub nodes.** A declared `DagSpec` with **≥1 fan-out and ≥1
   join** (e.g. a diamond: A → {B, C} → D), run end-to-end. Use **STUB nodes** (canned `RunSummary`s,
   the `chain-core.test.ts` fake discipline) — **NOT a live cast** (a real-play graph is a downstream
   metered proof; this proves the SUBSTRATE deterministically). Assert: all four nodes run, B and C ran
   as parallel branches off A's `produced`, and **D (the join) received BOTH B's and C's `produced`**.
3. **The AC test that FAILS against the old linear engine.** Per "Done looks like": a test the
   linear-chain-only engine can't pass. The join is the proof — `runChain` threads a SINGLE upstream,
   so it structurally cannot deliver D both B's and C's `produced`. Assert `runGraph` converges the
   join (D sees 2 upstreams) where `runChain`/`ChainStep[]` cannot express it (e.g. show the same
   4-node graph has no faithful `ChainStep[]` linearization that feeds D both — or that `runChain` over
   any linearization drops one upstream). This is the "genuinely non-linear" evidence.

**Scope (E-046):** fan-out + join over the typed substrate; **OUT** — conditional edges, cross-branch
budget accounting, a live real-play graph cast (deterministic stubs here), the open-model runner.
Respects N4 (the graph orchestrates Vend's own clearing plays; lisa still executes).
