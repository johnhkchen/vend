## Context

Declare the open-model BAML client and show request-shaping can target it. Two parts:

1. **The client (`OpenModelStub`).** Add `client<llm> OpenModelStub` to `baml_src/clients.baml`,
   **provider `openai-generic`** (the universal OpenAI-compatible provider — Ollama/llama.cpp/vLLM; the
   same shape E-035's `OpenAICompatExecutor` targets). Options: `base_url env.VEND_OPENAI_BASE_URL`
   (a local default if BAML allows, else env), `model env.VEND_EXECUTOR_MODEL` (the E-035 env names),
   an optional/dummy `api_key` — **render-only, never sent**, mirroring `ClaudeStub`'s comment + guard
   (the client exists only to shape `.request`; the dispense rides E-035's `Executor`, not BAML). Keep
   the "never actually CALL this client" discipline. `baml:gen` must stay green with it declared.
2. **Render targets it.** Extend the render path (the `decompose-bridge.ts` pattern — `b.request.*`) so
   a render op can target a **selectable client** (BAML's runtime client selection — `ClientRegistry`
   or a `b.request` client override; **verify the exact API in the generated `baml_client/`** — this is
   the one external contract, confirm it, don't assume). Render a function against `OpenModelStub` and
   show the built request is in the **openai-generic provider format** — assert on the **request shape**
   (the body's messages/model/endpoint differ from `ClaudeStub`'s anthropic format), NOT on the prompt
   text (which may be identical — do not contrive a text difference). Run through the existing
   **subprocess bridge** (the BAML one-call-per-process limit), the `decompose-bridge.ts` precedent.

Do **not** dispatch to a live model (render-only); do **not** make BAML the transport; do **not** touch
the SAP-parse logic (T-036-02 proves it's provider-agnostic). Keep `ClaudeStub` + every existing
function's default client unchanged (back-compat — the default render is byte-identical).

**Codebase-memory (use it — project coding-task suggestion):** confirm the BAML render bridge + client
pattern with the **codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`, indexed/ready) rather
than re-deriving by raw grep:
- `search_code(pattern: "b.request|extractPromptText|runOp|ClientRegistry|ClaudeStub", regex: true, mode: "full", path_filter: "^src/baml/")` — the render-op shape + how the request body is reached, to add client selection.
Confirmed coordinates at authoring: `ClaudeStub` `baml_src/clients.baml:8`; `extractPromptText`/`runOp`
`src/baml/decompose-bridge.ts:35`/`:47`; `generators.baml` (codegen); the generated `baml_client/`
(verify the runtime client-selection API).

**Cites:** `baml_src/clients.baml` (`ClaudeStub` ~8 — the render-only client to mirror); `baml_src/generators.baml`
(codegen); `src/baml/decompose-bridge.ts` (`b.request.*`/`extractPromptText`/`runOp` — the render bridge
to extend; the subprocess pattern); the generated `baml_client/` (the client-selection API to verify);
`src/executor/openai-compat.ts` (E-035 — the provider shape this mirrors at authoring).
