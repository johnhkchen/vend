## Context

`src/play/decompose-epic.ts`) while the **real** id — `claude-opus-4-8[1m]` —
rides on the dispense stream (the proof read it off every transcript;
`work/T-002-04/proof.md` kaizen #3, `results/summary.json`). Capture the real id in
the executor and stamp it in the run log so the consistency layer reads truth
(`steering-data-model.md`).

**Depends on `T-004-02`** — both edit `src/play/decompose-epic.ts` (the
`appendRunLog` stamping site); serialized per the file-overlap rule, not because
