## Context

Generalize the decompose variance probe (`run-probe.ts`, currently decompose-only +
gated-vs-ungated) into an **any-play, run-to-run consistency probe** (R12 shared-contract).
The new measurement: cast a named play **N× on a fixed input** and report (a) **output
variance** (reuse `variance.ts`'s line-set-Jaccard dispersion) and (b) the **outcome mix**
(signal / honest-empty / budget-exhausted counts — the honest-empty rate on a *grounded*
input is the over-eagerness signal). Cast into a **disposable temp ledger** (no real-log
pollution — the `run-probe.ts` seeding discipline: `lisa init` + the dirs each play needs).

The pure part — the **variance + outcome-mix computation** over a set of (outcome, output)
results — is unit-tested; the cast-loop harness is the impure sweep verb (not unit-tested,
like `run-probe.ts`).

**Cites:** `src/probe/run-probe.ts` (the harness + temp-seeding to generalize);
`src/probe/variance.ts` (the Jaccard core to reuse); `src/engine/play.ts` (the registry to
resolve a play by name); the recalibrated play budgets (expand 250k / survey 300k / steer 400k).

## Acceptance Criteria

- [ ] A **pure** core computes, over a list of `{outcome, output}` results: the run-to-run
      variance (reusing `variance.ts`) **and** the outcome mix (signal / honest-empty /
      budget-exhausted counts + rates). **Unit-tested** on fixtures (all-same → 0 variance;
      mixed outcomes counted; honest-empty rate computed).
- [ ] An any-play N-cast harness casts a named play N× on a fixed input into a disposable
      ledger/board (no real `.vend/runs.jsonl` or board pollution), collecting the results.
- [ ] `bun run check:*` green; the existing decompose `run-probe.ts` path is unaffected
      (generalize, don't break it).
