## Context

**Make the hackathon template write its tuned charter where steer reads it (the coupled charter gap).** Steer
reads `docs/knowledge/charter.md` (`CHARTER_PATH`, `src/play/project-context.ts:18`). On the hackathon seed
the *rich* charter ships at **root** (`examples/templates/hackathon-seed/charter.md` — the 5 demonstrable-
slice criteria), but `vend init` overlays only a generic `CHARTER_STUB` into `docs/knowledge/charter.md`, and
`TEMPLATE_REGISTRY.hackathon` (`init-core.ts:173`) injects only a `SEED.md` stub. So even after T-059-01
wires the seed's idea, steer grades it against the **wrong value function** (the generic stub). This is the
intended-but-unbuilt T-058-03 enrichment.

- `src/init/init-core.ts`: add a `{ kind: "file", path: "docs/knowledge/charter.md", contents:
  HACKATHON_CHARTER }` entry to `TEMPLATE_REGISTRY.hackathon` (define `HACKATHON_CHARTER` from the seed's
  tuned charter — the demonstrable-slice value function). `mergeManifests` (`init-core.ts:238`) already
  overrides the base slot, so the overlay's charter wins over the base `CHARTER_STUB` BEFORE the disk is
  consulted; no-clobber + idempotency hold (never overwrites a charter the user already edited).
- Keep `SEED.md` in the overlay (T-058-01). Bare `vend init` (no `--template`) stays byte-identical to E-040.
- Honest-empty held: the overlay still adds only structure + charter + SEED, **never demand** — the board
  starts empty. One-way-to-lisa held (vend-owned paths only).

**Cites:** `src/init/init-core.ts` (`TEMPLATE_REGISTRY`, `mergeManifests`, `CHARTER_STUB`,
`HACKATHON_SEED_STUB`); `src/play/project-context.ts:18` (`CHARTER_PATH` — the path steer reads);
`examples/templates/hackathon-seed/charter.md` (the tuned charter source); E-040; `docs/active/epic/E-059.md`.
