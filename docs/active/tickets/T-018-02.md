## Context

Wire the pure core (T-018-01) into a registered, castable play with a gesture — the
headline: `vend steer` reads the project and **stages a steer** (the ranked board **and**
the real forks) in one gesture. Mirrors `survey.ts` / `expand-fragment.ts`: assemble project
context (`project-context.ts`), `castPlay(steerProjectPlay, …)` through the metered seam, and
a **staging effect that writes the steer** — the board rows (`demand.md` shape) **and** the
forks (each: question · options · why · recommendation) — to `docs/active/pm/` staging, never
auto-promoted.

**Budget — heed E-016/E-017:** the heaviest read yet (board + forks). Pre-fill
`steerProjectPlay.budget` **above Survey's 300k** (Survey held; this does more); leave a
`// recalibrate from the log (E-013)` note.

**Cites:** `src/play/survey.ts` + `survey-effect.ts` (the register/effect/gesture pattern to
mirror); `src/play/project-context.ts` (`assembleInputs`); `src/engine/play.ts` (registry) +
`cast.ts` (`castPlay`); `src/cli.ts` (the gesture); `docs/active/pm/` (the staging contract).

## Acceptance Criteria

- [ ] `steerProjectPlay` registered with a **generously pre-filled budget** (> Survey's 300k);
      `castSteer({ project, budget, … })` casts it (render → dispense → gates → effect → log).

      the effect writes to staging, not the live `demand.md`.

      real forks — is the human verification at sweep.)
