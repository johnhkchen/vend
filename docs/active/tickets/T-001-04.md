## Context

Every run is logged in a **countable** shape so the consistency layer is later
just reading data we already kept (E-001 done-looks-like; charter's "you got what
you paid for" must be *demonstrable*, not asserted). Append-only JSONL, one record
per run. A separate module → parallel with T-001-02 / T-001-03.

## Acceptance Criteria

- [ ] `src/log/run-log.ts` appends one JSONL record per run to `.vend/runs.jsonl`
      (path configurable) with at least: `runId`, `play`, `epic`, `model`,
      `usage` (input/output tokens), `costUsd`, `outcome` (`success` |
      `gate-failed` | `timed-out` | `budget-exhausted`), `gateResults`,
      `startedAt`, `endedAt`.
- [ ] Records are append-only and one-line-per-run (countable with `wc -l` / `jq`);
      a failed run still writes a record carrying its failure outcome.
- [ ] Pure record construction is unit-tested; the append is a thin fs call.
- [ ] **No dependency** on the seam/budget modules — the runner passes it the data.
