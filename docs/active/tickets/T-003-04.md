## Context

The press, end to end — the convergence node. `vend <sel>` parses the selection
(T-003-03), resolves the indices against the **persisted** `.vend/menu.json`
(T-003-02 — *the same list the user just saw*), and dispatches each picked action's
playbook under its warranted budget, in order, each appended to the run log. Today
the only dispatchable play is `DecomposeEpic` → call `runDecomposeEpic`.

**Shares `src/cli.ts` with `T-003-02`** → depends on it (rule R4, file ownership).

**Cites:** `epic/E-003.md` (dispatch + default/`--budget` override + `--all`);
`src/play/decompose-epic.ts` (`runDecomposeEpic`, the dispatch target);
`src/log/run-log.ts`; `src/budget/budget.ts`.

## Acceptance Criteria

- [ ] `vend <sel> [--budget <ms>,<tokens>]` in `src/cli.ts`: `parseSelection`

      "re-run vend" and stop** (never act on stale indices).
- [ ] Each resolved index → its action's playbook run under the **warranted
      budget** (overridable), **in order**; each run appended to the run log
      (`T-001-04`). Multi-select is sequential (concurrency is out of scope).
- [ ] `--all` reveals hidden blocked/leaf rows; an out-of-range index hard-errors
      **before** any dispatch.
