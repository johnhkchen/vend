## Context

Run the consistency probe (T-019-01) on **expand / survey / steer** as a **bounded,
cost-aware sweep** (N small — e.g. 3 — since survey/steer are heavy at 300k/400k), and
synthesize a **findings note + verdict**. Each play is cast on a **fixed, known-grounded
input** so the honest-empty rate reads as over-eagerness directly.

The note reports, per play: **run-to-run output variance** and the **outcome mix** (signal /
honest-empty / budget rate), and returns a verdict — **is the articulation-play consistency
acceptable, or do the honest-empty / fork-genuineness gates or the budgets need tuning?**
Honest about the sample (small N → directional, not proof — the E-014 discipline). Log what
was sampled (no silent caps).

**Cites:** T-019-01 (the harness); E-016 finding (2) (the 3-casts-3-outcomes warning to
confirm/refute); E-014 findings (the verdict/honest-sample shape); `demand.md` (the verdict
bridges back — a "tune the gates" branch becomes a signal).

## Acceptance Criteria

- [ ] The probe runs on expand/survey/steer (bounded N, logged); results land in a findings
      note at `docs/active/work/T-019-02/`.
- [ ] The note states, per play, variance + outcome mix + honest-empty rate, and a clear
      **verdict** (consistency acceptable / gates over-fire / budgets still off), honest about
      the small sample.
- [ ] If the verdict is "tune the gates", it names the concrete next signal (bridges to
      `demand.md`), the E-014 pattern.
- [ ] `bun run check:*` green. (The live sweep is the human verification at sweep.)
