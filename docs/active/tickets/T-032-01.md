## Context

The pure foundation: the `tools` declaration on the `Play` contract + the pure seam wiring + the pure
resolver. **No I/O, no project-file reading** (T-032-02 does that). Three parts:

1. **`tools?` on the `Play` contract.** Add `readonly tools?: PlayTools` to `Play<I,O>`
   (`src/engine/play.ts` ~127) where `PlayTools = { mcp?: readonly string[]; allow?: readonly string[];
   skills?: readonly string[] }`. **Optional** ‚Äî exactly like `maxTurns?` (~157): an undeclared play is
   unchanged (back-compat). Document it in the same key as `maxTurns` (a per-play sibling resolved at
   cast, threaded to the seam). `mcp` = server ids the play requires; `allow` = built-in tool allowlist
   (e.g. `"Read"`, `"Grep"`); `skills` = forward-compatible, **wiring deferred** (E-032 scope cut ‚Äî the
   field may exist, but this slice does not inject skills).
2. **`buildArgs` emits the scoping flags.** Extend `buildArgs` (`src/executor/claude.ts` ~115 ‚Äî the
   PURE argv builder where `--max-turns` is appended) to accept `mcpConfig?: string`,
   `allowedTools?: readonly string[]`, `strictMcp?: boolean` and append the corresponding flags ‚Äî
   `--mcp-config <path>`, `--allowedTools <comma-or-space list>`, `--strict-mcp-config`. **Verify the
   EXACT flag names + list syntax against `claude -p --help`** (this is the one external contract ‚Äî do
   not assume the spelling; confirm `--allowedTools` delimiter and whether `--strict-mcp-config` is the
   current flag). Omitting them leaves today's argv byte-identical (the `maxTurns` discipline).
3. **Pure `resolveTools(declared, available)`.** Given a play's `PlayTools` and the set of server ids
   the project provides (passed in ‚Äî NO file read here), return a tagged result: either
   `{ ok: true, mcp: string[], allowedTools: string[], strict: true }` (the declared servers that
   resolve + the allowlist, strict on) **or** `{ ok: false, missing: string[] }` (one or more **required**
   `mcp` ids absent from `available`). An **undeclared** play (`tools` undefined) ‚Üí a distinct
   `{ ok: true, passthrough: true }` (no flags, inherit global ‚Äî back-compat). This is the pure core
   T-032-02's cast path consumes to decide between wiring the flags and raising the andon.

Do **not** read `.mcp.json` or spawn anything here; do **not** change `dispense`'s behavior for
undeclared plays. Keep `buildArgs` total and pure (the existing tests must stay green byte-for-byte for
the no-tools path).
