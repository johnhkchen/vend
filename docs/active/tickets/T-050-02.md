## Context

**Thread the funding-headroom core into the path where a cast is FUNDED — so an under-calibrated run is
RUN under the headroomed guard, while the price the shelf quotes stays the honest p90.** T-050-01 built
the pure `fundingEnvelope`; this wires it into the impure pricing/funding seam so the censoring ratchet
actually breaks at runtime.

**Where it threads:**
1. **`src/play/chain-propose-decompose-core.ts` (`resolveStepBudgets`)** — the per-step funding the E-049
   `vend chain` decompose took (`per-step ?? uniform budget ?? play default`). The play-default rung is
   where a cold-start cast is funded at the thin prior; apply `fundingEnvelope` there so the decompose
   step is funded over its observed censored lower bound. (An explicit `--budget` / per-step override
   still wins — the human's ceiling is the human's, unchanged.)
2. **`src/play/work.ts` (the `recalibrate` pricing path, ~L193–199)** — the macro loop prices
   propose/decompose via `recalibrate` cold-starting to `budgetForTier`. Fund each cast under
   `fundingEnvelope` so a cold-start cast in `vend work` can record too.

**The honest-price invariant (IA-8, load-bearing):** the funding headroom is the GUARD a cast runs under,
NOT the price the user is quoted. The shelf / affordability / `formatEnvelopeLabel` (E-013/E-030) keep
showing the measured p90 / estimate — do NOT display the widened funding number as the price. This is
exactly E-038's `timeoutMsFor` split (the wall-clock guard is `× headroom`; the price is bare `T`).

**P7 stays a hard contract:** `fundingEnvelope`'s headroom is a FINITE multiple of an OBSERVED lower
bound (never "spend whatever it takes"), so a run still cannot overspend its guard. Confirm the macro
wallet (`spendDown`) still hard-stops on the funding envelope, and that a non-cold-start, well-calibrated
play is funded EXACTLY as today (funding == priced) — pure back-compat.

**The deterministic proof (stub run-records — NO live model):** build a ledger fixture matching the
E-049 shape — a `decompose-epic` with a 120k cold-start prior and one `budget-exhausted` record logging
~265k actual tokens. Assert: under the OLD path the next decompose is funded at 120k (would censor again);
under the NEW path it is funded ≥ ~265k × `MEASUREMENT_HEADROOM` (room to complete and RECORD), while the
QUOTED price for that same cast is unchanged (still the honest p90/estimate). This is E-050's "Done looks
like" — proven deterministically, no `claude -p`.

**Codebase-memory (use it — project coding-task suggestion):** confirm the funding/pricing seam with the
