## Context

Wire the pure core (T-016-01) into a registered, castable play with a gesture — the
headline: `vend expand "<fragment>"` turns a felt "this is rough" into a **staged, priced
signal** in one gesture. Mirrors `propose-epic.ts` / `note.ts`: assemble inputs (the
fragment + project context), `castPlay(expandFragmentPlay, …)` through the metered seam,
and an **effect that STAGES the signal** (not auto-promoted) for a human pull.

**Staging, not promotion:** the candidate signal lands where a human reviews + pulls it
(the `docs/active/pm/` staging contract), never appended straight onto `demand.md`.

**Cites:** `src/play/propose-epic.ts` (`castProposeEpic`, register + effect to mirror);
`src/play/note.ts` (the cast/effect/gesture pattern); `src/engine/play.ts` (registry) +
`cast.ts` (`castPlay`); `src/cli.ts` / `src/shelf/press.ts` (the gesture); `docs/active/pm/`
(the staging contract); the pre-filled measured budget (E-013 `recalibrate`).

## Acceptance Criteria

- [ ] `expandFragmentPlay` registered on the engine; `castExpandFragment({ fragment, budget,
      … })` casts it (render → dispense → gates → effect → log), like the other plays.
- [ ] A gesture — `vend expand "<fragment>"` — casts the whole play: on success it **stages a
      structured signal** (in the `demand.md` shape) for review; a read-never-invent / honest-
      empty refusal halts with a clear andon (no fabricated signal materialized).
- [ ] A fixture/canned-reply test proves fragment → staged signal end to end; the effect
      writes to the staging location, not the live board.
- [ ] `bun run check:*` green. (The live cast — a real fragment → a real staged signal — is
      the human verification at sweep.)
