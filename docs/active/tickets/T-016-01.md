## Context

The pure core of `expand-fragment` (R1 pure-core-first, R3 foundation), mirroring
`propose-core.ts`. Define the **`Signal` output shape** (what · why · value tier ·
pre-filled budget envelope · `advances` invariant · readiness), the **`ExpandFragment`
render/parse** (a new BAML function — `b.request.ExpandFragment(fragment, charter, project)`
renders; `b.parse.ExpandFragment(text)` SAP-parses; authoring-only, via an `expand-bridge`
like `propose-bridge.ts`), and the **pure gates**:

- **Read-never-invent (the signature gate, PE-1):** the signal must trace to the fragment /
  real project state; a fabricated/speculative signal is refused.
- **Honest-empty (IA-4):** a fragment that closes no vision-distance → a refusal/empty result,
  not manufactured busywork.
- **Value-link:** the signal names the invariant or core-feature advance it serves (PE-7-style).

**Cites:** `src/play/propose-core.ts` (the pure-gate pattern); `baml_src/` +
`src/baml/propose-bridge.ts` (the BAML authoring + bridge to copy); `demand.md` (the signal
shape); `playbooks/propose-epic.md` PE-1; `information-architecture.md` IA-4; E-013 (the
measured budget the signal pre-fills).

## Acceptance Criteria

- [ ] A typed `Signal` output shape + the `ExpandFragment` BAML function (render + parse),
      wired authoring-only via an `expand-bridge` (no transport in BAML).
- [ ] **Pure** gate functions — read-never-invent, honest-empty, value-link — each
      **unit-tested**: a grounded fragment passes; a speculative one is refused; a
      vision-distance-zero fragment yields honest-empty (not a fabricated signal).
- [ ] The core is pure (no fs / no spawn) and composes into the `Play.gates` contract shape.
- [ ] `bun run check:*` green.
