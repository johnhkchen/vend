## Context

R12 (shared-contract-first): define the contract every play hangs on. A **`Play`**
is a named bundle of the things currently *welded* into `runDecomposeEpic` — read
`src/play/decompose-epic.ts` and note exactly what varies per play: the BAML render
(`b.request.X` → `extractPromptText`), the SAP parse (`b.parse.X`), the gates
(`clear`), the effect (`materialize` + `lisaValidate`), the default budget, and the
card metadata. A **registry** maps name → `Play`. This is what makes the engine
play-agnostic; it is pure types + a map, no orchestration yet.

**Cites:** `src/play/decompose-epic.ts` (the hardcoded runner — the per-play
variation to abstract); `card-model.md` (the `card = { color, type, rarity, … }`
metadata); `playbook-decompose-epic.md` (a play's gates/effect).

## Acceptance Criteria

- [ ] `src/engine/play.ts` exports a `Play<I, O>` interface: `name`;
      `render(inputs: I) -> string`; `parse(text: string) -> O`;
      `gates(out: O, ctx) -> GateResult`; `effect(out: O, ctx) -> Promise<EffectResult>`;
      a default `budget`; and `card` metadata (color / type / rarity).
- [ ] A `registry` (name → `Play`) with `register` / `get`; an unknown name returns a
      typed error, never undefined-deref.
- [ ] Pure — types + the map only; no dispense, no fs. Unit-tested (register/get +
      the unknown-name error).
- [ ] `bun run check:test` / `check:typecheck` green.
