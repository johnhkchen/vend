## Context

The foundation for measured envelopes (R3): the run log must record **cost vs budget**,
and the recalibration core must be able to **read it back**. Today `run-log.ts` writes
actuals (`usage`, `costUsd`, `startedAt`/`endedAt`, `outcome`) but **not the allocated
envelope** — so we can't tell how close a cast ran to its budget, nor mark censored
(andon'd-at-envelope) runs. Add the allocated `Budget` to the run record
(backward-compatible — additive/optional field, existing records still parse), and
provide a **pure** reader that loads `.vend/runs.jsonl` and filters a play's runs by
outcome.

**Cites:** `src/log/run-log.ts` (the record shape + writer — extend); `.vend/runs.jsonl`
(existing shape: `play`, `outcome`, `usage`, `costUsd`, timestamps); `src/budget/budget.ts`
(the `Budget` type to embed); `information-architecture.md` IA-13 (successes vs censored).

## Acceptance Criteria

- [ ] The run record carries the **allocated envelope** (`Budget`) alongside actuals,
      written on every cast; the change is **backward-compatible** (existing
      `.vend/runs.jsonl` records without the field still parse — optional/defaulted).
- [ ] A **pure** reader (e.g. `readRuns(jsonl): RunRecord[]` + a `forPlay(records, play,
      {outcome?})` filter) loads and filters runs; tolerates malformed/partial lines
      without throwing (skip + count). No fs in the pure core — fs read is the thin shell.
- [ ] Unit-tested: a fixture log → correct parse, play-filter, outcome-filter, and
      a derived wall-clock (`endedAt − startedAt`) and total-tokens per record.
- [ ] `bun run check:*` green; existing casts still write valid records.
