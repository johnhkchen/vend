## Context

`vend audit` conflates two kinds of E1 evidence: **attested** back-fill (post-hoc — `attest-
intervention.ts` writes an `intervenedAttestation` marker) and **forward** live self-reports (the
real measurement). E-026's verdict counted 13 attested + 2 forward as "15 forward, 93%, forward-
confirmed" (`work/T-026-04/verdict.md`, corrected). Root cause is below the verdict: **`reviveRecord`
strips the marker** — confirmed, the revived record has no `intervenedAttestation` key — so the audit
can't see provenance even though the raw JSONL carries it.

Three layers (house split — reader + pure stat + render):

1. **`src/log/run-log.ts`** — make the provenance survive read. Add a normalized
   `readonly intervenedAttested?: boolean` to `RunRecord` (+ `RunRecordInput` for write symmetry);
   `reviveRecord` sets `intervenedAttested: true` when the parsed line carries a truthy
   `intervenedAttestation` **or** `intervenedAttested === true`. **Existing back-fill records are
   classified by their existing marker — no ledger rewrite.** Live records read as not-attested.
2. **`src/ledger/walk-away.ts`** — split `InterventionStat`: add `attested` and `forward`
   sub-readings (each `{ reported, intervened, rate }`), partitioned on `r.intervenedAttested`. Keep
   the combined `reported`/`intervened`/`rate`/`trend` exactly as-is (back-compat). Pure.
3. **`src/cli.ts` / `formatWalkAwayFindings`** — render the combined walk-away line **plus** the
   split: e.g. `forward (live): X/Y · attested back-fill: 13/13`. The forward count is the one a
   verdict cites.

Keep `intervened === false` as the walk-away definition; the split is additive, not a redefinition.

**Cites:** `src/log/run-log.ts` (`reviveRecord` ~326, `RunRecord`/`RunRecordInput` ~121/154);
`src/ledger/walk-away.ts` (`InterventionStat` ~66, `auditWalkAway` ~166, `formatWalkAwayFindings`
~194); `src/ledger/attest-intervention.ts` (the marker writer); `work/T-026-04/verdict.md` (the
over-claim this prevents).

## Acceptance Criteria

- [ ] `reviveRecord` surfaces `intervenedAttested: true` for records carrying the
      `intervenedAttestation` marker (and for an explicit `intervenedAttested: true`), absent
      otherwise; **unit-tested** (back-fill line → attested; plain forward line → not). No ledger
      rewrite; existing records reclassify on read.
- [ ] `auditWalkAway` returns `attested` + `forward` sub-stats (reported/intervened/rate) split on the
      flag, with the combined stat unchanged; **unit-tested** on a mixed fixture (e.g. 13 attested
      all-walk-away + 2 forward 1-intervened → forward rate distinct from combined).
- [ ] `vend audit` renders the split (forward vs attested) beside the combined rate.
- [ ] **Live proof (free, no cast):** `bun run src/cli.ts audit` over the real ledger shows **forward
      (live): 2** (1 intervention) and **attested back-fill: 13/13**, combined 14/15. `bun run check:*`
      green.
