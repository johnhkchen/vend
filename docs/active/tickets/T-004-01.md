## Context

The pure, addon-free heart of the guard: given the ids a `WorkPlan` would
materialize and the ids already on the target board, return the colliding ones.
This is the structural poka-yoke (`playbook-decompose-epic.md` gate 4) extended to
*cross-board* uniqueness. The E-001 live proof (`work/T-002-04/proof.md` kaizen #4,
`runs.jsonl` A1) showed the play minting `S-001`/`T-001-01…` — safe only because A1
materialized into a sandbox. Pure + testable so it never loads the BAML addon.

## Acceptance Criteria

- [ ] `src/play/id-guard.ts` exports
      `detectCollisions(generated: readonly string[], existing: readonly string[])
      -> string[]` — returns the colliding ids (empty array = clear). PURE: no fs,
      no network, no addon.
- [ ] `src/play/id-guard.test.ts`: a colliding fixture returns **exactly** the
      reused ids; a disjoint fixture returns `[]`; order/dedup behavior pinned.
- [ ] `bun run check:test` and `bun run check:typecheck` stay green.
- [ ] No dependency on `materialize`/`project-context` — it takes plain string
      arrays (composed by T-004-02).
