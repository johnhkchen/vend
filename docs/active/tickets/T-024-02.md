## Context

The walk-away engine (P4). Given a funded wallet (T-024-01) and the board's ranked work, **spend the
wallet down**: predict each next pull's price (E-013 `recalibrate.ts`), check `canAfford`, cast it
(via `castChain` — pull→clear), **debit the wallet by the cast's actuals**, and repeat **until a
clean stop**: the wallet can't afford the next cast · the board is cleared · an andon fires. No
overspend past the wall (P7); a gate-stop ends the session as a *successful refusal* (IA-9), not a
crash.

Split per the house pattern:
- **Pure decision core** (unit-tested): `fitNext(wallet, candidates, priceOf) → next | null` (the
  highest-leverage candidate that still fits, IA-1 rank as the policy — no new optimizer) and
  `shouldContinue(wallet, boardState, lastOutcome) → {continue|stop, reason}` (the three stop
  conditions). Deterministic, no I/O.
- **Impure loop** (not unit-tested, like `castChain`): drive the decision core, call `castChain`,
  read each cast's **actuals** to debit. **Requires the `RunSummary` actuals seam:** `castPlay`
  computes usage/cost for the log but does not return it — extend `RunSummary` to surface the cast's
  actual `Usage` (+ wall-clock ms) so the wallet debits precisely; the log read is the fallback if a
  cast surfaces none.

The loop emits, per step, which pull is running against the wallet burn (the IA-7 production-line
signal T-024-03 renders) and ends with a structured **session result** (what cleared, per-cast cost,
remaining, stop reason) for the Settle summary.

**Cites:** T-024-01 (the wallet it depletes); `src/engine/chain.ts` `castChain` + `src/engine/cast.ts`
`RunSummary` (the cast it drives + the actuals seam to extend); `src/ledger/recalibrate.ts` (the
measured price `fitNext` ranks against); `src/play/survey.ts` (the ranked board it pulls from);
`information-architecture.md` IA-1 (leverage rank = the pull policy), IA-9 (andon = clean stop), P7
(no overspend); E-014 (this loop's runs are the forward, variance-bearing E1).

## Acceptance Criteria

- [ ] A **pure** decision core — `fitNext` (highest-leverage affordable candidate, or null) +
      `shouldContinue` (stop on wallet-can't-afford / board-cleared / andon, with the reason) —
      **unit-tested** on fixtures covering each stop condition + the fits/doesn't-fit boundary.
- [ ] `RunSummary` surfaces the cast's **actuals** (usage + wall-clock ms); the existing cast path
      and tests are unaffected (extend, don't break).
- [ ] An impure loop that funds → fits → casts (`castChain`) → debits → repeats → stops cleanly,
      returning a structured session result (cleared casts, per-cast cost, remaining wallet, stop
      reason). **Never authorizes a cast the wallet can't afford** (P7); an andon ends the session.
- [ ] `bun run check:*` green.
