## Context

*"All I know is we usually overestimate by ~80%."* Turn the run log into a **calibration
dataset** that learns the systematic estimate-vs-actual bias and corrects raw estimates
(IA-16) — the reference-class / outside-view move. Two levels, as asked:

- **generic play level** — the play's cost pooled across projects (its intrinsic behaviour);
- **project level** — this codebase's deviation (big epics here run decompose long).

Combine by **partial pooling**: lean on the generic prior when this project's data is thin,
shift to project-specific as it accumulates (empirical-Bayes shrinkage — the two-level form
of T-013-02's cold-start). A **pure** core over the reader (T-013-01) and the recalibrate
core (T-013-02): given (allocated, actual) pairs, learn the **bias factor** (the actual/
allocated ratio distribution) per {play, project} and return a **corrected estimate** with
the hierarchical fallback project → generic play prior → authored default.

This ticket also adds the one dataset-shape field the two levels need that T-013-01 doesn't:
a stable **project identifier** on the run record (backward-compatible) so runs are
groupable by project as well as play. (Serialized after T-013-01's record work — R4.)

**Cites:** `src/budget/recalibrate.ts` (T-013-02 — extend); `src/log/run-log.ts` + reader
(T-013-01 — add the project tag); `information-architecture.md` IA-16 (reference-class,
hierarchical, bias-corrected), IA-13 (cold-start prior, censoring); `charter.md` P5
(local-first — a generic *cross-project* corpus via a user-global store is a documented
follow-up, not this slice).

## Acceptance Criteria

- [ ] The run record carries a stable **project identifier** (backward-compatible — absent →
      a derived default); the reader can group a play's runs by project.
- [ ] **Pure** `calibrate(estimate, {play, project}, projectRecords, genericPrior)`: learns
      the empirical bias factor from (allocated, actual) pairs of **successful** runs
      (censored excluded, IA-13) and returns `{ corrected, factor, confidence: { projectN,
      genericN } }`. Direction is data-driven (over- *or* under-estimate).
- [ ] **Partial pooling**, unit-tested across three regimes: N=0 (pure generic prior),
      small-N (shrunk toward generic), large-N (project-dominant). The corrected estimate
      moves monotonically from prior to project-specific as project-N grows.
- [ ] T-013-02's measured default feeds **through** the bias correction when a raw estimate
      is supplied (e.g. an allocated 2h → a corrected, confidence-tagged figure).
- [ ] `bun run check:*` green. (Live check at sweep: feed a raw estimate, see it corrected
      against this play's measured history, tagged with how much data backs it.)
