## Context

rc.4's default executor registry fabricates a codex reviewer on
`DEFAULT_OPENAI_BASE_URL = http://localhost:11434/v1` (src/executor/openai-compat.ts:34), so
`resolveComplementExecutor` (src/cross-review/resolve-complement.ts) always resolves and the
cross-review gate is never inert — every default-config metered cast dials an endpoint nobody
configured. Make the DEFAULT inert: the complement resolves to a live reviewer only when the
operator has explicitly provisioned one (follow the existing executor-config convention — the
"how" is your Research finding). The comment on the gate ("deliberately inert for a one-seat
registry") describes the intent this ticket makes true.

_Advances: P2 — The run is two gestures (a fresh brew install must never require a second live
service); P3 — Gates are the contract (a gate that binds to a phantom is not a contract)_

## Acceptance Criteria

- [ ] With default config, `resolveComplementExecutor` returns null for every author seat —
  pinned in a unit test against the exact rc.4 shape (author `claude`, default registry).
- [ ] An explicitly provisioned reviewer still resolves; the provisioning mechanism follows an
  existing config convention and is documented in the work dir.
- [ ] No code path constructs a dialable OpenAI-compat executor from `DEFAULT_OPENAI_BASE_URL`
  without explicit provisioning.
- [ ] `bun run check` green.
