## Context

Prove the gate is real and **drift-free** — the whole reason for Dagger. With the
Docker daemon up, `dagger call test` runs `check:test` *in the container* and must
**agree** with `bun run check:test` run standalone; and the gate must be able to
**fail** (a deliberately-broken test fails it). The same `check:test` is what the
play will invoke as an andon gate — the three-way agreement (standalone · play · CI)
is the point.

**Andon:** the Docker daemon must be up (engine cold-start ~18s; `ci-strategy.md`
measured it — keep-warm is mandatory *later*, not this slice). If Docker is down,
**surface it** — do not fake the verification.

**Cites:** `ci-strategy.md` (the first slice; the three-way no-drift agreement; the
18.4s cold-start measurement); `playbooks/ci-structural-gate.md` (step 6 — confirm
no-drift).

## Acceptance Criteria

- [ ] `dagger call test` (Docker up) runs the suite in-container and **agrees** with
      standalone `bun run check:test` (both green on the current tree).
- [ ] A deliberately-broken test makes the gate **fail** (non-zero / red), proving it
      can fail — then revert the break.
- [ ] A short note records the cold-start / run time and confirms `/ci` stays a
      separate program (imports nothing from the app).
- [ ] **Out of this slice:** `lint`, `typecheck`, `consistency`, keep-warm — each
      generalizes out of this one honest gate; do not bundle them in.
