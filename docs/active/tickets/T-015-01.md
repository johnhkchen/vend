## Context

The mechanism (R3/R12): add an optional turn cap that reaches the `claude -p` invocation.
`buildArgs` (PURE) already threads `--model`/`--effort`/`--system-prompt` — add
`--max-turns <n>`, emitted only when a cap is set (absent ⇒ unchanged behaviour, byte for
byte). Thread it from the cast: `CastOptions.maxTurns?` → `dispense` → `buildArgs`.

**Cites:** `src/executor/claude.ts` (`buildArgs` line ~107, the pure argv builder;
`dispense` spawns it); `src/engine/cast.ts` (`CastOptions` ~line 34, and the `dispense({…})`
call ~line 120 — thread `maxTurns`); `information-architecture.md` IA-8 (the mid-flight
bound this enables).

## Acceptance Criteria

- [ ] `buildArgs` accepts an optional `maxTurns` and emits `--max-turns <n>` when set; when
      absent, the argv is **identical to today** (the existing flags + order unchanged).
- [ ] **Pure** and unit-tested: `maxTurns` present → flag + value in argv; absent → no flag;
      composes with the existing `model`/`effort`/`system` flags.
- [ ] `CastOptions` carries an optional `maxTurns`, threaded through `dispense` to `buildArgs`;
      a cast with no `maxTurns` dispenses exactly as before.
- [ ] `bun run check:*` green; existing casts and the stream-parse path unaffected.
