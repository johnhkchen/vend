## Context

The policy + observability on top of T-015-01's mechanism. Wire a **warranted default**
turn cap into the dispense path (generous enough not to cut off legitimate runs, tight
enough to rein in the wandering E-014 quantified), with the per-cast override still
winning. And make the **turns each cast used observable** — so the cap is tunable from real
data (decompose's clean runs vs its wandering ones), not frozen as a guess.

The default is a **judgment, not a guess to freeze**: pick a defensible number, log
turns-used, and let data refine it. Err generous (a false andon on a legitimate run is
worse than letting one tail run through).

**Cites:** `src/executor/claude.ts` / `src/engine/cast.ts` (the dispense path + the
`maxTurns` thread from T-015-01); `src/log/run-log.ts` (where turns-used could surface, like
`usage`); `demand.md` (the fat-tail evidence — the default's justification); E-014 findings
(the ~85–95k tail).

## Acceptance Criteria

- [ ] A **default** `maxTurns` applies to dispense when no per-cast override is given; the
      override (T-015-01) still takes precedence; the chosen default is justified in-code
      against the observed tail.
- [ ] The **turns used** by a cast are observable (logged/surfaced — e.g. on the run record
      or the stream summary) so the cap can be calibrated from data.
- [ ] **Live bound-check (sweep):** a `decompose-epic` cast on a meaty epic (e.g. E-013) caps
      turns and lands its token spend **below the prior ~85–95k tail** — the fat tail shrinks.
- [ ] `bun run check:*` green; a cast that legitimately needs few turns is unaffected (no
      false andon).
