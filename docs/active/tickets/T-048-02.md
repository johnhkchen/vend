## Context

**Thread ONE shared wallet through `castGraph`'s wave dispatcher — so concurrent branches draw from a
single envelope with a correct hard stop.** T-048-01 built the pure wave algebra (`authorizeWave` /
`debitWave`); this wires it into the impure shell so P7 holds under concurrency.

**The change in `castGraph` (`src/engine/graph.ts`):** the wave dispatcher takes ONE shared wallet (or
a macro `Budget` it `allocate`s once). For each topological ready-set:

1. **Authorize before dispatch** — `authorizeWave(wallet, readySet, priceOf)` → `{dispatch, stopped}`.
   Only the **`dispatch`** subset is run (the existing `Promise.all` over those nodes); the
   **`stopped`** nodes are a **clean hard stop at the wave boundary** (and their dependent subgraph is
   skipped, the `runGraph` halt semantics) — the wave never authorizes casts it cannot collectively
   afford. (The `shouldContinue` wallet-exhausted analog, at wave granularity.)
2. **Debit after settle** — once the wave's casts settle, `debitWave(wallet, actuals)` (tokens summed,
   wall-clock MAX) → the new shared wallet, threaded into the next ready-set. Concurrent debits are
   serialized through this one fold (no racing wallet mutation — the wallet is immutable, `debitWave`
   returns a fresh one).
3. **Stop the run** when no further node fits the shared wallet (the hard wall) — a clean stop with the
   remaining readout, mirroring `spendDown`'s `wallet-exhausted`.

**Single-chain / linear path UNCHANGED (back-compat, AC):** a linear graph is a sequence of single-node
waves, where `authorizeWave` == `canAfford`+`fitNext` and `debitWave` == `debit` (T-048-01's
single-node equivalence). `spendDown` (the `vend work` macro loop) is **untouched** — it already
threads one wallet sequentially; this epic only fixes the *concurrent* wave path. Confirm `vend work`'s
behavior is unchanged.

**The worked example (DETERMINISTIC, stub nodes — no live cast):** a fan-out (e.g. A → {B, C}) with
**known stub costs** whose combined predicted cost **exceeds** a small shared envelope. Assert: under
the **shared wallet**, the wave stops at the correct wall (only the affordable branch dispatched, the
other hard-stopped) and the total debited == the bounded envelope (tokens summed, wall-clock max); under
the **old per-node budgets**, the same fan-out would have **overspent** (both branches authorized
against the pre-wave balance). This is E-048's "Done looks like" comparison — proven deterministically,
the `graph-example.test.ts` stub discipline. **No live model.** *(A live re-cast that completes E-047's
join under the shared wallet is a downstream pull, not this ticket.)*
