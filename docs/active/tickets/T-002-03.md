## Context

Wire the hardcoded play end to end — the **"dispense something real"** moment, and
the convergence node of E-001. Read an epic file + charter + project context,
render with BAML, dispense via the seam under a budget, SAP-parse, run the gates,
and on **pass** materialize the `WorkPlan` as lisa-valid story/ticket files; on any
**stop**, surface the andon and write nothing but the run log. Stream every message
to **both surfaces** (live stdout + the run log). Depends on all four
foundation/play modules.

## Acceptance Criteria

- [ ] `src/play/project-context.ts` assembles inputs (epic md, `charter.md`, a thin
      project snapshot) for `DecomposeEpic`.
- [ ] `src/play/decompose-epic.ts` orchestrates render → dispense (seam + budget) →
      parse → gate; on pass calls `src/play/materialize.ts` to write
      `docs/active/stories/*` + `docs/active/tickets/*`; on stop writes nothing but
      logs.
- [ ] `src/cli.ts` exposes `vend run decompose-epic <epic.md> --budget <ms>,<tokens>`
      (the hardcoded single play) as the entry point.
- [ ] Every message streams to stdout **and** the run log (T-001-04); the run's
      outcome is logged once, countably.
- [ ] Budget (T-001-03) is enforced: wall-clock via the seam timeout, tokens
      checked against the result; exhaustion stops the line.
- [ ] Materialized files pass `lisa validate` (the final structural poka-yoke, run
      after write).
