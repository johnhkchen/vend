## Context

The four value-ordered clearing gates from `playbook-decompose-epic.md`: **value →
allocation → bounds → structural**. Gates are the contract (P3): a failed gate
**stops the line** with a reason (andon) and the play emits nothing. For the slice,
keep them honest but minimal — rule-based checks over a `WorkPlan`, never judgment
calls that belong to a human. Imports the generated `WorkPlan` type (hence the dep
on T-002-01); no other app dependencies.

## Acceptance Criteria

- [ ] `src/gate/gates.ts` exports `clear(workPlan, { epic, charter }) -> GateResult`
      running, in order:
      1. **value** — every ticket names what it `advances` + a `doneSignal`;
      2. **allocation** — each ticket is bounded and `depends_on` edges form a DAG
         (no cycles, all refs resolve);
      3. **bounds** — no non-goal violated / no unbacked `advances` claim
         (rule-checkable subset);
      4. **structural** — every `TicketDraft` carries all required lisa frontmatter
         fields.
- [ ] A failed gate returns a `STOP` naming the gate + the offending unit + why;
      the runner must not materialize on a `STOP`.
- [ ] Fully unit-tested with passing and failing `WorkPlan` fixtures.
- [ ] Depends only on `baml_client` types — no seam/budget/log imports.
