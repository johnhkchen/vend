## Context

The single metered seam: dispense to Claude via the `claude -p` headless CLI over
the **subscription**, streaming every message to a hook so the runner can log
transcript + per-turn usage. Port the **proven** pattern from
`/Volumes/ext1/swe/repos/mc-design-eval/src/sdk-binding.mjs` — read it first; it is
the reference implementation (spawn → newline-delimited stream-json parse →
`onMessage` per message → capture terminal `result` → wall-clock timeout guard).
Only the **text path** is needed for the slice (no image/multimodal).

## Acceptance Criteria

- [ ] `src/executor/claude.ts` exports `dispense({ prompt, model?, effort?,
      system?, onMessage?, timeoutMs? })` that spawns `claude -p --output-format
      stream-json --verbose`, writes the prompt to stdin, parses stream-json lines,
      calls `onMessage` per message in order, and returns the terminal `result`
      (carrying `usage`, `total_cost_usd`, `subtype`).
- [ ] A non-returning child is SIGKILLed at `timeoutMs` and surfaces a typed
      `ClaudeTimeoutError` (`code === "ETIMEDOUT_CLAUDE"`) — port `awaitChildClose`
      (exactly one of timeout/close/error settles the latch).
- [ ] Pure helpers (stream-json line parsing, arg building, the timeout latch) are
      **unit-tested with a fake child / sample lines — no live `claude` spawn**
      (mirror mc-design-eval's test rule; the live spawn is not unit-tested).
- [ ] The seam is **budget-agnostic**: it accepts `timeoutMs` but does not own the
      budget (composed by the runner with T-001-03).
- [ ] `CLAUDE_CLI` env override for the binary path.
