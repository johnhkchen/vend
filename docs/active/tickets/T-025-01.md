## Context

E-024's first live `vend work` spend cleared **0** because authorization and execution used different
budgets: the wallet recalibrated the chain at propose 227k + decompose 227k and authorized on it, but
`castWork` called `castProposeDecomposeChain` **with no budget**, so each step ran under its *static
play default* (propose-epic 150k). The #1 pull's propose needed 175k → blew the 150k default →
`budget-exhausted` andon → nothing staged. (Full diagnosis: `work/T-024-03/sweep-logs/findings.md`.)

Thread the reserved price into the cast, **per step** (propose and decompose recalibrate separately
and can differ):

1. **`src/play/chain-propose-decompose.ts`** — extend `ChainProposeDecomposeOptions` with optional
   **per-step** budgets (e.g. `proposeBudget?` / `decomposeBudget?`). Each step casts under: its
   per-step budget if given, else the existing uniform `budget`, else the play default (the current
   behavior — back-compat preserved; a bare `vend run`/`vend chain` is unchanged).
2. **`src/play/work.ts`** — it already computes `recalibrate(propose…)` + `recalibrate(decompose…)`
   for `canAfford` and then only *sums* them into `price`. Keep those two envelopes and pass them as
   `proposeBudget` / `decomposeBudget` into `castProposeDecomposeChain` (the `castOne` thunk). Now
   the cast runs under exactly what the wallet authorized — **authorization == execution**.

The point is correctness of the macro-wallet's premise, not a new price: do **not** bump propose's
static 150k default (that stays the cold-start fallback), and do **not** re-derive per-play envelopes
(E-013 owns them).

**Cites:** `src/play/chain-propose-decompose.ts` (lines ~33–37 the options, ~72–84 the per-step
budget application — the uniform `budget` already wires to both steps; make it per-step); `src/play/
work.ts` (~line 119–130 — the two recalibrated envelopes + the `castOne` call site); `src/budget/
budget.ts` (`Budget`); `work/T-024-03/sweep-logs/findings.md` (the diagnosis); E-013 `recalibrate.ts`.

## Acceptance Criteria

- [ ] `castProposeDecomposeChain` accepts **per-step** budgets (propose / decompose), each falling
      back uniform-`budget` → play-default; the existing uniform `budget` + no-budget paths are
      unchanged (a bare `vend chain` casts exactly as before — back-compat).
- [ ] `castWork` passes the two recalibrated envelopes it already computes as the per-step budgets,
      so a fitted pull casts under the wallet-reserved price (227k), not the static default (150k).
- [ ] Coverage where the seam allows (the per-step budget selection is pure-testable; the live cast
      is not, like `castChain`). `bun run check:*` green.
- [ ] **Live re-sweep (the proof E-024 deferred):** a bounded `vend work` against the staged board
      now **clears ≥1** pull and Settles truthfully, hard-stops intact. *(Human-step — the live cast
      materializes real work + burns tokens; run it as the verification, like the E-024 sweep.)*
