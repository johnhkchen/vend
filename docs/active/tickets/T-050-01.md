## Context

**The pure funding-headroom core — derive the envelope a cast is RUN under (the funding guard), distinct
from the p90 the shelf QUOTES (the price).** `recalibrate` (`src/ledger/recalibrate.ts`) bounds a play's
cost at the tier percentile over its SUCCESSFUL runs and right-censors `budget-exhausted`/`timed-out`
runs out of the sample (IA-13, correct — their true cost is only a lower bound). But it then hands that
same thin percentile (or, cold-start, the hand prior) back as the envelope the run is FUNDED with — so an
under-bounding prior (decompose's 120k) censors the run, the run contributes nothing, and the next run is
funded at the same thin guess and censors again. `recalibrate.ts` (lines 14–16) names this exact gap:
it READS the censored rate into `confidence` but "does not yet ACTUATE on it (IA-14 — auto-widen)".

**The load-bearing fact this exploits:** a censored run **already logs its actual usage** — the E-049
`decompose-epic` recorded `totalTokens` = 264,866 before the 120k wall. That recorded magnitude is a
strong LOWER BOUND on the true cost, currently discarded. We fund the next under-calibrated run above it.

**The pure function (extend `recalibrate.ts` — it already imports `forPlay`/`totalTokens`/`wallClockMs`
and owns `RecalibrateResult`/`CENSORED_OUTCOMES`/cold-start; keep the type-only `budget` import — mirror
`TIMEOUT_HEADROOM` with a local `MEASUREMENT_HEADROOM` const rather than importing `timeoutMsFor`):**

`fundingEnvelope(play, records, result, opts?) → { envelope: Budget; widened: boolean }` (or thread the
records through `RecalibrateResult` — design's call). PER DIMENSION (tokens AND wall-clock, independently,
as `recalibrate` does):
- **Under-calibrated** = `result.source === "prior"` (cold-start) **OR** the windowed censored rate
  `censored / (successes + censored)` ≥ a threshold (`CENSORED_WIDEN_RATE`, e.g. ~1/3) — the envelope is
  provably under-bounding (runs keep hitting the wall).
- When under-calibrated: `funding = max(priced, maxCensoredActual × MEASUREMENT_HEADROOM)`, where
  `maxCensoredActual` is the max `totalTokens` / `wallClockMs` over the windowed CENSORED runs (their
  logged lower bounds). `MEASUREMENT_HEADROOM` (≥ 2; mirror `TIMEOUT_HEADROOM`) clears the observed wall
  so the run finishes and RECORDS. If there are no censored runs to read (pure cold-start, no history),
  fall back to `priced × MEASUREMENT_HEADROOM` so a first run still gets room to record.
- When **trusted-measured** (`source: "measured"` AND censored rate below threshold): `funding = priced` —
  the bound is earned from real successes; no headroom (back-compat — a well-calibrated play is unchanged).
- Coerce to positive ints (budget contract, reuse the module's `positiveInt`); `widened` flags whether
  headroom was applied (for an honest funding label / the caller's log).

**This is a GUARD, not the price** — exactly as `timeoutMsFor` is "the per-cast runaway-guard, NOT the
price." It must NOT change `recalibrate`'s returned `envelope`, `formatEnvelopeLabel`, or the percentile
math — the quoted estimate stays honest (IA-8). PURE/TOTAL: plain values in, fresh `Budget` out; no fs,
clock, or process.

## Acceptance Criteria

- [ ] `fundingEnvelope(...)` returns, per dimension, `max(priced, maxCensoredActual × MEASUREMENT_HEADROOM)`
      when under-calibrated (cold-start OR censored rate ≥ threshold), reading the windowed censored runs'
      logged `totalTokens`/`wallClockMs` as lower bounds; `priced` verbatim when trusted-measured. PURE/TOTAL,
      positive-int dimensions, `widened` flag surfaced.
- [ ] Does NOT mutate `recalibrate`'s `envelope`, the percentile math, or `formatEnvelopeLabel` — the
      QUOTED price stays the honest p90 (IA-8); `MEASUREMENT_HEADROOM` is finite (P7 — a bounded guard).
- [ ] Unit-tested (no live model): the **E-049 shape** (120k prior, a censored run logging ~265k actual ⇒
      funding ≥ ~265k × headroom, completing room); pure cold-start with no censored history ⇒
      `priced × headroom`; trusted-measured + clean ⇒ funding == priced (no headroom); the high-censored-rate
      auto-widen on a `measured` source; empty/degenerate inputs total.
- [ ] `bun run check:*` green.
