## Context

Extract the seam `castPlay` depends on into a real `Executor` interface, put `ClaudeExecutor` behind it,
and route casts through a selector that defaults to Claude (byte-identical). Three parts:

1. **The `Executor` interface.** Define `Executor` (in `src/executor/`) capturing the one boundary
   `castPlay` uses today: `dispense(opts): Promise<ResultMessage>` where `opts` is the existing
   `DispenseOptions` (prompt, model?, effort?, maxTurns?, mcpConfig?, allowedTools?, strictMcp?,
   onMessage, timeoutMs); it streams `StreamMessage`s to `onMessage` and **throws on wall-clock
   timeout**. Give it an `id`/`name` (for the run-log + selection). **Generalize `ClaudeTimeoutError`
   → `ExecutorTimeoutError`** (keep a Claude alias/subclass so existing `instanceof` checks in
   `castPlay` still hold). Document that the agentic options (`maxTurns`/`mcpConfig`/`allowedTools`/
   `strictMcp`) are **hints an executor honors when it can** — Claude honors them; a non-agentic
   executor ignores them. The universal core is prompt + `timeoutMs` + `onMessage` + `model` → a
   metered `ResultMessage`.
2. **`ClaudeExecutor` behind it.** Refactor today's `dispense`/`buildArgs` (`src/executor/claude.ts`)
   into a `ClaudeExecutor` implementing `Executor` — **no behavior change** (the spawn, the stream
   consume, the result surfacing all identical; the existing `claude.test.ts` stays green). `dispense`
   may remain as the impl's method or a thin free-function the class calls — whichever keeps the
   existing tests byte-green.
3. **The selector + `castPlay` routing.** An `executorFor(opts/env)` that returns the chosen `Executor`
   — **default Claude** (so every existing cast is byte-identical; no env ⇒ Claude). Have `castPlay`
   (`src/engine/cast.ts` ~188) resolve the executor via the selector and call `executor.dispense(...)`
   instead of the bare `dispense` import. The `instanceof` timeout check uses the generalized error.

Back-compat is the contract: with no env set, behavior is unchanged. Do **not** change the
parse→gate→effect→log pipeline (the abstraction's whole point is to leave it untouched); do **not** add
the OpenAI executor here (T-035-02).

**Codebase-memory (use it — project coding-task suggestion):** confirm the seam + its callers with the
**codebase-memory-mcp** (project `Volumes-ext1-swe-repos-vend`, indexed/ready) rather than re-deriving
by raw grep:
- `search_code(pattern: "dispense|DispenseOptions|ResultMessage|ClaudeTimeoutError|StreamMessage", regex: true, mode: "full", path_filter: "^src/executor/")` — the exact contract to lift into the interface.
- `query_graph("MATCH (f)-[:CALLS]->(g) WHERE g.name = 'dispense' RETURN f.qualified_name")` — every dispenser, so the routing change lands only at `castPlay` and nothing else regresses.

`dispense` `src/executor/claude.ts:296`; `ResultMessage` `:34`; `buildArgs` `:115`; `ClaudeTimeoutError`.
