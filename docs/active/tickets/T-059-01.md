## Context

**Make `vend steer` read the seed's stated intent — the E-058 make-or-break finding.** Today
`assembleSteerInputs` (`src/play/steer.ts:109`) builds steer's `project` input from `buildProjectSnapshot({
root, srcFiles: [], stories, tickets })` — a board-ids listing — plus `docs/knowledge/charter.md`. The seed's
product idea lives in a **root `SEED.md` that no code reads** (grep: `SEED.md` is only *written* as a stub by
`init-core.ts`), so on a fresh seed steer sees an empty board + a generic charter + no idea, and the honest-
empty rule (`baml_src/steer.baml:71`) correctly fires → empty steer. The seed's own docs promise steer reads
the seed; this ticket makes that true.

Wire the intent through the **existing** snapshot — the `{{ project }}` block the steer prompt already tells
the model to ground demand against (`baml_src/steer.baml:114`). **No BAML change, no regen.**

- `src/play/project-context.ts`: add an optional `intent?: string` to `SnapshotParts`; when present,
  `buildProjectSnapshot` emits a `## Stated intent (SEED.md)` section with the content verbatim (one line by
  design — the deliberate exception to the snapshot's "names, not contents" rule, justified because SEED *is*
  the intent doc). Export a `SEED_PATH = "SEED.md"` constant.
- `src/play/steer.ts` `assembleSteerInputs`: read root `SEED.md` tolerantly (`readFile(...).catch(() =>
  undefined)` — absent ⇒ `undefined`, the `listIdsIn` tolerance precedent) and pass it as `intent`.
- `src/play/survey.ts` `assembleSurveyInputs`: same shared snapshot — wire `intent` too (survey has the
  identical gap; both reuse `buildProjectSnapshot`).
- **Honest-empty preserved:** no `SEED.md` ⇒ `intent` undefined ⇒ no section ⇒ snapshot byte-identical to
  today ⇒ vend-on-itself unchanged. This is the safety the optional field buys.

**Cites:** `src/play/project-context.ts` (`buildProjectSnapshot`, `SnapshotParts` — the pure formatter);
`src/play/steer.ts:109` (`assembleSteerInputs`); `src/play/survey.ts:120` (`assembleSurveyInputs`);
`baml_src/steer.baml:114` (the `{{ project }}` block — unchanged); `docs/active/epic/E-059.md`; the E-058
finding (`examples/templates/hackathon-seed/EXPECTED-OUTCOME.md`).
