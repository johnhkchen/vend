## Context

The pure core of `Survey` (R1 pure-core-first, R3 foundation), mirroring `expand-core.ts`
one scale up: where expand emits one `Signal`, Survey emits a **ranked `Signal[]`** (a
board). **Reuse E-016's `Signal` shape** (what · why · tier · pre-filled budget · `advances`
· readiness). Author a `Survey` BAML function (`b.request.Survey(project, charter)` renders;
`b.parse.Survey(text)` SAP-parses to the ranked set; authoring-only via a `survey-bridge`).
The **pure gates**:

- **Read-never-invent (PE-1):** every candidate traces to real project state; a fabricated
  candidate is refused.
- **Honest-empty (IA-4):** a project with no real demand gradient → an empty board, not a
  manufactured list. (The board-stocker's signature discipline.)
- **Leverage-rank:** the set is ordered highest-leverage first.

**Cites:** `src/play/expand-core.ts` (the `Signal` shape + gate pattern to reuse — copy the
slugify/no-shared-util idiom if needed); `baml_src/` + `src/baml/expand-bridge.ts` (the BAML
authoring + bridge to mirror); `demand.md` (the signal shape); `playbooks/propose-epic.md`
PE-1; `information-architecture.md` IA-3/IA-4.
