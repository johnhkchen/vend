## What changed

**`baml_src/expand.baml`** (prompt-only, the only authored surface — `baml_client/` is a gitignored
`baml:gen` artifact):

1. **`## Honest-empty` section** rewritten from a co-equal "If … ABSTAIN / Otherwise author" branch
   into a **rare, source-gated exception**: a concrete "can you cite ONE real thing — a phrase, a
   file/doc/TODO, a run-log fact?" test (cite → extract, put it in `grounding`; nothing to read →
   abstain), plus a **calibrated example pair** (thin "water the office plants" → ABSTAIN; grounded
   "run log records the bucket but not which gate" → EXTRACT), and an explicit "roughness/terseness
   is never a reason to abstain."
2. **`what` / `why` field `@description`s** tightened so the type-level instruction the model sees via
   `{{ ctx.output_format }}` matches the prose ("blank ONLY when the fragment grounds NOTHING at all
   … a rough-but-grounded fragment is a SIGNAL, not an abstention").

**Unchanged (as designed):** `src/play/expand-core.ts` (+ test), `src/play/expand-fragment.ts`,
`src/probe/run-consistency-probe.ts`, `baml_src/survey.baml`. No field names/types/enum touched — the
generated `Signal`/`SignalTier` shape is byte-stable, which `bun run check`'s 639 green tests confirm.
