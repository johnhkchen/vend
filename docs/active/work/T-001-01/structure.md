# Structure — T-001-01 scaffold-bun-project

The blueprint: exact files created, their shape, and ordering. No prose code
logic beyond what the scaffold literally is (config + one trivial test).

## File tree to create

```
/                              (repo root)
  package.json                 NEW  — runtime+pm metadata, check:* scripts, deps
  tsconfig.json                NEW  — strict TS, Bun baseline
  .gitignore                   NEW  — node_modules, .vend, build output
  bun.lock                     NEW  — generated by `bun install` (committed)
  src/
    smoke.test.ts              NEW  — one trivial passing test (feeds check:test)
    executor/.gitkeep          NEW  — owned later by T-001-02 (empty now)
    budget/.gitkeep            NEW  — owned later by T-001-03 (empty now)
    log/.gitkeep               NEW  — owned later by T-001-04 (empty now)
    gate/.gitkeep              NEW  — andon gate eval (S-002, empty now)
    play/.gitkeep              NEW  — DecomposeEpic play (S-002, empty now)
```

No files are modified or deleted. `node_modules/` and `baml_client/` may appear on
disk after install but are git-ignored.

## File specifications

### `package.json`
- `"name": "vend"`, `"private": true`, `"type": "module"`, `"version": "0.0.0"`.
- `"module": "src/index.ts"` omitted (no entrypoint yet) OR set to a future path —
  decision: omit, no entrypoint exists; avoids pointing at a missing file.
- `scripts`:
  - `"check:test": "bun test"`
  - `"check:typecheck": "tsc --noEmit"`
  - `"check": "bun run check:typecheck && bun run check:test"`
  - `"build": "tsc --noEmit"` (typecheck-only alias; bundling deferred — Design D5)
- `dependencies`: `"@boundaryml/baml": "^0.222.0"`
- `devDependencies`: `"typescript": "^5.7.0"`, `"@types/bun": "latest"`
- Optional hint: `"engines": { "bun": ">=1.3.9" }` to document the floor.

### `tsconfig.json`
Exactly the object in Design D3. Public contract: any `src/**/*.ts` typechecks
under strict + `noUncheckedIndexedAccess`. `include: ["src"]`. `noEmit: true`.

### `.gitignore`
```
node_modules/
.vend/
baml_client/
*.tsbuildinfo
dist/
```

### `src/smoke.test.ts`
```ts
import { expect, test } from "bun:test";

test("scaffold smoke: bun test runs", () => {
  expect(1 + 1).toBe(2);
});
```
Single file, single test. Lives at `src/` top level — collision-free (no sibling
ticket owns this path). Exercises `bun:test` types (proves `@types/bun` wired).

### `src/{executor,budget,log,gate,play}/.gitkeep`
Empty files. Sole purpose: make git track the otherwise-empty skeleton dirs. They
carry **no code** — Design D4 collision rule: siblings T-001-02/03/04 own files in
`executor/`/`budget/`/`log/`; the scaffold must not pre-place modules there.

## Module boundaries established

The boundaries it fixes for downstream:

| `src/budget/`   | T-001-03 budget control | `.gitkeep` only |

The `check:*` script names in `package.json` are the **public interface** of this

Renaming them later is a breaking change for two consumers; they are fixed now.

## Ordering of changes (matters)

1. `package.json` first — defines deps + scripts.
2. `tsconfig.json` — typecheck config must exist before `check:typecheck` runs.
3. `.gitignore` — before `bun install` so `node_modules/` is never staged.
4. `src/smoke.test.ts` + five `.gitkeep` files — source skeleton.
5. `bun install` — generates `bun.lock` + `node_modules/`; resolves `tsc` so
   `check:typecheck` can run. Must come AFTER package.json/.gitignore.
6. Verify gates: `bun run check:typecheck` clean, `bun run check:test` passes.

This ordering is also the commit ordering in Plan (single atomic scaffold commit
is acceptable since the unit is indivisible — see Plan).

## Verification surface (what "structurally correct" means here)

- `bun install` succeeds from clean state, writes `bun.lock`.
- `bun run check:typecheck` exits 0 (no `.ts` under `src` violates strict).
- `bun run check:test` exits 0, reports 1 pass.
- `git status` shows `node_modules/`/`baml_client/` ignored, `bun.lock` tracked.
- The five skeleton dirs exist and are tracked (via `.gitkeep`).
