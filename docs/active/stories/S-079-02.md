**Scope:** A new src/sweep/ slice: sweep-core.ts, the pure flip computation — which epic cards flip to done, the pathspec limited to the flipped board files, the provenance message naming cleared tickets — reusing settle-core's all-done derivation and presweep-core's SweepVerdict/SWEEP_PREFIXES; plus the `vend sweep` verb with one-keystroke confirmation and pathspec-limited git staging. Board files under docs/active/ change only via the commit the operator confirms.

**Story acceptance:** On a fixture repo with one fully-done epic and one partly-done epic, `vend sweep` presents exactly one assembled commit — the done epic's card flip with provenance naming its cleared tickets, pathspec limited to those board files — lands it on a single confirm keystroke, and refuses with a named andon (no commit, non-zero exit) when presweep reports offenders or when no epic is fully done.

**Honest boundary:** Sweep flips cards and commits; it does NOT archive — no file moves to docs/archive/, which stays adjacent work. The confirmation keystroke is deliberately retained: the assembly is machine work, the consent is not (the epic's own constraint). Fixture-proven; the live closeout of a real epic happens in operation, not in this story's tests.
