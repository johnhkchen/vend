Part of **epic E-048 (cross-branch-budget-wallet)** — make the budget hard contract (P7) hold under
the real concurrency E-046/E-047 shipped. Today `castGraph`'s wave dispatcher runs parallel branches
that each authorize against their own static per-node budget with no shared accounting, so two
concurrent casts both pass `canAfford` against the pre-wave balance and both spend — the ceiling leaks
across branches (the exact gap E-047's live cast hit: propose-1 starved). The fix: concurrent casts in
a wave draw from **one shared wallet**.

The load-bearing correctness (the whole epic): under concurrency the two IA-8 denominations **diverge**
— **wall-clock = the wave's ELAPSED = MAX(branch times)** (concurrency stretches the wall-clock budget),
**tokens = SUM** (all real). The existing per-cast `debit` (sums both) is correct sequentially but
over-charges wall-clock for a concurrent wave. Mirror the pure-core ⊥ impure-shell split.

Pure core → thread it: T-048-01 (`authorizeWave` — greedily fit a ready-set into the shared wallet:
each node's time ≤ remaining, cumulative tokens ≤ remaining; dispatch vs hard-stop — the `fitNext`
generalization; + `debitWave` — fold a settled wave, **tokens summed, wall-clock MAX**, IA-8 overshoot;
single-node == old `debit`) → T-048-02 (thread ONE shared wallet through `castGraph`'s wave dispatcher —
authorize before `Promise.all`, debit after settle, hard stop at the wave boundary; single-chain path
unchanged; + a deterministic worked example: a fan-out the shared wallet stops which per-node budgets
would have overspent). FREE/deterministic. Epic-scoped ids (R6 — `048` clean).

## DAG

```
T-048-01  pure wave-budget core — authorizeWave (fit a ready-set: time each-fits, tokens collective) + debitWave (tokens SUM, wall-clock MAX); single-node == sequential debit; unit-tested
   └─ T-048-02  thread ONE shared wallet through castGraph's wave dispatcher (authorize→Promise.all→debit, hard stop at wave boundary) · single-chain unchanged · deterministic shared-vs-per-node worked example
```

**Out of this slice:** per-sub-DAG budget sub-allocation ("reserve 50k for this fan-out"); conditional
edges / cycle/error semantics; the open-model runner; a LIVE re-cast (the deterministic stub example
proves the budget arithmetic — a live graph that completes E-047's join under a shared wallet is a
downstream pull).
