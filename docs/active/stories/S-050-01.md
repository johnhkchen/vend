Part of **epic E-050 (measurement-funding-headroom)** — break the envelope censoring ratchet so early /
under-calibrated runs are FUNDED generously enough to record their actual usage, instead of terminating
off a thin guessed envelope that censors the very tail the calibration needs (twice confirmed: E-045,
E-049). This is the IA-14 **auto-widen** actuation `recalibrate.ts` explicitly defers, and the
token-dimension twin of E-038's wall-clock `timeoutMsFor` headroom (GUARD ≠ PRICE).

The load-bearing fact: a censored run (`budget-exhausted`/`timed-out`) **already logs its actual usage**
(the E-049 decompose recorded 264,866 real tokens before the 120k wall) — a strong lower bound on true
cost that `recalibrate` currently reads only as a count and throws the magnitude away. The fix funds the
next under-calibrated run ABOVE that observed lower bound (× a finite measurement headroom) so it
finishes and records — feeding the success sample so `recalibrate`'s p90 rises honestly from data.

Pure core → thread it: T-050-01 (`fundingEnvelope` — funding ceiling distinct from the priced p90:
cold-start OR high censored rate ⇒ `max(priced, maxCensoredActual × MEASUREMENT_HEADROOM)` per
dimension; trusted-measured ⇒ funding == priced; PURE, unit-tested) → T-050-02 (thread it into the cast
funding path — `resolveStepBudgets` + `work.ts` — while the quoted price / shelf / `formatEnvelopeLabel`
stays the honest p90 (IA-8); deterministic E-049-shaped proof that the decompose now funds-to-complete;
P7 holds — the headroom is a finite guard). FREE/deterministic (stub run-records, no live model).
