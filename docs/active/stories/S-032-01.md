Part of **epic E-032 (per-play-tooling)** — a play declares the MCPs/tools it uses (on the `Play`
contract, author-once, mirroring the `maxTurns?` precedent from E-015), the project supplies the
binding (a committed `.mcp.json`, not the global `~/.claude.json`), and the cast wires in **only** the
declared servers (`--mcp-config`/`--allowedTools`/`--strict-mcp-config`) — least-privilege. A required
MCP the project doesn't provide → amber andon (IA-9). Back-compat: an undeclared play is unchanged.
`decompose-epic` becomes the proof play — it references `codebase-memory-mcp` itself.

Build contract+seam → bind+wire: T-032-01 the `tools?` field + the pure `buildArgs` flags + pure
`resolveTools` (resolved flags | missing list), unit-tested → T-032-02 the project `.mcp.json` +
cast-time resolution + the missing-MCP andon + decompose's `tools` (free deterministic proof over the
built argv). Epic-scoped ids (R6 — `032` clean).

## DAG

```
T-032-01  tools? on Play + buildArgs MCP/tool flags + pure resolveTools(declared, available)
   └─ T-032-02  project .mcp.json + cast-time resolution + missing-MCP andon + decompose proof play
```

**Out of this slice:** skill-injection wiring (the `skills` field may exist forward-compatibly, the
wiring is downstream); strict-by-default for undeclared plays (back-compat now, strict opt-in); per-play
secrets management (env-var indirection only); a Vend-specific config format (reuse `.mcp.json`).
