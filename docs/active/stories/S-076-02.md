**Scope:** the settlement tail of `src/engine/cast.ts` (the cross-review dispense call and
everything between the effect landing and the run-record write) and the run-record/artifact write
path. Three invariants: (1) a provisioned reviewer that fails at dispense time (unreachable,
timeout, malformed reply) lands as the existing amber andon shape (E-074's executor-unreachable
precedent, `missing-capability` family) with cause + fix-it hint — an outcome, never an uncaught
rejection; (2) the run record is written even when the settlement path throws — the cast spent
real tokens and MUST leave its ledger line (try/finally or equivalent around the record write);
(3) a captured diff artifact and its ledger row land together — no more
`.vend/artifacts/run-….diff` with no matching record (the boilerplate-demo stray).

**Story acceptance:** a unit test injects a throwing reviewer and asserts: named andon outcome on
the record, ledger line present, no unhandled rejection; a test proves record-write-on-throw for
a non-reviewer settlement error too (the invariant is general, not reviewer-specific); the
no-network characterization test (T-076-02-03) runs a full castPlay with default config and NO
listener on 11434 and asserts success + `crossReviewSkipped` + ledger line + artifact/record
consistency — the exact shape both field failures hit, with real fetch semantics (mock the MODEL
executor, never the reviewer resolution or the failure path under test).

**Honest boundary:** the characterization test must not require tokens — mock the primary
executor's dispense, keep everything from settlement onward real. If full-fidelity no-network
testing proves impossible in-suite, the honest fallback is a documented manual probe script plus
the unit tests — say so in review, don't fake it.
