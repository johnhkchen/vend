Part of **epic E-027 (board-freshness-gate)** — refuse to spend a wallet down a stale staged board.
E-025's live re-sweep proved `vend work` will clear the #1 signal of a ~10h-old board (minting E-026
on an already-resolved premise); a wallet spent on a stale board is overproduction. The fix is a
board-level freshness gate in the andon family (IA-9): the board is stale iff it predates the newest
`docs/active/{epic,stories,tickets}` change → refuse (amber), hand back the re-survey move, with a
`--stale-ok` override (IA-5). Single ticket; the live proof is free + deterministic (no LLM cast).

## DAG

```
T-027-01  freshness gate — pure isBoardStale + mtime gather + stale-board outcome + andon + --stale-ok
```

**Out of this slice:** auto-re-survey (a `--refresh` opt-in is downstream — silent auto-survey
violates the funding gesture, IA-5); per-signal liveness filtering (board-level gate, not signal);
a board-embedded state fingerprint (a stronger signal, downstream).
