**Scope:** The run ledger's per-lane heat substrate — `src/log/run-log.ts` gains a `seatOfExecution` field (which KNOWN_SEATS lane a run burned on), and the cast loop (`src/engine/cast.ts` + `src/engine/cast-core.ts`) stamps it from the resolved executor's lane at end-of-cast, forwarded exactly like the `seatDefaulted` marker already is. Only the ledger write/read path changes; materialize, the decompose effect, and the reader are untouched here.

**Story acceptance:** A cast through a stub executor writes a run record whose `seatOfExecution` equals the executor's KNOWN_SEATS lane, `reviveRecord` round-trips it through `readRuns`, and a pre-E-071 record with no such field still parses byte-identically (field omitted, reads as unknown, never as a lane); `bun run check` green.

**Honest boundary:** Fixture-proven and FREE — stub executor, pure unit tests, no tokens; no live metered cast in this story. Vend's own casts only ever burn the claude lane, so this substrate populates the claude side live; capturing codex-lane burn (lisa's dispatch) and any 429/cap signal is the FULL substrate deferred to follow-on epic (b), named here, not folded into a ticket.
