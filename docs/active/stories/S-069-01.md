**Scope:** The `--agent <seat>` thread from the two board-writing gestures (`vend chain`, `vend run decompose-epic`) end to end, mirroring the existing `--after` plumbing exactly: a small pure seat-contract module (`KNOWN_SEATS` + validation guard + typed error) and the new `agent?` field on `ContextSources`→`DecomposeInputs` in src/play/project-context.ts; the frontmatter stamp + write-side seat guard in src/play/materialize.ts; the effect relabel + new run-log outcome in src/play/decompose-epic.ts and src/log/run-log.ts; the chain option field in src/play/chain-propose-decompose.ts; and the CLI parse/dispatch in src/cli.ts. Epic cards, ticket BODY prose, the gates, render, and lisa's DISPATCH are untouched — only the ticket frontmatter gains one line and the input shape gains one optional field.

**Story acceptance:** Casting decompose-epic (via `vend run` or `vend chain`) with `--agent codex` through a stub executor yields ticket files whose frontmatter carries `agent: codex` on the line immediately after `priority:` on every materialized ticket, with no hand editing; omitting the flag yields output byte-identical to today's mint (no `agent:` key written); and passing an unknown seat (anything outside `claude | codex`) is refused with a named andon relabeled to an `unknown-seat` run-log outcome BEFORE any file is written (zero partial output), not deferred to `lisa validate`.

**Honest boundary:** Everything here is fixture-proven and FREE — stub executor, golden-byte materialize tests, addon-free unit tests, no tokens. The live metered cast that actually routes real codex work onto the board (and confirms lisa's codex seat picks up the stamped `agent:` key end to end) is the deferred, human-authorized step, named here rather than hidden inside a ticket. This epic only WRITES the seat onto the board; lisa already READS it, so no dispatch change is proven or needed (N4 — Not an executor).
