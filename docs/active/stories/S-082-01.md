**Scope:** The run ledger's write/read faces (src/log/run-log.ts: one new one-way cap/429 marker interface, normalized atomically on build and revived on read, following the seatDefaulted/seatInferred pattern) and the cast settlement shell (src/engine/cast.ts plus its pure classification helpers in src/engine/cast-core.ts) where a rate-limit-shaped executor failure is recognized and stamped onto the record castPlay already appends. lane-heat, budget, and wallet are untouched.

**Story acceptance:** A fixture cast through a stub executor that fails with a rate-limit-shaped error settles into runs.jsonl as exactly one record carrying the complete cap marker alongside its seatOfExecution (countable with jq); a marker-less record — including every revived historical line — stays byte-identical through build→serialize→revive; a partial or malformed marker is omitted atomically without losing the record; the existing run-log and cast suites pass unchanged.

**Honest boundary:** Fixture-proven and FREE end to end — a stub executor fabricates the 429-shaped failure; no live cast is burned to trigger a real provider cap. Real cap evidence accrues only from future live runs, so the learned denomination stays dormant until history contains cap events (PE-5, by design). Detection classifies at settle time only — no mid-run interception and no rerouting (N4 — Not an executor; E-071's deferred item (c) stays deferred).
