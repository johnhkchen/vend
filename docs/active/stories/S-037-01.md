Part of **epic E-037 (macro-wallet-live-proof)** — cast the headline autonomy gesture for real,
once, bounded. The wallet (E-024), priced casts (E-025), and the freshness gate (E-027) are built
and green; what's never been *watched* is the founding move: **fund a block, walk away, and Vend
spends it down across casts to a clean P7 stop** — and the forward-E1 trust evidence sits at a thin
**1/2 live**. This story de-risks, runs, and settles one bounded live spend.

The deliverable is a **watched metered run**, not a free proof (`castWork` is impure / "proven LIVE"
by design). The budget is the contract (P7); one session moves the forward-E1 count and sets the
cadence — it does **not** meet the ≥10-genuine bar alone, and must not over-claim it.

De-risk → run → settle: T-037-01 the **pre-flight** (free/deterministic — prove the run *will* clear:
price from the live ledger, a bounded budget affords ≥1 chain, `--no-intervened` threads the
forward-E1 bit, freshness math passes; go/no-go that auth==exec holds, E-025) → T-037-02 the **live
sweep** (metered — stage a fresh board, fund a bounded budget at the counter, `vend work
--no-intervened`, spend down to a clean stop; capture receipt + ≥1 cleared pull + forward-E1
records) → T-037-03 the **settle** (free — `vend audit` forward count up, assess QUALITY of what
cleared + P7 + auth==exec, honest verdict + named cadence to ≥10, no over-claim). Epic-scoped ids
(R6 — `037` clean).

## DAG

```
T-037-01  pre-flight (free) — prove the metered run WILL clear (price · affordability · forward-E1 thread · freshness)
   └─ T-037-02  the live sweep (METERED) — fresh board · bounded budget · vend work --no-intervened · clean P7 stop · ≥1 cleared
        └─ T-037-03  settle (free) — vend audit forward↑ · quality+P7+auth==exec · honest verdict + cadence · NO over-claim
```

**Out of this slice:** meeting the ≥10-genuine-forward bar in one session (cadence, named not
claimed); removing the P7 budget bound; hand-grounding what the sweep clears (autonomous clearing is
the point — quality is *assessed*, not pre-engineered); the 2M default budget (bounded ~1M here);
open-model parity (Claude executor).
