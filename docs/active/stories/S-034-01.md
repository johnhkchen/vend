Part of **epic E-034 (red-commit-audit)** — `check:history`: a bounded **linear** sweep that rebuilds +
tests each commit in a range (isolated worktree, the E-010 machinery) and flags any that wasn't green.
The post-hoc audit complement to E-033's pre-commit prevention — catches red commits that predate the
gate (the known-red `3dfb95f`) or slipped a bypass. **A linear walk, not a bisect** (red is
non-monotonic — green→red→green; bisect assumes monotonic). Bounded by default with a loud "covered N of
M" note (no silent caps). Pure core decides + reports; impure sweep does the worktree builds.

Build core → sweep: T-034-01 the pure `history-core.ts` (`classifyHistory` report + exit + honest-empty;
the bound helper), unit-tested → T-034-02 the `check-history.ts` sweep + `check:history` script
(generalize `buildCommittedHead` to a commit-ish; free proof flagging `3dfb95f`). Epic-scoped ids
(R6 — `034` clean).

## DAG

```
T-034-01  pure history-core.ts — classifyHistory(results) + the bounded-range/no-silent-cap helper
   └─ T-034-02  check-history.ts sweep (reuse E-010 worktree machinery) + check:history; free proof
```

**Out of this slice:** a true `git bisect run` integration (the monotonic "when did X first break"
question — a different tool, downstream); parallelizing the sweep (sequential first, speed later);
auto-fix / history rewrite (report only); per-commit-in-hot-path (E-033 owns prevention).
