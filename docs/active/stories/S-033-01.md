Part of **epic E-033 (per-commit-green-gate)** — the third gate layer: a git `pre-commit` hook runs
`check:test` before every commit and **blocks a red commit before it enters history**, naming the
failure (the E-008 style). Closes the hole E-031/T-031-01 exposed — `check:committed` (on-stop) only
checks source is committed, `check:head` (on-clear) only verifies HEAD builds at clear, so a mid-ticket
red commit survived. The three layers compose: per-commit (green) · per-stop (committed) · per-clear
(builds). Hook **only invokes**; policy lives in a pure core (the `ci-strategy.md` rule); **fail-open**
on inability to run, **fail-closed** on a genuine red.

Build core → wire: T-033-01 the pure `precommit-core.ts` (`classifyPrecommit` block/allow + reason +
message; `hookInstallState`), unit-tested → T-033-02 the `.githooks/pre-commit` invoker + `hooks:install`
+ the `check:hooks` guard, activated (free deterministic proof). Epic-scoped ids (R6 — `033` clean).

## DAG

```
T-033-01  pure precommit-core.ts — classifyPrecommit(runResult) + hookInstallState(hooksPath)
   └─ T-033-02  .githooks/pre-commit invoker + hooks:install + check:hooks guard; free live proof
```

**Out of this slice:** a retroactive `git bisect`-for-red-commits history sweep (prevention first,
post-hoc audit downstream); running the full `check` (with `baml:gen`) per commit (`check:head` owns the
full build at clear); a lisa `on-commit` hook (lisa has none — git pre-commit is the seam); changing
`check:committed`/`check:head` (this is the layer beside them).
