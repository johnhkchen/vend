**Scope:** The route-the-diff rung end to end: capture the working-tree diff a completed cast produced (cast.ts/cast-core.ts effect path, EffectResult/RunRecord), resolve the complement executor seat from the run's seatOfExecution (reusing the cast-core.ts:53-62 seat map + executor/select.ts executorFor), dispense an adversarial review prompt on that seat via the existing Executor.dispense (executor.ts:52-61 — no new interface method), and append the parsed pass/fail verdict to the run ledger (src/log/run-log.ts). A new cross-review module owns the routing; the reviewing seat is openai-compat's context-complete single-turn shape.

**Story acceptance:** Casting a play on seat A with a resolvable complement seat B, through a STUB reviewer executor, captures the run's diff, dispenses a review prompt on seat B, parses a structured pass/fail verdict, and appends a runs.jsonl line carrying the cross-vendor verdict (authoring seat, reviewing seat, pass/fail, detail); a run with no resolvable second seat records NO verdict and is otherwise unchanged (cross-review inert). All verified against a stub reviewer with no tokens spent.

**Honest boundary:** The recorded verdict is NON-BLOCKING in this story — it records, it does not yet refuse (enforcement is S-073-02). Everything is fixture-proven and FREE: stub reviewer executor, a temp-git working tree for the diff, no tokens. The live metered cross-vendor drive against a real openai-compat endpoint is the deferred metered step, authorized by the human at the counter — named here, never hidden inside a ticket. Reviewers that must explore the repo (agentic open-model runner) are out of bounds: review is context-complete single-turn only, matching the openai-compat seat's documented shape.
