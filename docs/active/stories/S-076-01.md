**Scope:** `src/cross-review/resolve-complement.ts` (the default registry's resolution semantics)
and the settlement wiring in `src/engine/cast.ts:376-397`, plus the run-record marker in
`src/log/run-log.ts`. The complement resolves to a live reviewer ONLY when the operator has
explicitly provisioned one; the built-in default registry must never fabricate a dialable
endpoint (`DEFAULT_OPENAI_BASE_URL`/localhost:11434 stops being reachable-by-default). When a
complement WOULD be relevant (lane-ful author, diff captured) but no reviewer is provisioned, the
gate is inert and the run record carries a `crossReviewSkipped` marker (the seatDefaulted /
seatInferred analog: why it was skipped, what would have to be true to bind).

**Story acceptance:** with default config, `resolveComplementExecutor` returns null (inert) for
every author seat — pinned in unit tests against the exact rc.4 shape (claude author, default
registry); an explicitly provisioned reviewer still resolves; a skipped review lands
`crossReviewSkipped` on the run record with a stated reason; the marker round-trips the ledger
read path like seatDefaulted does.

**Honest boundary:** FREE and unit-testable throughout — resolution and marker are pure;
no metered cast needed. What "provisioned" means (config file key, env, registry argument) is the
implementer's Research finding — follow the existing executor-config convention rather than
inventing a new surface; the ticket does not prescribe the mechanism, only that the DEFAULT is
inert.
