Part of **epic E-010 (committed-head-build-gate)** — verify the *committed HEAD*
itself builds (isolated checkout + `check`), catching an inconsistent-but-committed
commit like E-007's `cast.ts`-without-`play.ts` that the in-place `check` masks. The
build-soundness companion to E-008's commit-completeness gate.

*The pipeline running its own output: ProposeEpic minted this card; DecomposeEpic
(by hand) clears it.* Shared contract first (R12 → the `check:head` script). Epic-scoped
ids (R6 — `010` is clean).

**Design boundaries settled at decompose** (the card flagged both as "settle here"):
- **git worktree / temp checkout of HEAD, NOT Docker** — charter **P5 (local-first)**;
  the literal "container" in the signal is left optional.
- **a coarse trigger (on-clear / loop-end), NOT `on-stop`** — a full HEAD build on the
  per-turn `on-stop` hook would be catastrophic latency.

## DAG

```
T-010-01  check:head — isolated committed-HEAD build (git worktree, P5)
   └─ T-010-02  trigger it at a coarse boundary (on-clear, not on-stop)
```

**Out of this slice** (follow-ups): a debounce; a Dagger-CI-on-HEAD variant (E-002
family) for the fast-fleet case; auto-repair (no — detect only, per the card).
