**Scope:** The inferred-default-seat path end to end — a new pure reader `src/play/lane-heat.ts` (recent per-lane cost-weighted burn → cooler seat, reusing `KNOWN_SEATS` and run-log's `totalTokens`), the `seatInferred` provenance marker in `src/log/run-log.ts` (the overflow analog of `seatDefaulted`), and the injection in `src/play/decompose-effect.ts` (load ledger → infer when `--agent` omitted → stamp via `materialize`), with `EffectResult.seatInferred` (`src/engine/play.ts`) threaded onto the run record by `src/engine/cast.ts`. Both `decompose` (run) and `chain` mint through this one effect, so one injection covers both gestures.

**Story acceptance:** Casting a board-writing gesture with NO `--agent` against a ledger fixture showing the claude lane recently hot stamps `agent: codex` on every materialized ticket and writes a `seatInferred` marker (chosen seat + heat reason); a fixture with both lanes cool mints unrouted, byte-identical to today (no `agent:` key, no marker); an explicit `--agent claude` overrides the inference and stamps claude with no `seatInferred` marker. `bun run check` green.

**Honest boundary:** Fixture-proven and FREE — ledger fixtures + stub executor, no tokens; the live metered mint that closes the epic is the deferred step, authorized at the counter, named here not hidden in a ticket. Heat is RELATIVE recent burn between the two lanes, NOT an absolute quota fraction — no reset-window quota convention is sourceable in-repo today, so surfacing quota-per-reset-window as a first-class budget denomination is deferred to follow-on epic (d). Inference is at-mint only: no live lane monitor (N2 — Not a babysitting dashboard), no runtime 429 interception (N4 — Not an executor/follow-on c).
