Part of **epic E-066 (decompose-story-contract)** — make `decompose-epic` emit stories that are
contracts, and make a gate enforce it. The whole epic in one slice: the story type gains the five
contract fields (scope, acceptance, honest boundary, wave rationale, out-of-slice), the render
demands them with an exemplar, a completeness gate STOPs any shell before a file is written, the
materializer writes the full body instead of today's one-line shell, and the workflow doc gives the
story layer its job and readers so lisa injects the contract into every worker's context.

**Scope:** the decompose pipeline end to end — BAML schema + render (`decompose-bridge.ts`,
`decompose-epic-core.ts`), the play's gate list (`decompose-epic.ts`), the story writer
(`materialize.ts`), and one new section in `rdspi-workflow.md`. Ticket bodies and epic cards are
untouched; only the story artifact changes shape.

**Story acceptance:** casting `decompose-epic` against a fixture epic through a stub executor
yields story files carrying all five sections plus a DAG block consistent with the tickets'
`depends_on` edges — and replaying today's ten-line shell through the same cast is refused with a
`story-incomplete` andon, no file written.

**Honest boundary:** everything here is fixture-proven and FREE (stub executor, no tokens). The
live metered decompose cast and its EXPECTED-OUTCOME gold-master capture close the *epic*, not this
story — that cast is authorized by the human at the counter, named here as the deferred step, not
hidden inside a ticket.

## DAG

```
T-066-01-01  story contract fields in the decompose schema + render (keystone — settles the shape)
   ├─ T-066-01-02  story-completeness gate — a shell STOPs with a named andon, before the effect
   ├─ T-066-01-03  materialize writes the contract body (shell line demoted to provenance footer)
   └─ T-066-01-04  rdspi-workflow.md names the story layer's job + readers (lisa injects it)
```

Wave rationale: T-066-01-01 runs alone — all three consumers depend on the settled field shape.
T-066-01-02/-03/-04 then run in parallel: disjoint files (gate core / materialize / workflow doc),
no overlap, so one wave at `max_threads: 3` clears the story.

**Out of this slice:** automated charter-code resolution at cut time (snapshot-at-cut — its own
epic, converges with the honey-kitchen "strip N-codes" fix); story/epic status derivation and the
archive sweep (the sibling status-rot epic); backfilling shell stories already on any board
(history stays); per-ticket `agent:` routing (the routing epic consumes this story's output, it
does not belong to it); the live gold-master cast (deferred, metered, human-authorized — see
honest boundary).
